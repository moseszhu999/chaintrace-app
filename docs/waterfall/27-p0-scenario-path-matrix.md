# 27 P0 场景执行路径矩阵

## 1. 定位

本文件定义 ChainTrace P0 阶段的场景执行路径矩阵。

它的作用是：

> 明确不同业务场景应该走银行辅助路径、纯 DeFi 路径，还是 Agent + 智能合约低人工路径。

本文件只写业务规则和产品边界，不写页面、组件、数据库、API、Solidity、钱包签名、hash 或 proof 细节。

## 2. 路径定义

| 路径 | 核心含义 | 银行是否必需 | 人工参与 |
|---|---|---:|---|
| 银行辅助路径 | 银行、保理机构或资金方参与审查和授权。 | 是 | 中到高 |
| 纯 DeFi 路径 | 规则、资金池、抵押/保证和事实状态满足后由合约执行。 | 否 | 低 |
| Agent + 智能合约低人工路径 | Agent 整理事实和待办，合约按明确规则锁定、释放、结算或阻断。 | 否 | 很低，异常升级 |
| 人工升级路径 | 任一路径中出现争议、高风险、事实冲突或规则例外。 | 视场景而定 | 高 |

## 3. 路径选择原则

| 判断项 | 银行辅助路径 | 纯 DeFi 路径 | 低人工路径 |
|---|---|---|---|
| 是否需要授信判断 | 需要 | 不需要 | 通常不需要 |
| 是否需要银行放款授权 | 需要 | 不需要 | 不需要 |
| 规则是否清楚 | 可以不完全清楚，由人审查 | 必须清楚 | 必须清楚 |
| 资金池是否可自动约束 | 可作为执行条件 | 必须可约束 | 视资金动作而定 |
| 事实是否结构化 | 越结构化越好 | 必须足够结构化 | 必须结构化 |
| 风险是否高 | 可承接高风险审查 | 只适合低到中风险 | 只适合低风险或规则明确场景 |
| 是否存在重大争议 | 人工处理 | 人工升级 | 人工升级 |

## 4. P0 场景矩阵

| 场景 | 推荐路径 | 是否需要银行 | 必要事实 | 主要输出 |
|---|---|---:|---|---|
| 标准出口贸易融资 | 银行辅助路径 | 是 | 出口商材料、质检事实 | 允许放款 / 拒绝 / 补材料 / 人工复核 |
| 银行保理融资 | 银行辅助路径 | 是 | 贸易材料、买方付款责任、质检事实 | 授权放款或拒绝 |
| 高金额应收账款融资 | 银行辅助路径 | 通常是 | 贸易、质检、买方、保险或物流事实 | 人工审查和授权 |
| 新出口商首次融资 | 银行辅助路径 | 通常是 | 贸易材料、质检事实、风险摘要 | 人工复核或补材料 |
| 小额规则化融资 | 纯 DeFi 路径 | 否 | 贸易材料、质检或其他必要事实 | 自动执行 / 规则阻断 |
| 抵押充足的自动放款 | 纯 DeFi 路径 | 否 | 抵押/保证状态、基础贸易事实 | 执行 / 等待条件 / 阻断 |
| 资金池自动结算 | 纯 DeFi 路径 | 否 | 资金池、规则、事实状态 | 结算完成 / 资金池不足 |
| 保证金按条件释放 | Agent + 智能合约低人工路径 | 否 | 条件达成事实、无争议状态 | 释放 / 锁定 / 阻断 |
| 到期付款提醒和结算 | Agent + 智能合约低人工路径 | 否 | 到期日、付款责任、收货状态 | 提醒 / 结算 / 升级 |
| 法律条款明确的资金锁定 | Agent + 智能合约低人工路径 | 否 | 条款条件、事实状态、授权边界 | 锁定 / 释放 / 阻断 |
| 质检 fail | 人工升级路径 | 视原路径而定 | fail 原因、报告、照片 | 拒绝 / 补材料 / 争议处理 |
| 质检 dispute | 人工升级路径 | 视原路径而定 | dispute 原因、相关证据 | 人工复核 / 仲裁 / 阻断 |
| 买方提出争议 | 人工升级路径 | 视原路径而定 | 争议说明、订单、收货事实 | 暂停 / 补充 / 仲裁 |
| 资金池不足 | 人工升级或等待路径 | 否 | 资金池状态 | 等待 / 调整 / 阻断 |
| 平台暂停 | 人工升级路径 | 视场景而定 | 暂停原因、影响范围 | 阻断 / 恢复后继续 |

## 5. 银行辅助路径详细口径

### 5.1 进入条件

| 条件 | 说明 |
|---|---|
| 场景要求银行参与 | 授信、放款、保理或机构资金授权不可省略。 |
| 金额或风险较高 | 需要人做风险判断。 |
| 业务规则不足以自动执行 | 需要资金方审查材料和风险。 |
| 平台规则要求人工授权 | 当前规则不允许自动放款。 |

### 5.2 核心流程

```text
业务方创建融资意图
→ Agent 整理材料和风险摘要
→ 事实方确认必要事实
→ 资金方审查
→ 资金方允许 / 拒绝 / 补材料 / 人工复核
→ DeFi 智能合约检查执行条件
→ 执行或阻断
```

### 5.3 不允许

| 不允许 | 原因 |
|---|---|
| 合约绕过资金方授权 | 需要银行参与的场景中，资金方决策是前置条件。 |
| Agent 替资金方做高风险放款决策 | Agent 只能建议和整理。 |
| 平台替资金方审批 | 平台负责规则和运营，不负责放款审批。 |

## 6. 纯 DeFi 路径详细口径

### 6.1 进入条件

| 条件 | 说明 |
|---|---|
| 场景不要求银行审批 | 业务规则允许无银行执行。 |
| 规则明确 | 准入、额度、抵押/保证、风险边界清楚。 |
| 资金池可用 | 资金池状态允许执行。 |
| 事实状态满足 | 质检、物流、买方或其他必要事实已满足最低要求。 |
| 授权边界清楚 | 用户或平台预先授权范围明确。 |

### 6.2 核心流程

```text
业务方创建融资/结算意图
→ Agent 整理材料、事实和规则适配
→ 必要事实方确认事实
→ DeFi 智能合约检查规则、资金池、抵押/保证、暂停状态
→ 条件满足时执行
→ 条件不满足时阻断或等待
```

### 6.3 不允许

| 不允许 | 原因 |
|---|---|
| 把所有纯 DeFi 场景强制送银行审批 | 破坏场景分层。 |
| 合约在事实缺失时执行 | 事实来源不足。 |
| Agent 编造事实让规则通过 | 违反 Agent 边界。 |
| 规则例外由 Agent 自动绕过 | 规则例外必须升级人处理。 |

## 7. Agent + 智能合约低人工路径详细口径

### 7.1 进入条件

| 条件 | 说明 |
|---|---|
| 业务规则明确 | 条件可以被清楚表达。 |
| 授权边界明确 | 哪些动作可以自动推进已经预先授权。 |
| 事实来源结构化 | Agent 能整理，合约能使用状态结果。 |
| 争议处理路径明确 | 发生争议时进入人工或约定仲裁。 |

### 7.2 典型场景

| 场景 | 合约动作 | Agent 动作 |
|---|---|---|
| 保证金释放 | 锁定 / 释放 / 阻断 | 检查条件、提醒事实方、解释卡点。 |
| 付款责任到期 | 结算 / 等待 / 阻断 | 提醒买方、整理付款责任和争议状态。 |
| 条款明确的资金托管 | 锁定 / 释放 | 检查条件达成情况。 |
| 规则化运营费用结算 | 执行 / 阻断 | 汇总事实和生成待办。 |

### 7.3 不允许

| 不允许 | 原因 |
|---|---|
| Agent 自动裁决重大法律争议 | 重大争议需要人或约定仲裁。 |
| 合约替代真实法律判断 | 合约只执行预设规则和资金约束。 |
| 删除或覆盖关键记录 | 低人工不等于低审计。 |

## 8. 人工升级触发条件

以下情况无论原路径是什么，都必须升级人工：

| 触发条件 | 升级对象 |
|---|---|
| 贸易金额、主体、国家、品类明显冲突 | 出口商 / 资金方 / 平台方 |
| 质检 fail 或 dispute | 质检方 / 资金方 / 平台方 |
| 买方否认订单或收货 | 买方 / 出口商 / 资金方 |
| 物流损坏、丢失、改港或严重延误 | 物流方 / 出口商 / 资金方 |
| 保险拒保或出险争议 | 保险公司 / 出口商 / 资金方 |
| 资金池不足或异常 | 平台方 |
| 平台规则暂停 | 平台方 |
| 重大法律责任争议 | 人工角色 / 约定仲裁 |
| Agent 置信不足或越权风险 | 所属角色 / 平台方 |

## 9. MVP 范围

### 9.1 P0 必须完成

| 范围 | 说明 |
|---|---|
| 三类路径定义 | 银行辅助、纯 DeFi、低人工路径。 |
| 场景选择矩阵 | 常见 P0 场景能判断推荐路径。 |
| 银行可选原则 | 银行不是所有场景必经角色。 |
| 合约执行核心原则 | DeFi 智能合约负责资金执行和阻断。 |
| 人工升级原则 | 异常、争议、高风险、监管和重大法律责任交给人。 |

### 9.2 暂不做

| 范围 | 原因 |
|---|---|
| 具体合约函数 | 当前不写 Solidity 或 ABI。 |
| 具体风控模型 | 当前只定义业务口径。 |
| 具体 UI 路由 | 当前不做页面设计。 |
| 具体数据库字段 | 当前不做数据模型。 |

## 10. 本步结论

P0 场景执行路径矩阵明确：

> ChainTrace 不把银行作为所有场景的必经节点。需要银行判断的场景走银行辅助路径；规则、资金池和事实状态足够清楚的场景走纯 DeFi 路径；低风险、规则明确、授权边界清楚的结算、保证金和法律/运营场景走 Agent + 智能合约低人工路径。
