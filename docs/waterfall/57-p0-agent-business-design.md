# 57 P0 Agent 业务设计

## 1. 定位

本文件定义 ChainTrace P0 技术设计阶段的 Agent 业务设计。

它不是模型选型，不是提示词设计，不是工具调用方案，不是 Agent memory 结构，不是自动化代码，不是工作流引擎设计，也不是 UI 聊天组件说明。

它的作用是：

> 用业务语言定义 P0 中 Agent 是谁的助理、可以做什么、不可以做什么、什么时候能自动推进、什么时候必须人工确认或升级，以及 Agent 如何继承角色权限而不越权。

本文件只定义 Agent 的业务模式和边界，不进入实现。

## 2. Agent 业务定位

Agent 不是一个独立业务角色，也不是平台超级管理员。

Agent 在 ChainTrace 中的定位是：

> 每个业务角色身边的业务助理。

也就是说：

| 所属角色 | Agent 定位 |
|---|---|
| 出口商 Agent | 帮出口商准备融资案件、整理材料、发现缺口、跟踪到账。 |
| 资金方 Agent | 帮资金方阅读案件、整理风险、生成待办、提醒人工复核。 |
| 质检方 Agent | 帮质检方理解任务、检查报告完整性、提醒补充材料。 |
| 平台 Agent | 帮平台汇总异常、监控资金池/规则/合约状态、发现自动化堆积。 |
| 物流/保险/买方等 Agent | 帮对应事实方处理自己的事实确认任务。 |

Agent 的基本边界：

```text
Agent 可以辅助角色完成业务；
Agent 不可以取代角色成为事实制造者、资金审批人、争议裁决者或规则绕过者。
```

## 3. Agent 模式

每个角色可以选择三种 Agent 模式。

| 模式 | 业务含义 | 适用场景 |
|---|---|---|
| 全人工确认 | Agent 只提示和整理，关键动作都等用户确认。 | 新用户、高风险用户、重要案件、早期 MVP。 |
| 半自动托管 | Agent 自动处理常规整理和提醒，关键节点问用户。 | 大多数 P0 默认场景。 |
| 全自动 Agent | Agent 在授权范围内自动推进低风险流程，异常时提醒或升级。 | 事实完整、规则明确、风险低、用户授权清楚的场景。 |

模式边界：

- Agent 模式可以调整。
- Agent 模式调整不改变用户永久角色。
- Agent 模式越自动，人工升级条件越重要。
- 全自动不等于无限制自动。

## 4. Agent 权限继承

Agent 的权限来自所属角色。

| 原则 | 说明 |
|---|---|
| Agent 没有独立业务身份 | 它不是出口商、资金方、质检方之外的新角色。 |
| Agent 继承所属角色可见范围 | 出口商 Agent 只能看出口商有权看的案件。 |
| Agent 继承所属角色动作边界 | 所属角色不能做的动作，Agent 也不能做。 |
| Agent 不能因为自动化而扩大权限 | 自动化不是越权理由。 |
| Agent 高风险动作必须可回退人工 | 人可以接管、确认、拒绝或修改待办。 |
| Agent 行为必须可追踪 | 自动推进、请求补充、升级必须留下业务记录。 |

示例：

| 场景 | 是否允许 | 原因 |
|---|---|---|
| 出口商 Agent 整理出口商自己的发票 | 允许 | 属于出口商可见材料。 |
| 出口商 Agent 查看资金方内部风险模型 | 不允许 | 出口商无权查看。 |
| 资金方 Agent 要求出口商补材料 | 允许 | 资金方有权要求补材料。 |
| 资金方 Agent 修改出口商原始发票 | 不允许 | 资金方不能修改原始事实。 |
| 平台 Agent 暂停高风险自动化并提醒平台 | 允许 | 平台有系统运营职责。 |
| 平台 Agent 替资金方批准放款 | 不允许 | 平台无资金方审批权。 |

## 5. Agent 可以做什么

| 可做事项 | 业务说明 | 适用角色 |
|---|---|---|
| 整理材料 | 把合同、订单、发票、质检、物流等整理成摘要。 | 出口商、资金方。 |
| 生成业务摘要 | 让用户快速理解案件、事实、风险和卡点。 | 全部相关角色。 |
| 检查缺口 | 告诉用户缺什么材料或事实。 | 出口商、资金方、质检方、平台。 |
| 生成待办 | 把缺口和异常变成具体动作。 | 全部相关角色。 |
| 提醒状态变化 | 提醒提交、补材料、质检、放款、到账、逾期等。 | 全部相关角色。 |
| 请求相关角色补充 | 向质检、物流、买方、保险等请求事实补充。 | 授权范围内。 |
| 标记异常 | 发现金额不一致、质检 dispute、资金池不足、规则阻断等。 | 全部相关角色。 |
| 自动推进低风险流程 | 授权、事实完整、规则明确、风险低时继续下一步。 | 半自动/全自动。 |
| 升级人工处理 | 高风险、争议、置信不足、越权、规则冲突时交给人。 | 全部相关角色。 |
| 生成业务解释 | 把技术状态转成业务语言。 | 全部相关角色。 |

## 6. Agent 不可以做什么

| 不可做事项 | 原因 |
|---|---|
| 伪造贸易事实 | 贸易事实必须来自出口商或对应事实来源。 |
| 伪造质检事实 | pass/fail/dispute 必须由质检方确认。 |
| 伪造物流事实 | 物流节点必须来自物流方或可接受的事实来源。 |
| 伪造买方确认 | 买方订单、收货、付款责任必须由买方确认。 |
| 替资金方做高风险放款决策 | 高风险资金决策必须人工确认。 |
| 替平台绕过规则或暂停 | 平台规则和暂停状态是执行边界。 |
| 自动裁决重大争议 | dispute 必须人工处理或责任方处理。 |
| 删除或篡改关键记录 | 审计记录必须保留。 |
| 查看所属角色无权查看的案件 | 不能突破权限边界。 |
| 把建议伪装成事实或审批 | Agent 输出只是辅助，不是事实来源和最终决策。 |

## 7. Agent 输出类型

| 输出类型 | 业务含义 | 是否等于事实/决策 |
|---|---|---|
| 摘要 | 对材料、案件、状态的整理。 | 否。 |
| 缺口 | 还缺什么材料或确认。 | 否，但可触发待办。 |
| 建议 | 建议提交、补材料、人工复核、等待等。 | 否。 |
| 待办 | 某角色下一步要做什么。 | 否，但可进入业务流程。 |
| 风险提示 | 发现不一致、争议、阻断或高风险。 | 否。 |
| 自动推进记录 | Agent 在授权范围内完成了什么动作。 | 是自动化行为记录，不是事实结论。 |
| 人工升级记录 | 为什么需要人接管。 | 是升级原因记录。 |
| 业务解释 | 把状态翻译成用户能理解的话。 | 否。 |

关键边界：

```text
Agent 摘要 ≠ 原始材料
Agent 建议 ≠ 资金方审批
Agent 缺口 ≠ 拒绝结论
Agent 风险提示 ≠ 最终风险裁决
Agent 自动推进记录 ≠ 事实来源证明
```

## 8. 出口商 Agent 业务设计

### 8.1 出口商 Agent 可以做

| 动作 | 业务结果 |
|---|---|
| 帮出口商创建融资案件草稿 | 出口商不用从空白表单开始。 |
| 整理合同、订单、发票、装箱单 | 形成贸易摘要。 |
| 检查材料缺口 | 告诉出口商还缺什么。 |
| 提醒补充物流、质检、买方或保险信息 | 把缺口转成具体动作。 |
| 解释案件状态 | 告诉出口商卡在哪里。 |
| 建议是否提交 | 提示建议提交、补材料或人工确认。 |
| 跟踪资金结果 | 提醒放款中、到账、失败或等待。 |

### 8.2 出口商 Agent 不可以做

| 禁止动作 | 原因 |
|---|---|
| 替出口商伪造贸易材料 | 事实必须真实来源。 |
| 替质检方给出质检结论 | 越过事实方。 |
| 替资金方决定能不能放款 | 越过资金方。 |
| 绕过规则强制提交执行 | 破坏执行边界。 |
| 查看资金方内部审查细节 | 出口商不可见。 |

### 8.3 出口商 Agent 人工升级条件

| 条件 | 升级给谁 |
|---|---|
| 金额不一致且无法解释 | 出口商。 |
| 材料缺失影响提交 | 出口商。 |
| 质检 dispute | 出口商、质检方、资金方。 |
| 买方争议 | 出口商、买方、资金方。 |
| 规则阻断 | 出口商、平台。 |
| 放款执行失败 | 出口商、平台、资金方。 |

## 9. 资金方 Agent 业务设计

### 9.1 资金方 Agent 可以做

| 动作 | 业务结果 |
|---|---|
| 生成待审案件摘要 | 资金方先看业务摘要，不先翻文件堆。 |
| 检查证据完整性 | 合同、订单、发票、质检、物流等是否足够。 |
| 标记风险点 | 金额异常、质检 dispute、买方信息不足等。 |
| 建议处理动作 | 建议允许、补材料、人工复核或拒绝。 |
| 生成补材料请求 | 把缺口发给出口商或事实方。 |
| 跟踪执行条件 | 资金方批准后检查资金池、规则、暂停状态。 |
| 贷后提醒 | 回款、逾期、争议和理赔提醒。 |

### 9.2 资金方 Agent 不可以做

| 禁止动作 | 原因 |
|---|---|
| 替资金方自动批准高风险案件 | 高风险需要人工确认。 |
| 修改出口商原始材料 | 破坏证据可信。 |
| 冒充质检方或物流方确认事实 | 事实来源越界。 |
| 绕过资金池不足执行放款 | 合约/规则边界。 |
| 把 Agent 建议当最终审批 | 决策权属于资金方。 |

### 9.3 资金方 Agent 人工升级条件

| 条件 | 升级给谁 |
|---|---|
| 高金额案件 | 资金方人工。 |
| 高风险国家/品类/买方 | 资金方人工。 |
| 质检 fail/dispute | 资金方、质检方。 |
| 买方付款责任不清 | 资金方、买方。 |
| 保险不足 | 资金方、保险方。 |
| 合约执行失败 | 资金方、平台。 |

## 10. 质检方 Agent 业务设计

### 10.1 质检方 Agent 可以做

| 动作 | 业务结果 |
|---|---|
| 整理质检任务摘要 | 告诉质检方检查什么、在哪里、何时完成。 |
| 生成质检清单 | 货物品类、数量、包装、标签、照片等检查项。 |
| 检查质检材料完整性 | 报告、照片、结论、异常说明是否足够。 |
| 提醒补充照片或说明 | 防止资金方无法使用质检结果。 |
| 标记结论冲突 | 例如选择 Pass 但说明里写数量不一致。 |
| 汇总 dispute 点 | 方便质检方解释或复检。 |

### 10.2 质检方 Agent 不可以做

| 禁止动作 | 原因 |
|---|---|
| 代替现场质检 | 真实现场事实必须来自质检方。 |
| 自动伪造照片或报告 | 破坏可信事实来源。 |
| 自动裁决重大争议 | dispute 需要人工处理。 |
| 替资金方决定放款 | 质检方不是审批人。 |
| 修改出口商原始贸易材料 | 越权。 |

## 11. 平台 Agent 业务设计

### 11.1 平台 Agent 可以做

| 动作 | 业务结果 |
|---|---|
| 汇总平台运行状态 | 资金池、规则、合约、异常、Agent 堆积。 |
| 标记高风险异常 | 重复异常、疑似欺诈、执行失败、规则冲突等。 |
| 生成平台待办 | 谁处理什么异常、影响范围是什么。 |
| 建议暂停或降级自动化 | 出现误判、失败、风险堆积时提醒。 |
| 生成规则影响摘要 | 规则暂停或升级影响哪些案件。 |
| 监控 Agent 自动化失败 | 哪类自动化失败最多，是否需要人工接管。 |

### 11.2 平台 Agent 不可以做

| 禁止动作 | 原因 |
|---|---|
| 替平台直接执行高影响暂停/恢复 | 高影响动作建议人工确认。 |
| 替资金方批准放款 | 平台不是资金方。 |
| 替质检方出具结论 | 事实来源越界。 |
| 删除审计记录 | 审计不可破坏。 |
| 将正常案件全部交给平台逐单处理 | 破坏平台定位。 |

## 12. 事实方 Agent 业务设计

| 事实方 | Agent 可以做 | Agent 不可以做 |
|---|---|---|
| 物流方 | 提醒确认揽收、出港、到港、签收，整理异常说明。 | 伪造物流节点或签收记录。 |
| 保险方 | 整理核保材料、提醒补材料、汇总理赔材料。 | 自动承保高风险案件或伪造理赔事实。 |
| 买方 | 提醒确认订单、收货、争议、付款责任。 | 替买方确认付款责任。 |
| 海关/报关方 | 整理通关材料、标记监管异常。 | 伪造报关/清关事实。 |

P1/P2 事实方 Agent 在 P0 可保留占位，完整能力后续细化。

## 13. 自动推进条件

Agent 可以自动推进低风险流程，但必须同时满足以下条件。

| 条件 | 说明 |
|---|---|
| 用户已授权 | 所属角色选择半自动或全自动模式，并允许该类动作。 |
| 所属角色有权限 | Agent 不能触发所属角色无权动作。 |
| 案件属于可见范围 | Agent 只能处理所属角色可见案件。 |
| 事实完整 | 当前节点所需事实已经存在。 |
| 风险较低 | 不涉及高金额、高风险国家、重大争议或严重异常。 |
| 规则明确 | 规则输出清楚，不存在冲突。 |
| 无关键 dispute | 未存在未解决重大争议。 |
| 无平台暂停 | 平台、资金池、规则或案件没有暂停。 |
| 可回退人工 | 自动推进记录可追踪，人可接管。 |

## 14. 人工升级条件

出现以下情况，Agent 必须停止自动推进并升级给人。

| 条件 | 升级原因 | 责任方 |
|---|---|---|
| 高金额案件 | 资金风险高。 | 资金方。 |
| 高风险国家/品类/买方 | 风险不适合自动处理。 | 资金方、平台。 |
| 质检 Fail | 货物事实不支持自动推进。 | 资金方、质检方。 |
| 质检 Dispute | 事实不确定。 | 质检方、资金方。 |
| 金额不一致 | 材料之间冲突。 | 出口商、资金方。 |
| 买方争议 | 付款责任或收货状态不清。 | 买方、资金方。 |
| 保险不足 | 风险缓释不足。 | 保险方、资金方。 |
| 资金池不足 | 执行条件不足。 | 平台、资金方。 |
| 规则阻断 | 当前规则不允许继续。 | 平台。 |
| 平台暂停 | 系统级边界。 | 平台。 |
| 执行失败 | 执行层失败。 | 平台、资金方。 |
| Agent 置信不足 | 自动判断不可靠。 | 所属角色。 |
| 权限不清 | 动作责任角色不明确。 | 回流 waterfall 文档。 |

## 15. Agent 与合约的关系

| 场景 | Agent 可以做 | 合约负责 |
|---|---|---|
| 执行前准备 | 整理事实、检查缺口、提醒确认。 | 不参与材料解释。 |
| 触发执行检查 | 在授权和权限范围内请求检查。 | 检查执行条件。 |
| 规则阻断 | 解释阻断原因，生成待办或升级。 | 输出 Rule blocked。 |
| 资金池不足 | 提醒资金方、平台、出口商。 | 输出 Pool insufficient。 |
| 平台暂停 | 停止自动推进并提醒。 | 输出 Platform paused。 |
| 执行失败 | 升级平台或责任方。 | 输出 Execution failed。 |
| 执行完成 | 通知相关角色查看资金结果。 | 输出 Completed。 |

边界：

- Agent 不能绕过合约。
- 合约不解释业务材料。
- Agent 不能把合约输出改写成不同事实。
- 合约失败时 Agent 负责解释和分派，不负责伪造成功。

## 16. Agent 与数据的关系

| 信息类型 | Agent 如何使用 | Agent 不得做什么 |
|---|---|---|
| 原始材料 | 整理和摘要。 | 修改、伪造、覆盖原件。 |
| 事实节点 | 读取状态，提示缺口。 | 伪造事实。 |
| 资金方决策 | 读取和提醒。 | 替代资金方决策。 |
| 合约输出 | 翻译成业务语言。 | 改变输出含义。 |
| 异常记录 | 生成待办和升级。 | 删除异常历史。 |
| 审计记录 | 留下自动化行为记录。 | 无痕执行关键动作。 |

## 17. Agent 输出业务语言要求

Agent 输出必须是业务语言，不是技术错误。

| 不要说 | 要说 |
|---|---|
| Validation failed | 发票金额和订单金额不一致，需要出口商解释。 |
| Permission denied | 当前钱包注册为出口商，不能执行资金方审核动作。 |
| Risk score high | 该案件存在高风险，需要资金方人工复核。 |
| Workflow blocked | 当前案件卡在质检 dispute，需质检方补充说明。 |
| Execution failed | 放款执行失败，平台需要检查执行状态。 |
| Missing field | 还缺少物流文件或物流节点确认。 |

## 18. P0 必须覆盖的 Agent 能力

| 能力 | 是否 P0 必须 | 说明 |
|---|---|---|
| 材料摘要 | 必须 | 出口商和资金方都需要。 |
| 缺口检查 | 必须 | P0 主流程核心。 |
| 待办生成 | 必须 | 把异常变成动作。 |
| 状态提醒 | 必须 | 告诉用户卡点和结果。 |
| 补材料请求 | 必须 | 出口商、质检方、资金方之间协同。 |
| 人工升级 | 必须 | 防止自动化越界。 |
| 自动推进低风险流程 | P0 基础版 | 只在授权和低风险下。 |
| 平台 Agent 异常汇总 | P0 基础版 | 支撑平台运营。 |

## 21. 通过标准

| 检查项 | 通过标准 |
|---|---|
| Agent 定位清楚 | Agent 是角色助理，不是独立审批人。 |
| 权限继承清楚 | Agent 只继承所属角色权限。 |
| 可做事项清楚 | 整理、摘要、缺口、待办、提醒、请求补充、低风险推进、升级。 |
| 不可做事项清楚 | 不伪造事实、不越权、不绕过规则、不裁决重大争议。 |
| 自动推进条件清楚 | 授权、低风险、事实完整、规则明确。 |
| 人工升级条件清楚 | 高风险、争议、失败、置信不足、规则冲突等。 |
| 与合约边界清楚 | Agent 不绕过合约，合约不解释材料。 |
| 与数据边界清楚 | Agent 输出不等于事实或审批。 |
| 实现细节后置 | 未出现模型、提示词、工具、memory 或代码。 |

## 22. 一票否决项

| 一票否决项 | 原因 |
|---|---|
| Agent 被设计成独立超级用户 | 破坏权限模型。 |
| Agent 可以查看所属角色无权数据 | 数据越权。 |
| Agent 可以伪造贸易、质检、物流、买方或保险事实 | 破坏事实来源。 |
| Agent 可以自动批准高风险融资 | 破坏资金方责任。 |
| Agent 可以绕过合约或平台暂停 | 破坏执行边界。 |
| Agent 可以自动裁决重大 dispute | 破坏争议处理边界。 |
| Agent 输出被当成原始事实或最终审批 | 破坏业务责任。 |
| 文档直接进入模型、提示词、工具调用、memory 或代码 | 阶段越界。 |
| P1/P2 Agent 能力被塞入 P0 必做 | 破坏 MVP 范围。 |

## 23. 本步结论

P0 Agent 业务设计的本质不是“怎么实现 AI Agent”。

它的本质是：

> 在业务层定义 Agent 作为每个角色身边的助理，如何整理材料、生成摘要、检查缺口、生成待办、请求补充、低风险自动推进和异常人工升级，同时严格继承所属角色权限，不伪造事实、不替代审批、不绕过规则、不裁决重大争议。
