# 07 物流方 / 货代 / 承运方功能需求

## 1. 定位

物流方进入 ChainTrace，不是为了审批融资，也不是为了判断贸易是否应该放款。

物流方的核心定位是：

> 为融资案件提供真实、及时、可追溯的物流事实。

物流方是物流事实来源。它负责说明货物是否揽收、装柜、出港、到港、签收，以及是否发生延误、损坏、丢失、改港等异常。

资金方决定是否放款，平台维护规则和执行秩序，质检方提供货物质量事实。物流方只负责把运输过程讲清楚。

## 2. 进入条件

物流方进入本工作区前，必须已经完成：

| 条件 | 要求 |
|---|---|
| 钱包已连接 | 用户证明自己控制该钱包。 |
| 钱包已注册 | 该钱包已经完成 ChainTrace 注册。 |
| 角色已锁定 | 该钱包永久绑定为物流方 / 货代 / 承运方。 |
| 账户已激活 | 可以接收和处理物流确认任务。 |
| 服务能力已登记 | 已填写服务国家/地区、运输方式、支持文件类型等。 |

不处理登录前展示页。
不处理未注册用户的营销转化页面。

## 3. 物流方最想完成的业务结果

| 优先级 | 业务结果 | 说明 |
|---|---|---|
| P0 | 知道哪些物流任务需要确认 | 物流方不应该自己搜索融资案件。 |
| P0 | 确认货物当前运输状态 | 揽收、装柜、出港、到港、签收等状态必须清楚。 |
| P0 | 上传物流文件和现场材料 | 提单、运单、签收记录、照片、异常说明等。 |
| P0 | 说明物流异常 | 延误、损坏、丢失、改港等异常必须有原因和影响。 |
| P0 | 让出口商和资金方知道货物是否真实运输 | 物流事实支持贸易真实性判断。 |
| P1 | 处理补充说明请求 | 当资金方、出口商或平台认为物流事实不足时补充。 |
| P1 | 让 Agent 帮我检查材料完整性 | 不让物流方反复被要求补同类材料。 |

## 4. 物流方核心功能模块

### 4.1 待确认物流任务

功能要求：

| 功能 | 说明 |
|---|---|
| 查看待确认物流任务 | 只显示分配给该物流方或与其服务能力匹配的任务。 |
| 查看任务来源 | 出口商发起、资金方要求、平台异常分派或 Agent 请求。 |
| 查看货物摘要 | 货物品类、数量、包装、贸易双方、目的地。 |
| 查看运输要求 | 起运地、目的地、运输方式、预计时间。 |
| 接受或拒绝任务 | 无法确认时必须给出原因。 |
| 标记优先级 | 普通、加急、高金额、高风险、异常补充。 |

Agent 责任：

| Agent 动作 | 说明 |
|---|---|
| 生成物流任务摘要 | 告诉物流方需要确认哪批货、哪个运输节点。 |
| 提醒关键文件 | 根据运输方式提示提单、运单、签收记录、照片等。 |
| 标记融资影响 | 告诉物流方哪些缺口会影响资金方判断。 |
| 提醒截止时间 | 防止案件因为物流确认延迟而卡住。 |

### 4.2 运输状态确认

物流状态必须使用统一业务语言，避免每家物流方使用不同说法。

| 状态 | 含义 | 对融资案件的影响 |
|---|---|---|
| Pending pickup | 尚未揽收。 | 通常不能证明货物已出运。 |
| Picked up | 已揽收。 | 初步支持货物存在和运输启动。 |
| Loaded | 已装柜或装车。 | 支持货物进入运输环节。 |
| Departed | 已出港或离开发货地。 | 对出货后融资更有支持力。 |
| In transit | 运输中。 | 需要继续跟踪后续节点。 |
| Arrived | 已到港或到达目的地。 | 支持货物到达事实。 |
| Delivered | 已签收。 | 支持收货和付款责任判断。 |
| Delayed | 运输延误。 | 需要解释原因和预计影响。 |
| Damaged | 货物损坏。 | 可能触发质检、保险或争议流程。 |
| Lost | 货物丢失。 | 高风险异常，可能阻断融资或触发理赔。 |
| Rerouted | 改港或改路线。 | 需要解释原因、目的地变化和买方影响。 |

状态确认必须带来源：

| 来源 | 说明 |
|---|---|
| 提单 / 运单 | 证明运输安排和承运信息。 |
| 签收记录 | 证明到货或交付。 |
| 现场照片 | 证明货物、包装、装柜、损坏等现场事实。 |
| 物流系统记录 | 证明节点时间和运输轨迹。 |
| 异常说明 | 解释延误、损坏、丢失、改港等情况。 |

### 4.3 物流材料上传

功能要求：

| 材料类型 | 说明 |
|---|---|
| 提单 / Bill of Lading | 海运核心运输文件。 |
| 运单 / Waybill | 空运、陆运或快运运输文件。 |
| 装柜记录 | 证明货物装柜、装车或交接。 |
| 签收记录 | 证明买方、仓库或指定收货方已接收。 |
| 现场照片 | 货物、包装、箱号、封条、损坏点等。 |
| 运输状态记录 | 运输节点、时间、地点。 |
| 异常说明 | 延误、损坏、丢失、改港等原因和影响。 |

输入原则：

| 原则 | 说明 |
|---|---|
| 固定状态优先 | 运输状态必须从统一选项中选择。 |
| 文件上传优先 | 用已有物流文件和现场材料证明事实。 |
| 异常说明允许手写 | 异常细节无法完全标准化，可以手写。 |
| Agent 自动检查完整性 | 上传后检查是否缺关键节点或文件。 |
| 物流方最终确认 | 物流事实提交前必须由物流方确认。 |

### 4.4 物流异常处理

物流异常不是融资审批结论，但会影响资金方判断。

| 异常 | 处理方式 |
|---|---|
| 延误 | 说明原因、预计到达时间、是否影响交付。 |
| 损坏 | 上传照片、损坏说明，并提示质检或保险方介入。 |
| 丢失 | 标记高风险异常，说明丢失节点和责任信息。 |
| 改港 / 改路线 | 说明变更原因、目的地变化、是否得到买方确认。 |
| 文件不一致 | 说明提单、运单、订单、发票之间差异。 |
| 无法确认 | 说明无法确认的原因和可补充路径。 |

Agent 责任：

| Agent 动作 | 说明 |
|---|---|
| 识别异常类型 | 把自由说明转成明确异常分类。 |
| 生成补充任务 | 缺照片、缺签收、缺改港说明时生成待办。 |
| 通知相关角色 | 需要出口商、资金方、保险方或质检方知道异常。 |
| 阻止误表达 | 不把物流异常直接写成融资拒绝结论。 |

### 4.5 Agent 材料检查

物流方提交材料后，Agent 先检查是否足够给融资链条使用。

| 检查项 | 输出结果 |
|---|---|
| 是否确认运输状态 | 已确认 / 未确认 |
| 是否上传核心物流文件 | 已上传 / 未上传 / 不适用 |
| 是否覆盖关键节点 | 覆盖 / 部分覆盖 / 未覆盖 |
| 是否有签收或到港证据 | 有 / 无 / 不适用 |
| 异常是否有说明 | 有 / 无 / 不需要 |
| 文件信息是否一致 | 一致 / 不一致 / 无法判断 |
| 是否需要补充材料 | 不需要 / 需要补充 |

Agent 给物流方的表达必须是业务语言：

| 不要说 | 要说 |
|---|---|
| Missing logistics evidence | 还缺少提单或运单，资金方无法确认货物已进入运输。 |
| Invalid status | 你选择了已签收，但没有上传签收记录。 |
| Exception incomplete | 你标记了延误，但还没有说明预计到达时间。 |
| Mismatch detected | 运单目的地和发票目的地不一致，需要补充说明。 |

## 5. 物流方 Agent 模式差异

| 功能节点 | 全人工确认 | 半自动托管 | 全自动 Agent |
|---|---|---|---|
| 接收任务 | 用户手动接受 | Agent 推荐可接任务 | 授权范围内自动接受低风险确认任务 |
| 运输状态确认 | 用户手动选择 | Agent 根据材料建议状态，用户确认 | Agent 可生成状态草稿，异常仍需人确认 |
| 材料整理 | 用户手动上传 | Agent 自动整理文件摘要 | Agent 自动归类文件并检查缺口 |
| 异常说明 | 用户手写说明 | Agent 辅助生成结构化说明 | Agent 只整理异常，不自动编造事实 |
| 补充材料 | 用户处理 | Agent 生成补充任务 | Agent 自动提醒和请求补充 |
| 通知相关方 | 用户或系统通知 | Agent 建议通知对象 | Agent 自动通知低风险缺口和状态变化 |
| 高风险异常 | 用户人工处理 | Agent 汇总原因 | 必须交给人确认 |

重要边界：

| 边界 | 说明 |
|---|---|
| Agent 可以整理物流材料 | 可以归类提单、运单、签收、照片。 |
| Agent 可以检查缺口 | 可以提示缺状态、缺文件、缺异常说明。 |
| Agent 可以请求补充 | 可以向物流方或相关角色生成待办。 |
| Agent 不可以伪造运输事实 | 不能编造揽收、出港、签收等节点。 |
| Agent 不可以自动裁决重大异常 | 丢失、损坏、改港等必须人处理。 |

## 6. 权限边界

物流方可以：

| 权限 | 说明 |
|---|---|
| 查看分配给自己的物流任务 | 不能看无关融资案件。 |
| 查看必要贸易信息 | 只看完成物流确认所需信息。 |
| 确认运输状态 | 揽收、装柜、出港、到港、签收等。 |
| 上传物流文件和照片 | 为案件提供物流事实。 |
| 提交异常说明 | 对延误、损坏、丢失、改港等进行说明。 |
| 补充物流材料 | 对资金方、出口商或平台要求进行补充。 |

物流方不可以：

| 禁止事项 | 原因 |
|---|---|
| 修改自己的永久角色 | 钱包角色不可变。 |
| 决定是否放款 | 放款权属于资金方。 |
| 修改出口商贸易材料 | 防止证据污染。 |
| 伪造质检、海关、保险事实 | 物流方只负责物流事实。 |
| 查看无关融资案件 | 数据隔离。 |
| 修改资金方决策 | 决策权属于资金方。 |
| 修改平台规则 | 规则权属于平台/合约运营。 |

## 8. 本步结论

物流方工作区的本质不是“运输管理系统”。

它的本质是：

> 物流方作为独立物流事实来源，确认货物是否真实运输、运输到哪里、是否发生异常；Agent 帮它整理物流材料、检查缺口、请求补充，但不替代真实运输事实。

下一步应进入：

> 海关 / 报关 / 通关数据方功能需求。
