# 72 P0 实现阶段 2：Financing Case Core 计划

## 1. 定位

本文件定义 ChainTrace P0 受控实现阶段的第二个阶段：Financing Case Core。

它不是接口实现方案，不是数据库表设计，不是前端页面组件，不是资金方审批模块，不是材料上传模块，不是合约执行实现，也不是 Agent 自动化实现。

它的作用是：

> 在 Phase 1 身份与角色基础成立之后，细化 Phase 2 的业务范围、案件主对象、融资意图、融资目标、案件状态、路径占位、资金结果入口和阶段验收口径，确保后续实现先建立“融资案件”这一 P0 主线核心。

本阶段的核心目标不是“上传文件”，而是：

```text
出口商工作区
→ 创建融资意图
→ 形成融资案件
→ 表达融资目标
→ 表达案件主状态
→ 保留路径占位
→ 保留资金结果入口
```

## 3. Phase 2 阶段目标

Phase 2 必须让系统具备以下基础能力：

| 目标 | 说明 |
|---|---|
| 出口商能创建融资意图 | 融资意图是 P0 主线起点。 |
| 融资意图能形成案件 | 案件不是文件夹，而是融资业务对象。 |
| 融资目标可表达 | 金额、币种、融资类型、期望到账等业务目标可表达。 |
| 贸易基本信息可表达 | 出口商、买方、商品、贸易金额等基础信息可表达。 |
| 案件主状态可表达 | 草稿、准备中、待提交、已提交、补材料、审查中、执行中、完成等。 |
| 案件路径可占位 | 银行辅助、纯 DeFi、低人工、人工升级四类路径可表达。 |
| 资金结果入口可保留 | 后续可以接入资金结果和卡点表达。 |
| 案件权限可控制 | 出口商只能看到和处理自己的案件。 |

## 5. 融资案件核心对象范围

Phase 2 的中心对象是 Financing Case。

| 概念对象 | 本阶段用途 | 边界 |
|---|---|---|
| Financing Case | 表达一笔融资案件。 | 由出口商创建，受权限控制。 |
| Financing Intent | 表达出口商融资意图。 | 不等于资金方批准。 |
| Financing Target | 表达融资金额、币种、融资类型、期望到账。 | 后续资金方可建议调整，但原始目标应可追踪。 |
| Trade Basic Info | 表达出口商、买方、商品、贸易金额等基本信息。 | 不等于完整贸易材料。 |
| Case Status | 表达案件主状态。 | 必须可被控制塔理解。 |
| Case Path | 表达银行辅助、纯 DeFi、低人工、人工升级路径。 | Phase 2 只占位，不实现完整路径。 |
| Funding Result Entry | 表达资金结果入口。 | Phase 2 可占位，后续接入执行和结果。 |

## 6. 融资案件主状态范围

Phase 2 应定义并表达以下案件主状态。

| 状态 | 业务含义 | 允许下一步 |
|---|---|---|
| Draft | 出口商已创建草稿。 | 补融资目标和贸易基本信息。 |
| Preparing | 案件正在准备材料和基本信息。 | 进入 Phase 3 材料准备。 |
| Ready for materials | 案件已具备材料准备入口。 | 提交合同、订单、发票、装箱单。 |
| Waiting for materials | 等待出口商补材料。 | Phase 3 补材料。 |
| Submitted | 案件已提交进入审查或规则路径。 | Phase 5/6 后续处理。 |
| Under review | 资金方或规则路径处理中。 | 等待资金方/规则输出。 |
| Need review | 需要人工复核。 | 资金方或平台处理。 |
| Execution pending | 等待合约/规则执行检查。 | Phase 6。 |
| Funding completed | 资金执行完成。 | 查看资金结果。 |
| Rejected | 融资被拒绝。 | 查看拒绝原因。 |
| Blocked | 被规则、资金池、平台暂停或异常阻断。 | 查看卡点和责任方。 |

Phase 2 不必实现全部状态流转，但必须预留状态表达，不允许只用 created / failed。

## 7. 融资路径占位范围

Phase 2 必须保留四类路径占位。

| 路径 | Phase 2 表达 | 后续阶段 |
|---|---|---|
| Bank-assisted path | 案件可进入资金方审查路径。 | Phase 5。 |
| Pure DeFi path | 案件可进入规则/资金池执行路径。 | Phase 6。 |
| Low-human path | 案件可由 Agent 和规则低人工推进。 | Phase 7。 |
| Human-escalation path | 案件可因高风险、争议或失败升级人工。 | Phase 5/7/8。 |

一票否决：

```text
Phase 2 不得把案件路径固定成单一银行审批，也不得固定成无责任自动 DeFi。
```

| 案件入口 | 创建融资意图，查看自己的案件。 |
| 案件草稿 | 展示草稿状态和下一步。 |
| 融资目标 | 展示金额、币种、融资类型、期望到账。 |
| 贸易基本信息 | 展示买方、商品、贸易金额等基础信息。 |
| 案件状态 | 展示当前主状态。 |
| 路径占位 | 显示银行辅助、纯 DeFi、低人工、人工升级可能路径。 |

## 11. Agent 表达范围

Phase 2 只做 Agent 在案件主线上的轻量表达，不做完整 Agent Assist Layer。

| Agent 表达 | Phase 2 要求 |
|---|---|
| 下一步提示 | 根据案件状态提示下一步。 |
| 案件摘要占位 | 可显示案件基本摘要。 |
| 缺口检查占位 | 指向 Phase 3 材料缺口。 |
| 人工升级占位 | 标记后续可升级。 |

禁止：

- Agent 自动生成融资事实。
- Agent 自动提交完整案件。

| Contract execution | 后续阶段接入。 |

```text
这是一笔融资案件；当前处于融资链路的哪个位置；下一步需要做什么。
```

## 13. 资金结果入口范围

Phase 2 不实现真正资金结果，但必须保留入口。

| 资金结果状态占位 | 业务含义 |
|---|---|
| Not submitted | 还未提交，暂无资金结果。 |
| Waiting for materials | 需要补材料后才能进入融资判断。 |
| Waiting for review | 后续进入资金方或规则路径。 |
| Result pending | 后续由资金方/DeFi/合约输出。 |
| Result unavailable | 当前阶段尚未到资金结果节点。 |

禁止：

- 把案件创建成功当作融资成功。
- 把材料准备当作资金批准。
- 在未接入资金方/DeFi/合约前显示已到账。

## 19. 一票否决项

出现以下任一项，Phase 2 不得通过：

| 一票否决项 | 原因 |
|---|---|
| 非出口商可以创建融资案件 | 角色边界失败。 |
| 出口商可以查看其他出口商案件 | 数据隔离失败。 |
| 案件只是文件夹，没有融资意图和融资目标 | 产品主线错误。 |
| 案件创建成功被表达成融资成功 | 资金结果误导。 |
| 案件路径被固定成单一银行审批 | 路径差异失败。 |
| 纯 DeFi 路径被删除 | Crypto Native 失败。 |
| Agent 自动审批或自动执行资金路径 | Agent 越界。 |
| Phase 2 混入完整材料上传、质检、资金方审批或合约实现 | 阶段越界。 |
| 旧 proof/hash 页面成为案件主线 | 产品定位倒退。 |
| 直接写 endpoint、表结构、UI 组件或代码 | 当前文档阶段越界。 |

## 21. Phase 2 出口条件

Phase 2 完成后，系统应具备以下能力：

```text
1. 出口商可以创建融资意图。
2. 融资意图形成融资案件，而不是普通文件夹。
3. 案件具有融资目标和基础贸易信息。
4. 案件具有可理解的主状态。
5. 出口商只能查看和处理自己的案件。
6. 非出口商不能创建融资案件。
7. 案件保留银行辅助、纯 DeFi、低人工、人工升级路径占位。
8. 案件保留资金结果入口，但不误导为已融资或已到账。
9. 创建、更新、状态变化和越权阻断具备留痕语义。
```

满足以上条件后，才允许进入：

> `73-p0-implementation-phase-3-material-gap-flow-plan.md`

即细化 Phase 3：Material and Gap Flow，开始定义贸易材料、完整性、Agent 缺口和补材料待办的受控实现范围。

## 22. 本步结论

Phase 2 的本质不是“做一个文件上传页”。

它的本质是：

> 建立 ChainTrace P0 的融资案件主对象：出口商从业务角色工作区创建融资意图，形成有融资目标、基础贸易信息、主状态、路径占位和资金结果入口的融资案件，并严格保持出口商自己的案件可见、非出口商不可创建、案件创建不等于融资成功。

只有 Financing Case Core 成立，后续材料、质检、资金方、合约、Agent 和控制塔才能围绕同一条融资主线展开。
