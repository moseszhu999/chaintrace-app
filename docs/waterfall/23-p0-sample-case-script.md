# 23 P0 示例案件脚本

## 1. 定位

本文件用虚拟贸易融资案件串起 ChainTrace P0 MVP 的业务流程。

示例案件脚本不是测试代码，不是 demo 数据结构，也不是 UI 页面脚本。

它的核心作用是：

> 用具体但虚拟的贸易融资案件，验证银行辅助路径、纯 DeFi 路径、Agent、事实方、平台方和智能合约的业务协作是否能讲通。

本文件不写页面、组件、数据库、智能合约实现、钱包签名、hash 或 proof 细节。

## 2. 示例案件背景

| 项目 | 示例内容 |
|---|---|
| 案件编号 | CT-P0-CASE-001 |
| 出口商 | Asia Components Export Ltd. |
| 买方 | North Harbor Retail Inc. |
| 贸易品类 | 工业零部件 |
| 贸易金额 | USD 120,000 |
| 申请融资金额 | USD 80,000 |
| 融资类型 | 出货后融资 |
| 期望到账 | 尽快 |
| 质检方 | Independent Goods Inspection Co. |
| 资金方 | Example Trade Finance Bank |
| 平台方 | ChainTrace Operator |
| Agent 模式 | 出口商半自动托管，资金方全人工确认，质检方半自动托管，平台方半自动托管。 |

该案件只用于业务说明，不代表真实公司、真实贸易或真实资金流。

## 3. 银行辅助正常通过脚本

### 3.1 出口商创建融资意图

| 步骤 | 业务动作 | 状态 |
|---|---|---|
| 1 | 出口商声明要基于一笔出货后贸易申请融资。 | Draft |
| 2 | 出口商选择融资币种 USD、申请金额 USD 80,000、期望到账尽快。 | Draft |
| 3 | 出口商选择半自动托管 Agent 模式。 | Draft |

Agent 应输出：

```text
你正在准备一笔 USD 80,000 的出货后融资申请。
请上传合同、订单、发票、装箱单和质检相关材料。
```

### 3.2 出口商上传贸易材料

| 步骤 | 业务动作 | 状态 |
|---|---|---|
| 1 | 出口商上传合同、订单、商业发票、装箱单。 | Preparing materials |
| 2 | Agent 整理贸易金额、买卖双方、货物、付款条款。 | Preparing materials |
| 3 | Agent 发现质检材料尚未完成。 | Materials incomplete |

Agent 应输出：

```text
贸易材料基本齐全，但还缺少独立质检结论。
建议生成质检任务，等待质检方提交 pass / fail / dispute。
```

### 3.3 生成质检任务

| 步骤 | 业务动作 | 状态 |
|---|---|---|
| 1 | Agent 根据案件生成质检任务。 | Inspection pending |
| 2 | 质检方接收任务。 | Inspection pending |
| 3 | 质检方查看货物摘要和质检清单。 | Inspection pending |

质检方应看到：

```text
任务：检查 CT-P0-CASE-001 的工业零部件。
重点：品类、数量、包装、标签、外观、现场照片。
```

### 3.4 质检方提交 pass

| 步骤 | 业务动作 | 状态 |
|---|---|---|
| 1 | 质检方上传报告、现场照片、包装和标签照片。 | Inspection pending |
| 2 | 质检方选择 pass。 | Inspection submitted |
| 3 | Agent 检查质检材料完整。 | Inspection submitted |

Agent 应输出：

```text
质检结论为 Pass。
报告和照片足够支持资金方审查。
```

### 3.5 提交给资金方审查

| 步骤 | 业务动作 | 状态 |
|---|---|---|
| 1 | Agent 生成资金方审查摘要。 | Submitted |
| 2 | 出口商确认提交。 | Submitted |
| 3 | 资金方收到待审案件。 | Under review |

资金方应看到：

```text
案件摘要：USD 120,000 工业零部件贸易，申请融资 USD 80,000。
材料完整性：核心贸易材料完整，质检 Pass。
Agent 建议：可进入人工审批。
```

### 3.6 资金方允许放款

| 步骤 | 业务动作 | 状态 |
|---|---|---|
| 1 | 资金方查看摘要、完整性和风险提示。 | Under review |
| 2 | 资金方选择允许放款。 | Approved |
| 3 | 案件进入执行条件检查。 | Execution checking |

资金方应输出：

```text
资金方允许放款 USD 80,000。
下一步检查资金池、规则和平台状态是否允许执行。
```

### 3.7 执行条件通过并到账

| 步骤 | 业务动作 | 状态 |
|---|---|---|
| 1 | 平台/规则/合约业务层检查资金池正常。 | Execution checking |
| 2 | 当前规则有效，平台未暂停。 | Disbursement pending |
| 3 | 放款执行完成。 | Funds arrived |
| 4 | 案件关闭或进入后续跟踪。 | Closed |

出口商应看到：

```text
资金已到账。
本次融资金额：USD 80,000。
案件状态：Funds arrived。
```

## 4. 补材料分支脚本

### 4.1 发票金额不一致

| 步骤 | 业务动作 | 状态 |
|---|---|---|
| 1 | Agent 发现订单金额为 USD 120,000，发票金额为 USD 128,000。 | Materials incomplete |
| 2 | Agent 生成出口商补充说明待办。 | Need materials |
| 3 | 出口商上传金额差异说明。 | Preparing materials |
| 4 | Agent 重新检查后标记为可提交或需人工确认。 | Ready to submit / Manual review |

Agent 应输出：

```text
发票金额比订单金额高 USD 8,000。
请出口商说明差异原因，或上传修正后的发票/订单。
```

## 5. 质检 dispute 分支脚本

| 步骤 | 业务动作 | 状态 |
|---|---|---|
| 1 | 质检方提交 dispute。 | Inspection submitted |
| 2 | 原因：现场数量与装箱单不一致。 | Manual review |
| 3 | Agent 汇总争议点并提醒资金方人工复核。 | Manual review |
| 4 | 资金方要求出口商和质检方补充说明。 | Need materials |

资金方应看到：

```text
质检结论为 Dispute。
原因：现场数量与装箱单不一致。
建议动作：要求出口商补充数量差异说明，并要求质检方补充现场照片。
```

## 6. 规则阻断分支脚本

| 步骤 | 业务动作 | 状态 |
|---|---|---|
| 1 | 资金方已允许放款。 | Approved |
| 2 | 执行条件检查发现当前规则暂停该品类案件。 | Blocked |
| 3 | Agent 通知出口商和资金方阻断原因。 | Blocked |
| 4 | 平台方查看规则影响并决定维持暂停或恢复。 | Blocked / Execution checking |

用户应看到：

```text
当前案件被规则阻断。
原因：平台暂停了该品类的放款执行规则。
下一步：平台方处理规则状态，出口商等待结果。
```

## 7. 资金池不足分支脚本

| 步骤 | 业务动作 | 状态 |
|---|---|---|
| 1 | 资金方允许放款。 | Approved |
| 2 | 执行条件检查发现资金池余额不足。 | Execution waiting |
| 3 | Agent 提醒平台方和资金方处理资金池。 | Execution waiting |
| 4 | 资金池恢复后继续执行。 | Disbursement pending |

出口商应看到：

```text
资金方已允许放款，但当前资金池余额不足。
你不需要重复提交案件，等待资金池处理即可。
```

## 8A. 纯 DeFi 自动执行脚本

### 8A.1 示例背景

| 项目 | 示例内容 |
|---|---|
| 案件编号 | CT-P0-DEFI-001 |
| 业务方 | Asia Components Export Ltd. |
| 资金来源 | DeFi 自动融资池 |
| 贸易品类 | 工业零部件 |
| 贸易金额 | USD 30,000 |
| 申请融资金额 | USD 15,000 |
| 路径 | 纯 DeFi |
| 银行参与 | 不需要 |
| Agent 模式 | 全自动 Agent，异常升级人工。 |

### 8A.2 自动执行流程

| 步骤 | 业务动作 | 状态 |
|---|---|---|
| 1 | 出口商创建小额融资意图。 | Draft |
| 2 | Agent 整理合同、订单、发票和质检 pass。 | Preparing materials |
| 3 | Agent 判断材料完整、金额低于自动融资上限、无争议。 | DeFi checking |
| 4 | DeFi 智能合约检查资金池余额、规则、抵押/保证和暂停状态。 | Execution checking |
| 5 | 条件满足，智能合约执行资金动作。 | Disbursement pending |
| 6 | 资金到账。 | Funds arrived |

用户应看到：

```text
该案件走纯 DeFi 路径，不需要银行审批。
Agent 已完成材料和事实检查，DeFi 智能合约已确认规则和资金池条件满足。
资金已到账。
```

### 8A.3 低人工法律/结算分支

| 步骤 | 业务动作 | 状态 |
|---|---|---|
| 1 | 双方预先接受保证金释放规则。 | Draft |
| 2 | Agent 检查到收货和质检条件均满足。 | DeFi checking |
| 3 | 智能合约根据规则释放保证金。 | Disbursement pending |
| 4 | 若买方提出重大争议，Agent 升级人工或约定仲裁。 | Manual review |

业务解释：

```text
该场景不需要银行。
Agent 负责整理事实和提醒，智能合约负责按规则锁定或释放资金。
如果出现重大法律或责任争议，系统不会自动裁决，而是升级人工或约定仲裁。
```
