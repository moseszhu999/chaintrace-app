# 34 P0 DeFi 合约业务规则清单

## 1. 定位

本文件定义 ChainTrace P0 阶段需要由 DeFi 智能合约执行或约束的业务规则。

DeFi 合约业务规则清单不是 Solidity 设计，不是 ABI 设计，不是部署方案，也不是钱包签名流程。

它的作用是：

> 从 P0 业务规则中拆出哪些规则必须由 DeFi 智能合约在资金执行前检查、在资金执行中约束、在执行失败时阻断，并向业务角色返回可理解的业务输出。

本文件不写页面、组件、API、数据库、Solidity、ABI、部署、钱包签名、hash 或 proof 细节。

## 2. 合约业务规则原则

| 原则 | 说明 |
|---|---|
| 合约是资金执行约束层 | 合约负责资金动作前的规则、资金池、授权、暂停和执行条件检查。 |
| 合约不是审批人 | 需要银行参与的场景中，银行/资金方授权仍是前置条件。 |
| 合约不是事实来源 | 贸易、质检、物流、通关、保险、买方事实来自业务角色。 |
| 合约不是 Agent | 合约不整理材料、不解释完整业务上下文。 |
| 合约不裁决重大争议 | 法律责任、买方争议、质检 dispute 等进入人工或约定仲裁。 |
| 合约输出必须业务可读 | 不能只返回技术失败，必须能转成业务原因。 |
| 无银行场景可由合约执行 | 纯 DeFi 场景中，规则、资金池、授权和事实状态满足时可以执行。 |

## 3. 合约规则分类

| 规则类型 | 合约职责 |
|---|---|
| 执行准入规则 | 判断案件是否可以进入合约执行检查。 |
| 授权规则 | 判断银行授权、用户授权或平台授权是否满足。 |
| 资金池规则 | 判断资金池是否存在、可用、足额、未暂停、未异常。 |
| 规则版本规则 | 判断当前规则是否有效、适用、未暂停。 |
| 抵押/保证规则 | 判断纯 DeFi 或低人工场景的抵押/保证条件是否满足。 |
| 暂停规则 | 判断平台、资金池、规则、案件或角色是否暂停。 |
| 执行动作规则 | 判断放款、结算、锁定、释放、阻断是否可执行。 |
| 执行结果规则 | 返回允许、等待、阻断、失败、完成等业务结果。 |

## 4. 执行准入规则

| 规则 | 合约检查内容 | 业务输出 |
|---|---|---|
| 案件状态允许执行 | 案件是否处于 Approved、DeFi checking、Execution pending 或其他可执行状态。 | 允许继续 / 状态不允许执行 |
| 案件未关闭 | 案件不是 Closed、Rejected 或已终止状态。 | 允许继续 / 案件已关闭 |
| 必要事实已形成 | 业务系统已标记必要事实满足最低要求。 | 允许继续 / 等待事实 |
| 无未处理重大异常 | 业务系统未标记重大争议、高风险阻断或人工复核未完成。 | 允许继续 / 等待人工处理 |
| 场景路径匹配 | 银行辅助、纯 DeFi 或低人工路径与执行规则匹配。 | 允许继续 / 路径不匹配 |

合约不自行判断事实真假，只检查业务系统传入的事实状态是否满足执行条件。

## 5. 授权规则

| 场景 | 授权要求 | 合约业务结果 |
|---|---|---|
| 银行辅助路径 | 必须存在资金方允许放款或授权执行的结果。 | 授权满足 / 等待资金方授权 / 授权不足 |
| 纯 DeFi 路径 | 不需要银行授权，但需要用户、规则或资金池授权边界满足。 | 授权满足 / 授权不足 |
| 低人工路径 | 需要预设规则和用户授权边界满足。 | 授权满足 / 超出授权范围 |
| 平台暂停后恢复 | 需要平台恢复状态。 | 平台允许 / 平台暂停 |
| 授权金额限制 | 执行金额不得超过授权额度。 | 金额允许 / 超出授权金额 |
| 授权有效期 | 授权仍在有效期内。 | 授权有效 / 授权过期 |

合约授权规则不得：

| 不得 | 原因 |
|---|---|
| 在银行辅助路径中绕过资金方授权 | 资金方授权是银行辅助场景的前置条件。 |
| 把纯 DeFi 场景强制等待银行授权 | 纯 DeFi 场景不要求银行审批。 |
| 接受超额执行 | 不能超过授权金额或规则额度。 |

## 6. 资金池规则

| 规则 | 合约检查内容 | 业务输出 |
|---|---|---|
| 资金池存在 | 案件对应有效资金池。 | 资金池有效 / 资金池不存在 |
| 资金池未暂停 | 资金池不是暂停状态。 | 资金池可用 / 资金池暂停 |
| 资金池未异常 | 资金池不是冻结、维护、异常或失败处理状态。 | 资金池正常 / 资金池异常 |
| 余额足够 | 可用余额足以覆盖本次执行金额。 | 资金池足够 / 资金池不足 |
| 占用未超限 | 执行后占用不超过业务规则允许范围。 | 占用允许 / 占用超限 |
| 币种匹配 | 案件币种和资金池币种匹配。 | 币种匹配 / 币种不匹配 |
| 资金池适用范围匹配 | 资金池适用于该路径、国家、品类或业务类型。 | 适用 / 不适用 |

资金池规则命中异常时，业务系统必须能告诉平台方：

```text
哪个资金池受影响？
是余额不足、暂停、异常、占用超限还是币种不匹配？
哪些案件因此等待或阻断？
恢复后是否可以重新检查？
```

## 7. 规则版本规则

| 规则 | 合约检查内容 | 业务输出 |
|---|---|---|
| 当前规则有效 | 规则版本未过期、未废止。 | 规则有效 / 规则无效 |
| 规则未暂停 | 平台未暂停该规则。 | 规则可用 / 规则暂停 |
| 规则适用路径 | 规则适用于银行辅助、纯 DeFi 或低人工路径。 | 适用 / 路径不匹配 |
| 规则适用场景 | 规则适用于对应融资、结算、保证金、付款责任或资金托管场景。 | 适用 / 场景不匹配 |
| 规则适用范围 | 国家、币种、金额、品类、主体范围匹配。 | 匹配 / 范围不匹配 |
| 规则升级状态 | 规则未处于不可执行的升级或迁移状态。 | 可执行 / 等待规则处理 |

规则版本输出必须给平台方可处理原因，而不是只返回失败。

## 8. 抵押/保证规则

抵押/保证规则主要适用于纯 DeFi 和低人工路径。

| 规则 | 合约检查内容 | 业务输出 |
|---|---|---|
| 抵押存在 | 对应抵押或保证状态存在。 | 抵押有效 / 等待抵押 |
| 抵押覆盖足够 | 抵押或保证覆盖比例满足规则。 | 覆盖足够 / 覆盖不足 |
| 抵押未暂停 | 抵押或保证未处于冻结、争议或暂停状态。 | 可用 / 抵押暂停 |
| 保证条件满足 | 保证人、保证范围或保证金额满足业务规则。 | 保证满足 / 保证不足 |
| 抵押适用范围匹配 | 抵押或保证适用于当前案件、路径和金额。 | 匹配 / 不匹配 |
| 抵押释放条件满足 | 低人工场景中，释放条件已满足且无争议。 | 允许释放 / 等待条件 / 阻断 |

合约不负责评估抵押物真实价值，只检查业务系统已确认的抵押/保证状态是否满足规则。

## 9. 暂停规则

只要相关暂停状态生效，合约必须阻断资金执行。

| 暂停对象 | 合约响应 | 业务输出 |
|---|---|---|
| 平台暂停 | 阻断所有受影响执行。 | 平台暂停 |
| 路径暂停 | 阻断对应银行辅助、纯 DeFi 或低人工路径。 | 路径暂停 |
| 规则暂停 | 阻断适用该规则的执行。 | 规则暂停 |
| 资金池暂停 | 阻断对应资金池执行。 | 资金池暂停 |
| 案件暂停 | 阻断单个案件执行。 | 案件暂停 |
| 角色暂停 | 阻断相关角色触发或接收的执行。 | 角色暂停 |

暂停规则必须保留：

```text
暂停对象
暂停原因
影响范围
恢复条件
恢复后是否需要重新检查
```

## 10. 银行辅助路径合约规则

银行辅助路径中，合约只负责授权后的执行约束。

| 规则 | 合约检查内容 | 业务输出 |
|---|---|---|
| 资金方已授权 | 存在有效资金方允许放款结果。 | 授权满足 / 等待授权 |
| 授权金额足够 | 执行金额不超过授权金额。 | 金额允许 / 超出授权 |
| 授权未过期 | 授权仍在有效期内。 | 授权有效 / 授权过期 |
| 资金池满足 | 资金池状态和余额满足。 | 可执行 / 资金池不足 |
| 规则未阻断 | 当前规则允许该银行辅助场景执行。 | 可执行 / 规则阻断 |
| 无暂停状态 | 平台、案件、资金池、规则未暂停。 | 可执行 / 暂停阻断 |

银行辅助路径合约不得：

| 不得 | 原因 |
|---|---|
| 替资金方审批 | 审批属于资金方。 |
| 在拒绝融资后执行 | Rejected 状态不能执行。 |
| 在补材料未完成时执行 | Need materials 状态不能执行。 |
| 在人工复核未完成时执行 | Manual review 未完成不能执行。 |

## 11. 纯 DeFi 路径合约规则

纯 DeFi 路径中，合约是资金执行核心。

| 规则 | 合约检查内容 | 业务输出 |
|---|---|---|
| 无银行准入满足 | 场景不要求银行审批。 | 可进入执行检查 / 路径不允许 |
| 事实状态满足 | 必要事实状态满足最低要求。 | 事实满足 / 等待事实 |
| 规则准入满足 | 额度、路径、国家、币种、品类、主体范围满足规则。 | 规则通过 / 规则阻断 |
| 资金池满足 | 资金池可用且足额。 | 资金池通过 / 资金池不足 |
| 授权边界满足 | 用户、平台或规则授权范围允许。 | 授权满足 / 授权不足 |
| 抵押/保证满足 | 如场景要求，抵押或保证条件满足。 | 条件满足 / 条件不足 |
| 无暂停和异常 | 平台、路径、规则、资金池、案件未暂停。 | 允许执行 / 暂停阻断 |
| 全部条件满足 | 必要条件全部满足。 | 执行完成 / 执行失败 |

纯 DeFi 路径不能因为没有银行而进入等待资金方审批。

## 12. 低人工路径合约规则

低人工路径中，合约负责按明确规则锁定、释放、结算或阻断资金。

| 场景 | 合约规则 | 业务输出 |
|---|---|---|
| 保证金锁定 | 条件满足时锁定保证金。 | 锁定完成 / 锁定失败 |
| 保证金释放 | 条件达成、事实满足、无争议时释放。 | 释放完成 / 等待条件 / 阻断 |
| 付款责任结算 | 到期、责任确认、无争议时结算。 | 结算完成 / 等待付款责任 / 争议阻断 |
| 资金托管释放 | 条款条件满足且授权范围内。 | 释放完成 / 等待条件 |
| 自动阻断 | 出现争议、暂停、事实冲突或授权不足。 | 阻断并升级人工 |

低人工路径合约不得：

| 不得 | 原因 |
|---|---|
| 自动裁决重大法律争议 | 重大争议必须进入人工或约定仲裁。 |
| 在事实冲突时释放资金 | 事实冲突需要人工处理。 |
| 在授权不清时执行 | 授权边界不清必须等待或阻断。 |

## 13. 执行动作规则

| 动作 | 允许条件 | 不允许条件 |
|---|---|---|
| 放款 | 授权、规则、资金池、事实、暂停检查全部满足。 | 授权不足、资金池不足、规则阻断、暂停、事实缺失。 |
| 自动融资执行 | 纯 DeFi 准入、资金池、规则、授权、事实满足。 | 需要银行、事实缺失、风险阻断、资金池不足。 |
| 资金锁定 | 规则要求锁定且授权范围内。 | 规则不匹配、暂停、授权不足。 |
| 资金释放 | 释放条件满足、无争议、未暂停。 | 条件不足、争议、平台暂停。 |
| 结算 | 付款责任、到期、规则、资金池条件满足。 | 付款责任争议、资金池不足、规则阻断。 |
| 阻断 | 任一必要条件不满足。 | 不适用。 |

## 14. 执行结果规则

合约业务输出必须落到以下类型：

| 输出 | 含义 | 后续动作 |
|---|---|---|
| 允许执行 | 所有必要条件满足。 | 进入资金动作。 |
| 不允许执行 | 条件不满足且不能继续。 | 返回阻断原因。 |
| 等待条件 | 条件暂未满足但可补充。 | Agent 生成待办或等待。 |
| 规则阻断 | 当前规则不允许。 | 平台或人工处理。 |
| 资金池不足 | 资金池不能覆盖执行。 | 平台处理或等待资金池。 |
| 授权不足 | 银行、用户或规则授权不足。 | 重新授权或阻断。 |
| 平台暂停 | 平台、规则、路径、资金池或案件暂停。 | 等待恢复。 |
| 执行失败 | 执行动作尝试失败。 | 平台处理。 |
| 执行完成 | 资金动作完成。 | 通知相关角色并形成业务事件。 |

## 15. 合约不负责的规则

以下规则不应由合约自行判断：

| 不负责 | 原因 |
|---|---|
| 贸易真实性 | 来自出口商、买方、物流、通关等事实。 |
| 质检结论 | 由质检方提供。 |
| 物流事实 | 由物流方提供。 |
| 通关监管事实 | 由海关/报关方提供。 |
| 承保和理赔判断 | 由保险公司提供。 |
| 买方付款责任事实 | 由买方/进口商确认。 |
| 银行授信和审批判断 | 由资金方/银行/保理机构处理。 |
| Agent 摘要和解释 | 由 Agent 处理。 |
| 平台运营判断 | 由平台方处理。 |
| 重大法律责任争议 | 由人工角色或约定仲裁处理。 |

## 16. 与 Agent 的协作规则

| 协作点 | Agent 负责 | 合约负责 |
|---|---|---|
| 材料整理 | 整理摘要和缺口。 | 不处理材料。 |
| 事实状态整理 | 读取事实并生成业务摘要。 | 检查事实状态是否满足执行条件。 |
| 待办生成 | 根据合约输出生成待办。 | 返回等待、阻断、失败或完成原因。 |
| 自动推进 | 在授权范围内推进到执行检查。 | 判断能否执行。 |
| 异常升级 | 解释并分派给人。 | 阻断并返回业务原因。 |

Agent 不得绕过合约输出。

合约不得替 Agent 解释完整业务上下文。

## 17. 与平台方的协作规则

| 协作点 | 平台方负责 | 合约负责 |
|---|---|---|
| 规则启用/暂停 | 决定规则状态。 | 按规则状态执行或阻断。 |
| 资金池暂停/恢复 | 决定资金池状态。 | 按资金池状态执行或阻断。 |
| 案件暂停/恢复 | 决定案件是否暂停。 | 按案件状态执行或阻断。 |
| 执行失败处理 | 分析和处理失败。 | 返回失败结果。 |
| 规则版本迁移 | 决定业务迁移口径。 | 按当前有效规则检查。 |

平台规则变化必须影响合约执行结果。

## 18. MVP 范围

### 18.1 P0 必须定义

| 范围 | 说明 |
|---|---|
| 执行准入规则 | 哪些状态和条件允许进入合约检查。 |
| 授权规则 | 银行辅助、纯 DeFi、低人工路径的授权边界。 |
| 资金池规则 | 存在、足额、未暂停、未异常、未超限。 |
| 规则版本规则 | 有效、适用、未暂停、未迁移阻断。 |
| 抵押/保证规则 | 纯 DeFi 和低人工场景的条件约束。 |
| 暂停规则 | 平台、路径、规则、资金池、案件、角色暂停。 |
| 执行动作规则 | 放款、自动融资、锁定、释放、结算、阻断。 |
| 执行结果规则 | 允许、等待、阻断、失败、完成等业务输出。 |

### 18.2 暂不做

| 范围 | 原因 |
