# 54 P0 业务动作与接口边界

## 1. 定位

本文件定义 ChainTrace P0 技术设计阶段的业务动作与接口边界。

它不是接口规格，不是 API 文档，不是 endpoint 列表，不是请求响应结构，不是错误码设计，也不是接口实现计划。

它的作用是：

> 用业务语言定义 P0 中有哪些业务动作、由谁触发、需要什么业务前提、产生什么业务结果、失败时谁负责和下一步是什么，为后续接口规格设计提供业务边界。

本文件只定义业务动作和接口边界，不进入技术实现。

## 2. 边界原则

| 原则 | 说明 |
|---|---|
| 先定义业务动作 | 先说用户/Agent/合约/平台在业务上做什么。 |
| 不写 endpoint | 当前不定义接口路径、HTTP 方法或 RPC 方法。 |
| 不写请求响应 | 当前不定义 JSON、字段、类型或错误码。 |
| 动作必须有角色 | 每个业务动作必须有触发角色或系统责任方。 |
| 动作必须有前提 | 不能无条件触发关键动作。 |
| 动作必须有结果 | 每个动作输出业务结果，不只输出技术成功失败。 |
| 失败必须可解释 | 失败要有原因、责任方、解除条件和下一步。 |
| Agent 不越权 | Agent 只能触发所属角色授权范围内的动作。 |
| 合约不判断事实 | 合约动作只表达执行约束，不判断贸易真实性。 |
| 缺口必须回流 | 发现业务动作不清楚时回流 waterfall 文档。 |

## 3. P0 业务动作分组

| 动作组 | 目的 | 主要责任方 |
|---|---|---|
| 身份与注册动作 | 进入系统前确定钱包、角色、账户状态和 Agent 模式。 | 钱包用户、平台规则。 |
| 融资案件动作 | 出口商创建融资意图、准备材料、提交申请、查看结果。 | 出口商、Agent。 |
| 贸易材料动作 | 上传、补充、解释和锁定贸易材料。 | 出口商、Agent。 |
| 事实确认动作 | 质检、物流、保险、买方、海关等确认事实。 | 对应事实方。 |
| Agent 辅助动作 | 摘要、缺口、待办、提醒、请求补充、异常升级。 | 所属角色 Agent。 |
| 资金方决策动作 | 贷前审查、补材料、允许、拒绝、降额、人工复核。 | 资金方。 |
| 合约执行检查动作 | 检查规则、资金池、暂停状态、执行条件和输出结果。 | 合约/规则执行层。 |
| 平台运营动作 | 资金池、规则、合约、异常、Agent 自动化的暂停、恢复和维护。 | 平台方。 |
| 状态查询动作 | 角色查看自己有权看到的状态、结果和卡点。 | 各角色。 |
| 异常处理动作 | 缺材料、dispute、资金池不足、规则阻断、执行失败等处理。 | 对应责任方。 |

## 4. 通用业务结果类型

| 业务结果 | 含义 | 用户应理解 |
|---|---|---|
| Success | 动作完成。 | 当前动作已经完成，可以进入下一步。 |
| Waiting | 等待其他角色或条件。 | 不是失败，只是还缺外部确认或执行条件。 |
| Need materials | 缺材料。 | 需要补充指定材料或说明。 |
| Need confirmation | 需要人工确认。 | 当前动作不能自动继续，需要人确认。 |
| Need review | 需要人工复核。 | Agent 或规则无法安全自动处理。 |
| Permission denied | 权限不足。 | 当前角色不能做该动作。 |
| Role mismatch | 角色不匹配。 | 当前钱包角色不能执行该业务动作。 |
| Rule blocked | 规则阻断。 | 当前规则不允许继续。 |
| Pool insufficient | 资金池不足。 | 案件可能已通过，但暂时不能执行放款。 |
| Platform paused | 平台暂停。 | 平台或规则处于暂停状态。 |
| Dispute | 存在争议。 | 需要相关角色补充说明或人工处理。 |
| Execution failed | 执行失败。 | 执行层失败，需要平台或责任方处理。 |
| Completed | 闭环完成。 | 资金结果、事实确认或业务节点已经完成。 |

## 5. 身份与注册动作边界

| 业务动作 | 触发方 | 业务前提 | 业务结果 | 不进入内容 |
|---|---|---|---|---|
| 连接钱包 | 未连接用户 | 用户尚未连接钱包。 | 钱包已连接 / 连接失败。 | 钱包签名实现。 |
| 检查钱包注册状态 | 系统 | 钱包已连接。 | 未注册 / 已注册 / 受限 / 暂停。 | 查询接口实现。 |
| 选择唯一角色 | 未注册钱包用户 | 钱包未注册。 | 角色待确认。 | 前端选择组件。 |
| 提交注册信息 | 未注册钱包用户 | 已选择角色并填写必要信息。 | 注册成功 / 需补信息 / 角色冲突。 | 请求字段。 |
| 激活账户 | 系统 / 平台规则 | 注册信息满足要求。 | 已激活 / 受限 / 暂停。 | 账户状态字段。 |
| 设置 Agent 模式 | 注册用户 | 账户已激活。 | 全人工 / 半自动 / 全自动模式生效。 | Agent 配置实现。 |

业务边界：

- 注册前的钱包登录不等于业务账户激活。
- 一个钱包只能选择一个永久角色。
- 角色不可变，Agent 模式可以变。
- 未注册钱包不能进入业务工作区。

## 6. 融资案件动作边界

| 业务动作 | 触发方 | 业务前提 | 业务结果 | 不进入内容 |
|---|---|---|---|---|
| 创建融资意图 | 出口商 / 出口商 Agent | 钱包角色为出口商，账户已激活。 | 案件草稿创建 / 权限不足 / 需补信息。 | endpoint、请求体。 |
| 填写融资目标 | 出口商 / 出口商 Agent | 案件草稿存在。 | 目标金额、币种、到账期望被记录。 | 字段类型。 |
| 添加贸易基本信息 | 出口商 / 出口商 Agent | 案件草稿存在。 | 贸易双方、商品、金额、付款信息被记录。 | 数据表。 |
| 上传贸易材料 | 出口商 | 案件属于该出口商。 | 材料已接收 / 材料缺口仍存在。 | 文件存储实现。 |
| 请求 Agent 检查材料 | 出口商 / Agent | 材料已上传或信息已填写。 | 完整 / 不完整 / 需人工确认。 | 检查算法。 |
| 提交融资申请 | 出口商 / 授权 Agent | 最低材料要求满足或允许提交。 | 已提交 / 需补材料 / 规则阻断。 | 接口路径。 |
| 撤回草稿 | 出口商 | 案件未正式提交或允许撤回。 | 草稿撤回 / 不允许撤回。 | 状态字段。 |
| 查看融资结果 | 出口商 | 案件属于该出口商。 | 可融资 / 不可融资 / 补材料 / 放款中 / 已到账。 | 页面展示实现。 |

业务边界：

- 出口商只能创建和处理自己的案件。
- 已提交案件不能随意修改事实，只能补充或解释。
- 出口商不能替资金方审批，不能绕过规则触发放款。

## 7. 贸易材料动作边界

| 业务动作 | 触发方 | 业务前提 | 业务结果 | 不进入内容 |
|---|---|---|---|---|
| 上传合同 | 出口商 | 案件属于该出口商。 | 合同材料已接收 / 格式或内容需补充。 | 文件格式校验实现。 |
| 上传订单 | 出口商 | 案件属于该出口商。 | 订单材料已接收 / 缺关键信息。 | OCR 或解析实现。 |
| 上传发票 | 出口商 | 案件属于该出口商。 | 发票材料已接收 / 金额需解释。 | 字段提取。 |
| 上传装箱单 | 出口商 | 案件属于该出口商。 | 装箱信息已接收 / 数量需解释。 | 数据结构。 |
| 上传补充说明 | 出口商 | 存在异常、缺口或争议。 | 说明已提交 / 仍需补充。 | 富文本实现。 |
| Agent 生成材料摘要 | 出口商 Agent | Agent 可见该案件材料。 | 摘要生成 / 置信不足需人工确认。 | 模型调用。 |
| Agent 标记材料缺口 | 出口商 Agent | 材料可检查。 | 缺合同/发票/物流/质检/买方确认等。 | 规则算法。 |
| 锁定提交版本 | 系统 | 案件正式提交。 | 提交版本锁定。 | 版本存储实现。 |

业务边界：

- 材料来自出口商或相关事实方，Agent 只能整理和提示。
- 资金方不能修改出口商原始材料，只能要求补充或解释。
- 提交版本锁定后，后续只能补充记录，不应覆盖历史。

## 8. 事实确认动作边界

### 8.1 质检事实

| 业务动作 | 触发方 | 业务前提 | 业务结果 |
|---|---|---|---|
| 接收质检任务 | 质检方 | 任务分配给该质检方。 | 已接收 / 拒绝 / 需改派。 |
| 上传质检报告和照片 | 质检方 | 质检任务有效。 | 材料已提交 / 材料不足。 |
| 给出质检结论 | 质检方 | 已完成必要检查。 | Pass / Fail / Dispute。 |
| 补充质检材料 | 质检方 | 资金方/平台/Agent 要求补充。 | 补充完成 / 仍需补充。 |

### 8.2 物流事实

| 业务动作 | 触发方 | 业务前提 | 业务结果 |
|---|---|---|---|
| 确认揽收 | 物流方 | 物流任务与该物流方相关。 | 已揽收 / 无法确认。 |
| 确认装柜/出港/到港/签收 | 物流方 | 运输节点存在。 | 节点已确认 / 节点异常。 |
| 上传物流文件 | 物流方 / 出口商 | 案件需要物流事实。 | 提单/运单/签收记录已提交。 |
| 提交物流异常说明 | 物流方 | 存在延误、损坏、丢失、改港等。 | 异常已说明 / 需补材料。 |

### 8.3 保险事实

| 业务动作 | 触发方 | 业务前提 | 业务结果 |
|---|---|---|---|
| 查看待核保案件 | 保险方 | 案件需要保险判断。 | 可核保 / 需补材料。 |
| 给出核保结论 | 保险方 | 贸易、物流、质检、买方风险可评估。 | 承保 / 拒保 / 加费 / 补材料 / 人工核保。 |
| 处理理赔材料 | 保险方 | 发生出险或争议。 | 理赔受理 / 需补材料 / 拒赔。 |

### 8.4 买方事实

| 业务动作 | 触发方 | 业务前提 | 业务结果 |
|---|---|---|---|
| 确认订单相关性 | 买方 | 案件声称与该买方有关。 | 相关 / 不相关 / 需核实。 |
| 确认收货状态 | 买方 | 货物到达或需要确认。 | 已收货 / 未收货 / 部分收货 / 有争议。 |
| 提出 dispute | 买方 | 存在质量、数量、付款或交付争议。 | 争议已提交 / 需补说明。 |
| 确认付款责任 | 买方 | 付款义务需要确认。 | 确认 / 不确认 / 需核实。 |

业务边界：

- 事实确认动作只能由对应事实方执行。
- 事实方不决定融资放款。
- Agent 可以请求事实补充，但不能代替事实方伪造事实。

## 9. Agent 辅助动作边界

| 业务动作 | 触发方 | 业务前提 | 业务结果 | 不进入内容 |
|---|---|---|---|---|
| 生成案件摘要 | Agent | Agent 对该案件有可见权限。 | 摘要完成 / 置信不足。 | LLM 调用。 |
| 检查材料缺口 | Agent | 案件材料可见。 | 缺口清单 / 无缺口 / 需人工确认。 | 检查规则实现。 |
| 生成角色待办 | Agent | 存在缺口、异常或下一步动作。 | 待办生成 / 责任方不清需回流。 | 任务系统实现。 |
| 请求相关角色补材料 | Agent | 授权范围内且责任方明确。 | 请求已发出 / 权限不足 / 责任方不清。 | 消息系统实现。 |
| 提醒关键状态 | Agent | 案件状态发生关键变化。 | 提醒已生成。 | 通知实现。 |
| 自动推进低风险流程 | Agent | 授权、低风险、事实完整、规则明确。 | 已推进 / 需人工确认 / 被规则阻断。 | 自动化引擎。 |
| 升级人工处理 | Agent | 高风险、争议、失败、置信不足、规则冲突。 | 已升级给责任角色。 | 工单实现。 |

Agent 禁止触发：

| 禁止动作 | 原因 |
|---|---|
| 伪造贸易事实 | 事实必须来自责任角色。 |
| 伪造质检 pass/fail/dispute | 质检方负责结论。 |
| 高风险自动放款 | 资金方或明确规则路径负责。 |
| 自动裁决重大争议 | dispute 必须升级。 |
| 绕过规则和资金池状态 | 合约和规则是执行边界。 |
| 执行所属角色无权动作 | Agent 权限继承所属角色。 |

## 10. 资金方决策动作边界

| 业务动作 | 触发方 | 业务前提 | 业务结果 | 不进入内容 |
|---|---|---|---|---|
| 查看待审案件 | 资金方 | 案件分配或匹配给该资金方。 | 案件可见 / 无权限。 | 查询接口。 |
| 查看案件摘要 | 资金方 / 资金方 Agent | 资金方有处理权。 | 摘要可读 / 需补材料。 | UI 展示。 |
| 要求补材料 | 资金方 | 材料不足或异常未解释。 | 补材料请求发出。 | 消息格式。 |
| 允许放款 | 资金方 | 资金方有决策权，风险可接受。 | 已批准，进入执行条件检查。 | 审批接口。 |
| 拒绝融资 | 资金方 | 案件不符合融资要求。 | 已拒绝，并给出业务原因。 | 错误码。 |
| 降低融资金额 | 资金方 | 可做但需降低风险暴露。 | 调整建议发出 / 需出口商确认。 | 金额算法。 |
| 转人工复核 | 资金方 / Agent | 高风险、争议、置信不足。 | 已进入人工复核。 | 工单实现。 |
| 查看贷后状态 | 资金方 | 案件与该资金方相关。 | 回款/逾期/争议状态可见。 | 报表实现。 |

业务边界：

- 资金方可以决策融资，但不能修改事实来源材料。
- 资金方批准不等于合约一定可执行。
- 批准后仍需检查规则、资金池、暂停状态和执行条件。

## 11. 合约执行检查动作边界

| 业务动作 | 触发方 | 业务前提 | 业务结果 | 不进入内容 |
|---|---|---|---|---|
| 检查执行条件 | 合约/规则执行层 | 案件进入执行检查阶段。 | 允许 / 等待 / 阻断。 | 函数签名。 |
| 检查资金池状态 | 合约/规则执行层 | 放款需要资金池。 | 资金足够 / 不足 / 暂停。 | 合约结构。 |
| 检查规则版本 | 合约/规则执行层 | 案件适用规则。 | 有效 / 过期 / 暂停 / 不适用。 | 规则代码。 |
| 检查平台暂停状态 | 合约/规则执行层 | 可能存在平台暂停。 | 允许 / 平台暂停。 | 权限实现。 |
| 阻断执行 | 合约/规则执行层 | 条件不满足。 | 规则阻断 / 资金池不足 / 平台暂停。 | revert 细节。 |
| 输出执行结果 | 合约/规则执行层 | 执行动作完成或失败。 | 执行失败 / 执行完成。 | 交易细节。 |

业务边界：

- 合约输出业务执行结果，不判断贸易真实性。
- 合约不能替资金方审批，也不能替质检方出结论。
- 合约失败必须能解释责任方和下一步。

## 12. 平台运营动作边界

| 业务动作 | 触发方 | 业务前提 | 业务结果 | 不进入内容 |
|---|---|---|---|---|
| 查看运行总览 | 平台方 | 平台账户已激活。 | 资金池、规则、合约、异常、Agent 状态可见。 | 后台页面。 |
| 暂停资金池 | 平台方 | 资金池异常或系统级风险。 | 资金池暂停，影响范围明确。 | 合约调用实现。 |
| 恢复资金池 | 平台方 | 暂停原因解除。 | 资金池恢复。 | 权限代码。 |
| 暂停规则 | 平台方 | 规则存在风险或需维护。 | 规则暂停，影响案件明确。 | 规则引擎实现。 |
| 恢复规则 | 平台方 | 风险解除或维护完成。 | 规则恢复。 | 实现细节。 |
| 暂停案件 | 平台方 | 案件存在系统级异常或疑似欺诈。 | 案件暂停，责任方明确。 | 状态字段。 |
| 分派异常 | 平台方 / Agent | 异常责任方可判断。 | 异常分派给出口商、资金方、质检方等。 | 工单实现。 |
| 查看 Agent 异常 | 平台方 | Agent 自动化失败、堆积或误判。 | 异常可见，必要时降级人工。 | 日志实现。 |

业务边界：

- 平台不是逐单审批员。
- 平台不能替资金方批准放款。
- 平台不能伪造事实或删除关键记录。
- 高影响动作必须有业务原因和操作记录。

## 13. 状态查询动作边界

| 查询动作 | 触发方 | 可见前提 | 返回业务含义 | 不进入内容 |
|---|---|---|---|---|
| 查看账户状态 | 当前钱包用户 | 钱包已连接。 | 未注册 / 已激活 / 受限 / 暂停。 | 账户接口。 |
| 查看案件状态 | 出口商 / 资金方 / 平台 | 案件相关或异常相关。 | 草稿、已提交、补材料、审核中、批准、拒绝、放款中、到账。 | 状态字段。 |
| 查看事实状态 | 相关角色 | 与案件相关。 | 质检、物流、保险、买方、通关状态。 | 查询结构。 |
| 查看 Agent 待办 | 所属角色 | 待办属于该角色。 | 需确认、需补材料、需升级、已完成。 | 待办实现。 |
| 查看合约执行状态 | 相关角色 | 案件进入执行检查或执行阶段。 | 等待、阻断、资金池不足、平台暂停、失败、完成。 | 链上查询。 |
| 查看异常卡点 | 相关角色 / 平台 | 与案件或平台异常相关。 | 原因、责任方、下一步。 | 页面实现。 |

业务边界：

- 查询动作必须遵守角色可见范围。
- 查询结果应使用业务语言，不暴露无关内部细节。
- 平台可见异常，不等于平台可随意查看和修改正常案件。

## 14. 异常处理动作边界

| 异常 | 可触发动作 | 责任方 | 业务结果 |
|---|---|---|---|
| 缺合同/订单/发票 | 请求出口商补材料。 | 出口商。 | 材料补充 / 仍缺材料。 |
| 金额不一致 | 请求说明或修正材料。 | 出口商。 | 解释接受 / 需复核 / 拒绝。 |
| 质检 Fail | 阻断、拒绝或人工复核。 | 质检方、资金方。 | 拒绝 / dispute / 复检。 |
| 质检 Dispute | 请求补充或复检。 | 质检方、出口商、资金方。 | 争议解除 / 人工处理。 |
| 物流异常 | 请求物流说明。 | 物流方。 | 异常说明 / 仍阻断。 |
| 买方争议 | 请求买方确认和人工复核。 | 买方、资金方。 | 争议解除 / 拒绝 / 继续跟踪。 |
| 保险不足 | 要求保险补充或调整风险判断。 | 保险方、资金方。 | 承保 / 加费 / 拒保 / 补材料。 |
| 资金池不足 | 等待、切换路径或平台处理。 | 平台、资金方。 | 等待 / 恢复 / 失败。 |
| 规则阻断 | 平台复核或后置。 | 平台方。 | 恢复 / 继续阻断 / 后置。 |
| 平台暂停 | 等待恢复或人工处理。 | 平台方。 | 恢复 / 继续暂停。 |
| 执行失败 | 平台或执行责任方处理。 | 平台方、合约执行层。 | 重试 / 失败关闭 / 人工处理。 |
| Agent 置信不足 | 升级人工。 | 所属角色。 | 人工确认 / 继续等待。 |

## 15. 路径差异下的动作边界

| 路径 | 必须保留的动作 | 不允许变成 |
|---|---|---|
| 银行辅助路径 | 资金方审查、补材料、允许/拒绝、执行条件检查。 | 纯合约自动放款。 |
| 纯 DeFi 路径 | 规则、资金池、授权、暂停状态、执行输出。 | 必须银行审批。 |
| 低人工路径 | Agent 整理、缺口、低风险推进、合约执行、异常升级。 | 每一步都人工点击。 |
| 人工升级路径 | dispute、高风险、规则冲突、执行失败进入人工。 | Agent 自动裁决重大争议。 |

业务边界：

- 四条路径都必须保留。
- 技术设计不能把所有路径合并成银行审批流。
- 也不能把所有路径合并成无人负责的自动 DeFi 流。

## 16. P0 必须覆盖的业务动作

| 范围 | 必须覆盖动作 |
|---|---|
| 出口商 | 创建融资意图、上传材料、Agent 检查、提交申请、补材料、查看结果。 |
| 资金方 | 查看案件、查看摘要、要求补材料、允许、拒绝、转人工、查看执行状态。 |
| 质检方 | 查看任务、上传报告、给出 pass/fail/dispute、补充质检材料。 |
| 平台方 | 查看运行总览、查看异常、暂停/恢复资金池、暂停/恢复规则、处理 Agent 异常。 |
| Agent | 整理材料、生成摘要、检查缺口、生成待办、请求补充、提醒、升级。 |
| 合约/规则执行层 | 检查条件、检查资金池、检查规则、阻断、输出执行结果。 |
| 异常处理 | 缺材料、质检争议、资金池不足、规则阻断、平台暂停、执行失败。 |

## 19. 通过标准

| 检查项 | 通过标准 |
|---|---|
| 业务动作完整 | P0 角色、Agent、合约、平台和异常动作都有定义。 |
| 触发方清楚 | 每个动作知道由谁触发。 |
| 前提清楚 | 每个动作有业务前提。 |
| 结果清楚 | 每个动作输出业务结果。 |
| 失败可解释 | 失败有原因、责任方和下一步。 |
| 权限边界继承 | 动作没有突破 53 号权限边界。 |
| 路径差异保留 | 银行辅助、纯 DeFi、低人工、人工升级都保留。 |
| 实现细节后置 | 没有 endpoint、请求响应、错误码或代码。 |

## 20. 一票否决项

| 一票否决项 | 原因 |
|---|---|
| 业务动作直接写成 API endpoint | 阶段越界。 |
| 业务结果只有技术成功/失败 | 用户无法理解业务状态。 |
| 动作没有责任角色 | 责任不清，无法进入权限和验收设计。 |
| Agent 动作越过所属角色权限 | 破坏 Agent Native 边界。 |
| 合约动作判断贸易真实性 | 合约越界。 |
| 资金方动作允许修改原始事实 | 破坏证据可信。 |
| 平台动作允许替资金方批准 | 破坏决策权隔离。 |
| 所有路径被合并成单一审核流 | 破坏 DeFi/银行双路径。 |
| P1/P2 动作被塞入 P0 必做 | 破坏 MVP 范围。 |

## 21. 本步结论

P0 业务动作与接口边界的本质不是“写接口文档”。

它的本质是：

> 在业务层定义 ChainTrace P0 必须支持的动作、触发角色、业务前提、业务结果、失败解释、权限边界和路径差异，为后续接口规格设计提供清晰输入，同时阻止当前阶段提前进入 endpoint、请求响应、错误码和接口实现。

下一步应进入：

> `55-p0-data-information-domain-design.md`，定义 P0 数据信息域、事实来源、记录责任和可见范围，但仍不进入表结构、字段、索引或数据库实现。
