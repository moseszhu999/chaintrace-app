# 03 出口商 / 资金需求方功能需求

## 1. 定位

出口商不是来使用一个普通后台系统的。

出口商进入 ChainTrace 的核心目的只有一个：

> 用真实贸易事实更快获得融资，并清楚知道钱能不能借、能借多少、什么时候到账。

ChainTrace 对出口商而言，是银行/资金方前面的 Agent Native 融资入口。

## 2. 进入条件

出口商进入本工作区前，必须已经完成：

| 条件 | 要求 |
|---|---|
| 钱包已连接 | 用户证明自己控制该钱包。 |
| 钱包已注册 | 该钱包已经完成 ChainTrace 注册。 |
| 角色已锁定 | 该钱包永久绑定为出口商 / 资金需求方。 |
| 账户已激活 | 账户可以开始创建融资案件。 |

不处理登录前展示页。
不处理未注册用户的营销转化页面。

## 3. 出口商最想完成的业务结果

| 优先级 | 业务结果 | 说明 |
|---|---|---|
| P0 | 知道能不能借钱 | 系统必须直接告诉用户该案件是否有融资机会。 |
| P0 | 知道能借多少钱 | 用户需要看到可申请金额或建议融资区间。 |
| P0 | 知道还缺什么材料 | 材料不完整时，必须清楚告诉用户缺什么。 |
| P0 | 提交融资案件 | 用户最终要把贸易融资申请提交给资金方。 |
| P0 | 看到资金是否到账 | 融资成功后，用户最关心到账状态。 |
| P1 | 知道案件卡在哪里 | 如果没有通过，要知道是材料、质检、物流、买方、资金方还是平台规则问题。 |
| P1 | 让 Agent 替我处理重复工作 | 用户不应该手动整理一堆材料。 |

## 4. 出口商核心功能模块

### 4.1 融资意图确认

用户进入后，系统首先围绕一个业务问题展开：

> 你是否要基于一笔贸易申请融资？

功能要求：

| 功能 | 说明 |
|---|---|
| 创建融资意图 | 用户声明自己有一笔贸易需要融资。 |
| 选择贸易类型 | 例如订单融资、应收账款融资、发票融资、出货后融资。 |
| 选择融资币种 | 使用固定选项，不手写。 |
| 填写期望金额 | 可以手写金额，但币种必须固定选择。 |
| 填写期望到账时间 | 可以选择“尽快 / 指定日期 / 可协商”。 |
| 选择 Agent 模式 | 全人工确认、半自动托管、全自动 Agent。 |

Agent 责任：

| Agent 动作 | 说明 |
|---|---|
| 识别融资类型 | 根据用户输入和贸易材料判断更适合哪类融资。 |
| 提醒基础缺口 | 用户还没提交材料时，就先告诉他大概需要哪些证据。 |
| 生成案件准备清单 | 不让用户自己猜需要什么。 |

### 4.2 贸易证据准备

出口商需要提供贸易真实性证据，但不应该由人逐项整理。

功能要求：

| 证据类型 | 是否核心 | 说明 |
|---|---:|---|
| 合同 / Purchase Contract | 核心 | 证明交易双方和交易条款。 |
| 订单 / Purchase Order | 核心 | 证明买方采购意图。 |
| 发票 / Commercial Invoice | 核心 | 证明金额、品类、付款义务。 |
| 装箱单 / Packing List | 核心 | 证明货物数量和包装。 |
| 物流文件 | 核心 | 证明货物运输状态。 |
| 质检报告 / 照片 | 核心 | 证明货物质量或数量状态。 |
| 买方信息 | 核心 | 证明付款方身份和风险。 |
| 保险信息 | 可选 | 有保险时可降低风险。 |
| 报关 / 清关信息 | 可选或后续增强 | 有接入能力时可增强贸易真实性。 |
| 补充说明 | 可选 | 解释异常、特殊条款或背景。 |

输入原则：

| 原则 | 说明 |
|---|---|
| 文件上传优先 | 用户上传已有材料。 |
| 固定选项优先 | 国家、币种、语言、贸易品类、运输方式等使用选择项。 |
| 手写输入兜底 | 只有地址、备注、说明等允许手写。 |
| Agent 自动整理 | Agent 从材料中提取摘要、金额、日期、交易方、货物、付款条款。 |
| 用户只确认关键结果 | 不要求用户逐字段手工录入。 |

### 4.3 Agent 材料检查

出口商提交材料后，Agent 先做融资案件准备检查。

功能要求：

| 检查项 | 输出结果 |
|---|---|
| 材料是否完整 | 完整 / 不完整 / 需要人工确认 |
| 金额是否一致 | 一致 / 不一致 / 无法判断 |
| 买卖双方是否一致 | 一致 / 不一致 / 缺少信息 |
| 物流状态是否支持融资 | 支持 / 不支持 / 需要补充 |
| 质检状态是否支持融资 | Pass / Fail / Dispute / 未质检 |
| 买方信息是否足够 | 足够 / 不足 / 需要资金方判断 |
| 风险是否明显过高 | 正常 / 警示 / 阻断 |

Agent 给出口商的表达必须是业务语言：

| 不要说 | 要说 |
|---|---|
| Validation error | 发票金额和订单金额不一致。 |
| Missing field | 还缺少物流文件。 |
| Risk gate failed | 这个案件暂时不建议提交，因为质检结果存在争议。 |
| Workflow blocked | 当前案件卡在质检补充材料。 |

### 4.4 融资申请提交

当材料达到最低要求后，出口商可以提交融资案件。

功能要求：

| 功能 | 说明 |
|---|---|
| 查看提交前摘要 | 用户看到贸易金额、申请金额、买方、货物、状态、缺口。 |
| Agent 建议 | Agent 给出“建议提交 / 建议补材料 / 不建议提交”。 |
| 用户确认提交 | 半自动或人工模式下，需要用户确认。 |
| 全自动提交 | 全自动模式下，满足授权条件时 Agent 可以提交。 |
| 提交后锁定案件版本 | 提交后的案件进入资金方视角，不再随便修改。 |

提交前必须回答出口商四个问题：

```text
1. 我这笔贸易有没有融资机会？
2. 我现在提交会不会被拒？
3. 还缺什么会影响放款？
4. 提交后谁会处理？
```

### 4.5 案件状态追踪

出口商提交后，不应该看复杂流程图，而应该看到业务状态。

状态要求：

| 状态 | 出口商理解 |
|---|---|
| Draft | 案件还在准备中。 |
| Ready to submit | 材料基本够，可以提交。 |
| Submitted | 已提交给资金方。 |
| Need materials | 需要补材料。 |
| Under review | 资金方正在审查。 |
| Inspection pending | 等待质检。 |
| Inspection dispute | 质检有争议。 |
| Approved | 资金方允许融资。 |
| Rejected | 融资被拒绝。 |
| Disbursement pending | 等待放款执行。 |
| Funds arrived | 资金已到账。 |
| Blocked | 因规则、风险或平台状态被阻断。 |

每个状态都要带一句人话解释：

```text
当前状态：Need materials
解释：资金方需要你补充物流单据，否则无法继续审核。
下一步：上传物流文件，或让 Agent 帮你联系物流方补充。
```

### 4.6 补材料与异常处理

出口商不应该自己判断异常属于谁。

功能要求：

| 异常类型 | 处理方式 |
|---|---|
| 缺合同 / 订单 / 发票 | 出口商补充。 |
| 缺物流事实 | 出口商补充，或 Agent 请求物流方确认。 |
| 缺质检证据 | Agent 生成质检任务，质检方处理。 |
| 买方信息不足 | 出口商补充，或邀请买方确认。 |
| 金额不一致 | 出口商解释或上传修正材料。 |
| 质检 fail | 案件进入拒绝或争议处理。 |
| 质检 dispute | 出口商、质检方、资金方共同处理。 |
| 资金池不足 | 出口商等待或选择其他资金方。 |
| 规则阻断 | 平台或资金方处理，出口商看到原因。 |

Agent 责任：

| Agent 动作 | 说明 |
|---|---|
| 告诉用户缺什么 | 不让用户猜。 |
| 告诉用户找谁补 | 出口商、物流方、质检方、买方、资金方。 |
| 生成补材料任务 | 把异常转成具体动作。 |
| 完成后重新检查 | 补完自动回到材料检查。 |

### 4.7 融资结果查看

出口商最关心结果，不关心内部术语。

功能要求：

| 结果 | 说明 |
|---|---|
| 可融资 | 资金方愿意进入放款流程。 |
| 不可融资 | 当前案件不满足融资条件。 |
| 需要补材料 | 材料不完整，但有机会继续。 |
| 需要人工复核 | 案件存在异常，但未必失败。 |
| 等待资金池 | 案件通过，但资金池或执行条件未满足。 |
| 放款中 | 已进入放款执行。 |
| 已到账 | 出口商资金账户已收到款。 |

出口商最终必须看到：

```text
能不能借：可以 / 不可以 / 需要补材料
建议融资金额：xxx
预计到账时间：xxx
当前卡点：xxx
下一步动作：xxx
```

## 5. 出口商 Agent 模式差异

| 功能节点 | 全人工确认 | 半自动托管 | 全自动 Agent |
|---|---|---|---|
| 创建融资意图 | 用户手动确认 | 用户确认一次 | Agent 可根据授权创建草稿 |
| 整理材料 | Agent 辅助，用户逐项确认 | Agent 自动整理，关键项确认 | Agent 自动整理 |
| 检查缺口 | Agent 提示，用户处理 | Agent 生成任务 | Agent 自动请求相关方补充 |
| 提交融资案件 | 必须用户确认 | 建议提交时用户确认 | 授权范围内自动提交 |
| 补材料 | 用户自己上传 | Agent 指导用户补 | Agent 自动分发给相关角色 |
| 查看状态 | 用户主动查看 | Agent 推送关键状态 | Agent 只推送异常和结果 |
| 放款结果 | 用户查看 | Agent 提醒 | Agent 提醒到账结果 |

## 6. 权限边界

出口商可以：

| 权限 | 说明 |
|---|---|
| 创建自己的融资案件 | 只能创建自己作为资金需求方的案件。 |
| 上传自己的贸易材料 | 不能上传别人的案件材料。 |
| 查看自己的案件状态 | 只能看自己相关案件。 |
| 补充说明和材料 | 可以解释异常。 |
| 接受或取消提交 | 取决于案件状态。 |
| 查看融资结果 | 包括通过、拒绝、补材料、到账。 |

出口商不可以：

| 禁止事项 | 原因 |
|---|---|
| 修改自己的永久角色 | 角色绑定钱包后不可变。 |
| 冒充资金方审核 | 防止利益冲突。 |
| 冒充质检方出具结论 | 质检必须来自独立角色。 |
| 查看其他出口商案件 | 数据隔离。 |
| 直接修改资金方结论 | 决策权属于资金方。 |
| 直接修改平台规则 | 规则权属于平台/合约运营。 |

## 7. MVP 范围

### P0 必须做

| 功能 | 说明 |
|---|---|
| 出口商角色进入 | 已注册出口商钱包进入出口商工作区。 |
| 创建融资意图 | 形成案件草稿。 |
| 上传贸易材料 | 支持合同、订单、发票、物流、质检等材料。 |
| Agent 材料完整性检查 | 告诉用户缺什么。 |
| 提交融资申请 | 提交给资金方处理。 |
| 查看案件状态 | 用业务语言展示状态。 |
| 补材料 | 用户根据 Agent 提示补充。 |
| 查看融资结果 | 通过、拒绝、补材料、到账。 |

## 8. 本步结论

出口商工作区的本质不是“上传文件页面”。

它的本质是：

> 出口商提出融资需求，Agent 帮他把贸易事实变成可审查的融资案件，并持续告诉他能不能借、缺什么、卡在哪里、钱有没有到账。

下一步应进入：

> 资金提供方 / 银行 / 保理机构功能需求。
