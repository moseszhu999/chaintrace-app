# 38 P0 原型验收脚本

## 1. 定位

本文件定义 ChainTrace P0 阶段的原型验收脚本。

原型验收脚本不是自动化测试代码，不是 UI 测试用例，不是页面点击脚本，也不是 demo 数据结构。

它的作用是：

> 用具体业务走读方式检查控制塔式原型、角色权限、Agent 待办、银行辅助路径、纯 DeFi 路径、低人工路径、异常解释和 DeFi 执行输出是否能被业务用户理解。

本文件不写页面、组件、接口路径、请求响应格式、数据库、Solidity、ABI、钱包签名、hash 或 proof 细节。

## 2. 验收脚本原则

| 原则 | 说明 |
|---|---|
| 按业务走读 | 验收方式是业务角色走读，不是技术测试。 |
| 按控制塔表达 | 每一步都检查控制塔是否能表达节点、状态、卡点和下一步。 |
| 按角色权限 | 每个角色只能看到自己可见范围。 |
| 按路径差异 | 银行辅助、纯 DeFi、低人工路径必须明显不同。 |
| 按 Agent 边界 | Agent 能整理、提醒、生成待办和低风险推进，不能越权。 |
| 按合约输出 | DeFi 执行结果必须是业务可理解的允许、等待、阻断、失败或完成。 |
| 按异常回流 | 补材料、争议、规则阻断、资金池不足、执行失败都必须有责任方和下一步。 |

## 3. 原型验收总览

| 验收场景 | 核心检查点 | 通过标准 |
|---|---|---|
| 控制塔全链路 | 从源头到终端的节点流转。 | 能看清当前节点、已完成节点、等待节点和卡点。 |
| 出口商视角 | 自己案件端到端流转。 | 看到自己材料、事实进度、Agent 缺口、资金结果。 |
| 资金方视角 | 银行辅助审查。 | 看到相关案件、摘要、风险、授权和执行条件。 |
| 事实方视角 | 节点事实确认。 | 只看到自己任务和必要上下文。 |
| 平台方视角 | 全局运行控制塔。 | 看到资金池、规则、暂停、异常、Agent 堆积。 |
| 纯 DeFi 路径 | 无银行自动执行或阻断。 | 不出现强制银行审批，合约输出可解释。 |
| 低人工路径 | Agent + 合约自动推进，异常升级人。 | 正常节点自动走，争议和高风险不自动裁决。 |
| 权限隔离 | 角色不可越权。 | 无关案件、内部审查、事实修改入口不可见。 |
| 异常解释 | 等待、阻断、失败、争议。 | 原因、责任角色、解除条件和下一步清楚。 |

## 4. 验收脚本 A：出口商控制塔走读

### 4.1 进入条件

| 条件 | 说明 |
|---|---|
| 钱包角色 | 已永久绑定为出口商 / 资金需求方。 |
| 案件状态 | 至少存在一笔 Draft 或进行中的融资/结算案件。 |
| Agent 模式 | 半自动托管。 |

### 4.2 走读步骤

| 步骤 | 业务动作 | 控制塔应表达 |
|---|---|---|
| 1 | 出口商进入工作区。 | 看到自己案件的端到端链路，不看到其他出口商案件。 |
| 2 | 查看案件源头。 | 看到贸易主体、货物、金额、申请融资或结算意图。 |
| 3 | 查看材料节点。 | 看到合同、订单、发票、装箱单是否完整。 |
| 4 | 查看 Agent 摘要。 | 看到缺口、风险、建议和下一步。 |
| 5 | 查看事实节点。 | 看到质检是否完成，P1/P2 节点若未启用则显示 Skipped 或 Not required。 |
| 6 | 查看资金或执行节点。 | 银行辅助路径显示资金方审查；纯 DeFi 路径显示 DeFi 检查。 |
| 7 | 查看卡点。 | 看到缺材料、待事实、待审查、资金池不足或规则阻断。 |

### 4.3 通过标准

| 验收项 | 通过标准 |
|---|---|
| 全链路可见 | 出口商能看清自己案件从业务意图到资金结果的节点。 |
| 卡点可解释 | 出口商知道卡在哪里、谁负责、下一步是什么。 |
| Agent 可理解 | Agent 摘要不是技术日志，而是业务说明。 |
| 权限正确 | 出口商看不到资金方内部审查细节和平台全局数据。 |

## 5. 验收脚本 B：银行辅助路径走读

### 5.1 场景设定

| 项目 | 内容 |
|---|---|
| 路径 | 银行辅助路径 |
| 主要角色 | 出口商、资金方、质检方、平台方、Agent、DeFi 智能合约 |
| 业务目标 | 资金方授权后，由 DeFi 智能合约检查执行条件并执行或阻断。 |

### 5.2 走读步骤

| 步骤 | 角色 | 业务动作 | 应看到的结果 |
|---|---|---|---|
| 1 | 出口商 | 创建融资意图并提交材料。 | 案件进入 Draft / Material checking。 |
| 2 | Agent | 整理材料、生成缺口和摘要。 | 控制塔显示材料节点完成或 Need materials。 |
| 3 | 质检方 | 提交 pass / fail / dispute。 | 质检节点显示 Completed、Warning 或 Dispute。 |
| 4 | Agent | 生成资金方审查摘要。 | 资金方看到案件摘要、材料完整性、事实状态。 |
| 5 | 资金方 | 允许放款、拒绝、补材料或人工复核。 | 决策节点更新，出口商看到业务结果和原因。 |
| 6 | DeFi 合约 | 检查规则、资金池、授权、暂停状态。 | 执行节点显示允许、等待、阻断、失败或完成。 |
| 7 | 平台方 | 处理规则、资金池或执行异常。 | 平台控制塔显示影响范围和处理待办。 |

### 5.3 通过标准

| 验收项 | 通过标准 |
|---|---|
| 银行不是技术装饰 | 资金方在银行辅助路径中真实承担授权、拒绝、补材料或人工复核。 |
| 合约不是展示层 | 资金方授权后，DeFi 合约仍要检查执行条件。 |
| 授权边界清楚 | 合约不能绕过资金方授权直接执行。 |
| 执行结果可解释 | 允许、等待、阻断、资金池不足、平台暂停、执行失败都有业务原因。 |

## 6. 验收脚本 C：纯 DeFi 路径走读

### 6.1 场景设定

| 项目 | 内容 |
|---|---|
| 路径 | 纯 DeFi |
| 银行参与 | 不需要 |
| 主要角色 | 出口商 / 业务方、Agent、必要事实方、平台方、DeFi 智能合约 |
| 业务目标 | 在规则、资金池、事实和保证条件满足时自动执行资金动作。 |

### 6.2 走读步骤

| 步骤 | 角色 | 业务动作 | 控制塔应表达 |
|---|---|---|---|
| 1 | 出口商 / 业务方 | 创建小额融资、结算或保证金释放意图。 | 路径显示为纯 DeFi。 |
| 2 | Agent | 整理材料和事实，检查是否低风险。 | 银行审查节点显示 Skipped 或 Not required。 |
| 3 | 事实方 | 提交必要事实。 | 对应事实节点 Completed 或 Warning。 |
| 4 | DeFi 合约 | 检查规则、资金池、抵押/保证、暂停状态。 | DeFi 执行检查节点 In progress / Completed / Blocked。 |
| 5 | DeFi 合约 | 条件满足时执行资金动作。 | 资金结果节点 Completed。 |
| 6 | Agent | 若发现争议或高风险，升级人工。 | Manual review 节点出现，自动执行停止。 |

### 6.3 通过标准

| 验收项 | 通过标准 |
|---|---|
| 无银行路径清楚 | 原型不能强制等待银行或资金方审批。 |
| DeFi 是执行核心 | 规则和资金池满足时，合约负责执行或阻断。 |
| Agent 可自动推进 | 低风险、事实完整、授权清楚时 Agent 可推进。 |
| 异常不自动裁决 | 重大争议、高风险和法律责任冲突必须升级人工。 |

## 7. 验收脚本 D：Agent + 智能合约低人工路径

### 7.1 场景设定

| 项目 | 内容 |
|---|---|
| 路径 | Agent + 智能合约低人工路径 |
| 适用场景 | 保证金释放、付款责任确认、简单结算、规则清楚的运营流程。 |
| 人工参与 | 只处理授权、异常、争议、高风险和重大法律责任。 |

### 7.2 走读步骤

| 步骤 | 业务动作 | 验收重点 |
|---|---|---|
| 1 | 业务方创建低人工结算或保证金场景。 | 控制塔显示路径不是银行辅助。 |
| 2 | Agent 检查事实和规则条件。 | 输出事实完整、规则满足或缺口。 |
| 3 | Agent 自动推进低风险节点。 | 只能在授权模式允许范围内推进。 |
| 4 | DeFi 合约锁定、释放或结算资金。 | 输出执行完成或等待条件。 |
| 5 | 买方或事实方提出争议。 | Agent 升级人工或约定仲裁，不自动裁决。 |

### 7.3 通过标准

| 验收项 | 通过标准 |
|---|---|
| 低人工不是无人治理 | 正常自动推进，异常仍有人处理。 |
| Agent 不独立拥有权力 | Agent 权限来自所属角色和授权模式。 |
| 合约按规则执行 | 合约负责锁定、释放、结算或阻断。 |
| 法律责任不自动裁决 | 重大争议进入人工或约定仲裁。 |

## 8. 验收脚本 E：事实方节点视角

### 8.1 走读角色

| 事实方 | 节点 |
|---|---|
| 质检方 | 质检事实 |
| 物流方 | 运输事实 |
| 海关 / 报关方 | 通关事实 |
| 保险公司 | 承保和理赔事实 |
| 买方 / 进口商 | 订单、收货、争议和付款责任 |

### 8.2 走读步骤

| 步骤 | 业务动作 | 应看到 | 不应看到 |
|---|---|---|---|
| 1 | 事实方进入控制塔视角。 | 自己待确认任务。 | 无关案件。 |
| 2 | 查看必要上下文。 | 货物、订单、节点要求、补充说明。 | 资金方内部审批。 |
| 3 | 提交事实材料或结论。 | 对应节点 Completed / Warning / Dispute。 | 修改出口商贸易材料入口。 |
| 4 | 请求补充说明。 | 生成对应责任角色待办。 | 要求无关角色补无关材料。 |
| 5 | 查看节点影响。 | 看到事实如何影响当前案件状态。 | 决定放款或合约执行入口。 |

### 8.3 通过标准

| 验收项 | 通过标准 |
|---|---|
| 事实来源独立 | 谁提供事实，谁负责事实。 |
| 必要上下文足够 | 事实方能完成任务但不看到无关信息。 |
| 事实不能被篡改 | 其他角色只能查看、质疑或请求补充。 |
| 节点影响清楚 | 事实结论能影响风险、审查、规则或执行状态。 |

## 9. 验收脚本 F：平台全局控制塔

### 9.1 进入条件

| 条件 | 说明 |
|---|---|
| 钱包角色 | 平台方 / 合约运营。 |
| 可见范围 | 全局运行状态、规则、资金池、暂停、异常、Agent 堆积。 |
| 边界 | 可以运营规则和执行秩序，不能伪造业务事实。 |

### 9.2 走读步骤

| 步骤 | 业务动作 | 平台控制塔应表达 |
|---|---|---|
| 1 | 查看运行总览。 | 正常案件、异常案件、资金池、规则、Agent 状态。 |
| 2 | 查看规则阻断。 | 阻断原因、影响路径、影响案件、恢复条件。 |
| 3 | 查看资金池不足。 | 资金池、币种、影响范围、待处理动作。 |
| 4 | 查看执行失败。 | 失败原因、涉及案件、下一步责任角色。 |
| 5 | 查看 Agent 异常。 | 堆积、失败、越权风险、人工升级。 |
| 6 | 暂停或恢复规则/资金池/案件。 | 必须记录原因和影响范围。 |

### 9.3 通过标准

| 验收项 | 通过标准 |
|---|---|
| 平台能看全局运行 | 资金池、规则、暂停、异常、Agent 堆积清楚。 |
| 平台不逐单审批正常案件 | 正常案件由 Agent、规则、资金方或合约推进。 |
| 平台动作可审计 | 暂停、恢复、处理异常必须有原因。 |
| 平台不伪造事实 | 平台不能替出口商、质检、物流、保险、买方生成事实。 |

## 10. 验收脚本 G：权限隔离和越权阻断

| 越权尝试 | 期望结果 | 业务解释 |
|---|---|---|
| 出口商查看其他出口商案件。 | 阻断。 | 无关案件不可见。 |
| 出口商修改质检结论。 | 阻断。 | 质检事实属于质检方。 |
| 质检方查看资金方内部审批。 | 阻断。 | 质检方不是资金审批方。 |
| 资金方修改物流或保险事实。 | 阻断。 | 事实来源不可篡改。 |
| 平台方替资金方审批银行辅助案件。 | 阻断。 | 平台管规则和秩序，不替资金方决策。 |
| Agent 查看所属角色不可见案件。 | 阻断。 | Agent 权限继承所属角色。 |
| DeFi 合约绕过授权执行银行辅助案件。 | 阻断。 | 银行辅助路径必须先有资金方授权。 |
| 纯 DeFi 场景强制等待银行审批。 | 阻断或标记设计错误。 | 纯 DeFi 路径不需要银行。 |

## 11. 验收脚本 H：异常和失败解释

| 异常 | 控制塔应表达 | 下一步 |
|---|---|---|
| 材料不足 | 缺什么、谁补、完成标准。 | 责任角色补材料。 |
| 事实不足 | 缺哪个事实节点。 | 生成事实方任务。 |
| 事实冲突 | 哪些事实冲突、影响什么。 | 人工复核或补说明。 |
| 质检 dispute | 争议原因、影响审查或执行。 | 人工复核或补材料。 |
| 买方 dispute | 买方争议和付款责任不确定。 | 暂停自动执行，进入人工。 |
| 银行授权不足 | 授权金额、期限或条件不足。 | 资金方处理。 |
| 规则阻断 | 规则类型、阻断原因、解除条件。 | 平台或责任角色处理。 |
| 资金池不足 | 资金池、币种、影响范围。 | 平台或资金池运营处理。 |
| 平台暂停 | 暂停对象、原因、影响范围。 | 等待平台恢复。 |
| Agent 越权风险 | Agent 为什么不能自动推进。 | 降级人工确认。 |
| 执行失败 | 资金动作为什么没有完成。 | 平台处理并解释结果。 |

## 12. 验收输出记录

原型验收时，每个脚本应记录以下业务结论。

| 记录项 | 说明 |
|---|---|
| 场景名称 | 银行辅助、纯 DeFi、低人工、事实方、平台或异常场景。 |
| 角色视角 | 当前以哪个角色走读。 |
| 可见范围 | 该角色看到了哪些案件、节点和事实。 |
| 不可见范围 | 该角色不能看到哪些内容。 |
| 业务动作 | 用户或 Agent 触发了什么动作。 |
| 控制塔状态 | 节点状态、卡点、责任角色和下一步。 |
| Agent 输出 | 摘要、缺口、待办、建议或升级。 |
| DeFi 输出 | 允许、等待、阻断、失败或完成。 |
| 验收结论 | 通过、需改业务口径、需补需求、暂不纳入 P0。 |

## 13. MVP 范围

### 13.1 P0 必须验收

| 范围 | 说明 |
|---|---|
| 控制塔全链路表达 | 能看清供应链源头到终端的节点流转。 |
| 角色视角差异 | 不同角色看到不同范围。 |
| 银行辅助路径 | 资金方参与场景可讲通。 |
| 纯 DeFi 路径 | 无银行场景可讲通。 |
| 低人工路径 | Agent + 合约自动推进和异常升级可讲通。 |
| Agent 待办和摘要 | 责任角色、缺口、建议、升级清楚。 |
| DeFi 执行解释 | 允许、等待、阻断、失败、完成可解释。 |
| 异常回流 | 补材料、争议、资金池、规则、平台暂停可回流。 |
| 权限隔离 | 越权行为被阻断并能解释。 |

### 13.2 P0 暂不验收

| 范围 | 原因 |
|---|---|
| 视觉稿还原度 | 当前不是 UI 设计验收。 |
| 组件交互细节 | 当前不做组件验收。 |
| 自动化测试代码 | 当前不写测试实现。 |
| 接口路径和请求响应 | 当前不验收技术接口。 |
| 数据库数据结构 | 当前不做数据库验收。 |
| 智能合约代码 | 当前不验收 Solidity。 |
| 钱包签名流程 | 当前不进入签名实现。 |
| hash / proof 细节 | 当前不进入底层证明产品。 |

## 14. 本步结论

P0 原型验收脚本的本质不是“检查页面做得像不像”。

它的本质是：

> 用业务走读确认 ChainTrace 的控制塔、角色权限、Agent 协作、银行辅助路径、纯 DeFi 路径、低人工路径、异常解释和 DeFi 执行结果是否能形成一个可理解、可推进、可阻断、可回流的供应链金融系统。

下一步应进入：

> P0 产品原型冻结清单，用来确认进入视觉原型或技术设计前，哪些业务口径已经冻结，哪些仍然后置。
