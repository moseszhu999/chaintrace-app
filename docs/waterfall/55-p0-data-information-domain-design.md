# 55 P0 数据信息域设计

## 1. 定位

本文件定义 ChainTrace P0 技术设计阶段的数据信息域、事实来源、记录责任和可见范围。

它不是数据库设计，不是表结构，不是字段清单，不是索引设计，不是链上/链下存储方案，不是 ORM 模型，也不是数据迁移计划。

它的作用是：

> 用业务语言说明 P0 需要记录哪些信息域、哪些事实由谁产生、哪些记录由谁确认、哪些角色可以看到哪些信息，以及哪些记录必须可追踪，为后续数据模型设计提供业务边界。

本文件只定义信息域和记录责任，不进入数据库实现。

## 2. 设计原则

| 原则 | 说明 |
|---|---|
| 先定义信息域 | 当前只说业务上需要哪些信息，不说表和字段。 |
| 事实来源必须清楚 | 每个事实都要知道由哪个角色产生或确认。 |
| 记录责任必须清楚 | 谁提交、谁确认、谁可见、谁不能改必须清楚。 |
| Agent 输出不是事实 | Agent 摘要、建议、缺口不等于原始事实或审批结论。 |
| 合约输出不是贸易真实性 | 合约只输出执行条件和执行结果，不判断贸易真假。 |
| 权限可见必须继承 | 数据可见范围继承 53 号权限边界。 |
| 历史不可被覆盖 | 已提交、已决策、已暂停、已执行的关键记录不能被无痕覆盖。 |
| 异常要可追踪 | 缺材料、dispute、阻断、暂停、失败必须有原因和责任方。 |
| 实现细节后置 | 表结构、字段、索引、存储、链上/链下分层后置。 |

## 3. P0 信息域总览

| 信息域 | 业务含义 | 主要责任方 |
|---|---|---|
| 身份与角色信息域 | 钱包、角色、账户状态、Agent 模式。 | 用户、平台规则。 |
| 融资案件信息域 | 融资意图、融资类型、金额、状态、结果。 | 出口商、Agent、资金方。 |
| 贸易材料信息域 | 合同、订单、发票、装箱单、补充说明等。 | 出口商。 |
| 事实节点信息域 | 质检、物流、保险、买方、通关等事实状态。 | 对应事实方。 |
| 资金方决策信息域 | 允许、拒绝、补材料、人工复核、降额、贷后状态。 | 资金方。 |
| 合约业务输出信息域 | 允许、等待、阻断、资金池不足、平台暂停、失败、完成。 | 合约/规则执行层。 |
| Agent 输出信息域 | 摘要、缺口、待办、提醒、建议、升级。 | 所属角色 Agent。 |
| 异常信息域 | 缺材料、dispute、规则阻断、资金池不足、执行失败等。 | 责任角色、Agent、平台。 |
| 平台运营信息域 | 资金池、规则、合约、暂停、恢复、Agent 异常。 | 平台方。 |
| 审计记录信息域 | 授权、提交、决策、暂停、恢复、执行结果等关键记录。 | 系统、平台、各责任角色。 |

## 4. 身份与角色信息域

### 4.1 业务内容

| 信息类别 | 业务含义 |
|---|---|
| 钱包身份 | 用户进入 ChainTrace 的身份入口。 |
| 注册状态 | 未注册、已注册、已激活、受限、暂停。 |
| 永久角色 | 钱包绑定的唯一业务角色。 |
| 账户资料 | 国家、语言、账户类型、显示名称、联系方式等。 |
| Agent 模式 | 全人工、半自动、全自动。 |
| 账户限制原因 | 资料不足、风险、平台暂停、疑似异常等。 |

### 4.2 责任与可见范围

| 信息 | 产生/确认方 | 可见方 | 不可做事项 |
|---|---|---|---|
| 钱包身份 | 钱包用户。 | 钱包用户、必要的平台运营。 | 不得伪造。 |
| 永久角色 | 用户注册时选择，系统锁定。 | 用户、平台。 | P0 不允许修改。 |
| Agent 模式 | 用户设置。 | 用户、所属 Agent、必要的平台运营。 | 不得改变角色权限。 |
| 账户状态 | 系统/平台规则。 | 用户、平台。 | 不能绕过受限或暂停。 |
| 账户资料 | 用户提交。 | 用户、必要的相关角色。 | 不得无关扩散。 |

## 5. 融资案件信息域

### 5.1 业务内容

| 信息类别 | 业务含义 |
|---|---|
| 融资意图 | 出口商声明某笔贸易需要融资。 |
| 融资类型 | 订单融资、应收账款融资、发票融资、出货后融资等。 |
| 申请金额 | 出口商希望融资的金额和币种。 |
| 贸易主体 | 出口商、买方、资金方、相关事实方。 |
| 案件状态 | 草稿、待提交、已提交、补材料、审查中、批准、拒绝、放款中、到账等。 |
| 案件卡点 | 当前阻断原因、责任方和下一步。 |
| 资金结果 | 可融资、不可融资、等待、放款中、已到账、失败等。 |

### 5.2 责任与可见范围

| 信息 | 产生/确认方 | 可见方 | 不可做事项 |
|---|---|---|---|
| 融资意图 | 出口商。 | 出口商、匹配/分配资金方、必要的平台异常视角。 | 其他出口商不可见。 |
| 融资金额 | 出口商提交，资金方可调整建议。 | 出口商、资金方、必要的平台异常视角。 | 资金方不能无痕修改出口商原始申请。 |
| 案件状态 | 系统根据业务动作产生。 | 案件相关角色。 | 不得由无关角色手动篡改。 |
| 资金结果 | 资金方决策和合约/规则执行共同产生。 | 出口商、资金方、必要的平台运营。 | 出口商不能自改结果。 |
| 卡点信息 | Agent、规则、平台或责任角色产生。 | 案件相关角色。 | 不能只显示技术失败。 |

## 6. 贸易材料信息域

### 6.1 业务内容

| 信息类别 | 业务含义 |
|---|---|
| 合同 | 交易双方、条款、金额、付款条件。 |
| 订单 | 买方采购意图、商品、数量、金额。 |
| 发票 | 金额、币种、付款义务。 |
| 装箱单 | 数量、包装、货物明细。 |
| 补充说明 | 对异常、差异、特殊条款的解释。 |
| 材料版本 | 提交前、提交后、补充记录的历史。 |
| 材料缺口 | 还缺什么、谁负责补、影响什么。 |

### 6.2 责任与可见范围

| 信息 | 产生/确认方 | 可见方 | 不可做事项 |
|---|---|---|---|
| 原始贸易材料 | 出口商提交。 | 出口商、相关资金方、必要事实方和平台异常视角。 | 资金方、平台、Agent 不得修改原始材料。 |
| 补充说明 | 出口商或相关责任方。 | 案件相关角色。 | 不得覆盖原始提交历史。 |
| 材料摘要 | Agent 生成。 | 相关角色。 | 摘要不等于原始材料。 |
| 材料缺口 | Agent 或资金方提出。 | 出口商和相关责任方。 | 缺口不能被当作拒绝结论。 |
| 提交版本 | 系统锁定。 | 案件相关角色。 | 不得无痕覆盖。 |

## 7. 事实节点信息域

事实节点是 ChainTrace 供应链金融前站的核心。事实节点必须由对应事实方产生或确认，不能由 Agent 或平台伪造。

### 7.1 事实节点总表

| 事实节点 | 事实内容 | 责任方 | P0/P1/P2 |
|---|---|---|---|
| 质检事实 | 货物质量、数量、包装、现场状态、pass/fail/dispute。 | 质检方。 | P0 |
| 物流事实 | 揽收、装柜、出港、到港、签收、延误、损坏、丢失等。 | 物流方/出口商补充。 | P1，占位进入 P0。 |
| 保险事实 | 承保、拒保、加费、补材料、理赔状态。 | 保险方。 | P1，占位进入 P0。 |
| 买方事实 | 订单相关性、收货状态、争议、付款责任。 | 买方。 | P1，占位进入 P0。 |
| 通关事实 | 报关、清关、出口/进口记录、监管异常。 | 海关/报关方。 | P2，占位后置。 |

### 7.2 质检事实

| 信息 | 产生/确认方 | 可见方 | 不可做事项 |
|---|---|---|---|
| 质检任务 | 系统/Agent 分配，质检方接收。 | 质检方、出口商、资金方、平台异常视角。 | 无关质检方不可见。 |
| 质检报告和照片 | 质检方提交。 | 案件相关角色。 | 出口商、资金方、平台不能伪造。 |
| pass/fail/dispute | 质检方确认。 | 出口商、资金方、平台异常视角。 | Agent 不能最终伪造结论。 |
| 质检补充记录 | 质检方补充。 | 案件相关角色。 | 不得覆盖原质检历史。 |

### 7.3 物流事实

| 信息 | 产生/确认方 | 可见方 | 不可做事项 |
|---|---|---|---|
| 运输节点 | 物流方确认，出口商可提交辅助材料。 | 出口商、资金方、平台异常视角。 | 资金方不能伪造运输状态。 |
| 物流文件 | 物流方或出口商提交。 | 案件相关角色。 | Agent 只能整理。 |
| 物流异常 | 物流方说明。 | 案件相关角色。 | 不得无说明解除异常。 |

### 7.4 保险事实

| 信息 | 产生/确认方 | 可见方 | 不可做事项 |
|---|---|---|---|
| 核保结论 | 保险方。 | 出口商、资金方、平台异常视角。 | 资金方不能冒充保险方。 |
| 承保状态 | 保险方。 | 案件相关角色。 | Agent 不能自动承保。 |
| 理赔状态 | 保险方。 | 出口商、资金方、平台异常视角。 | 不能用平台规则替代理赔结论。 |

### 7.5 买方事实

| 信息 | 产生/确认方 | 可见方 | 不可做事项 |
|---|---|---|---|
| 订单相关性 | 买方确认。 | 出口商、资金方、平台异常视角。 | 出口商不能替买方确认。 |
| 收货状态 | 买方确认，物流可辅助。 | 案件相关角色。 | 资金方不能伪造收货。 |
| 付款责任 | 买方确认或贸易文件支持。 | 出口商、资金方、平台异常视角。 | Agent 不能自动制造付款责任。 |
| dispute | 买方提出。 | 案件相关角色。 | 不得由系统自动裁决重大争议。 |

## 8. 资金方决策信息域

### 8.1 业务内容

| 信息类别 | 业务含义 |
|---|---|
| 待审案件 | 资金方有权处理的融资申请。 |
| 审查摘要 | Agent 帮助资金方理解案件，但不替代决策。 |
| 证据完整性 | 合同、订单、发票、质检、物流等是否满足要求。 |
| 风险建议 | 可放款、补材料、人工复核、拒绝等建议。 |
| 资金方决策 | 允许、拒绝、补材料、降额、人工复核。 |
| 决策原因 | 为什么允许、拒绝或补材料。 |
| 贷后状态 | 回款、逾期、争议、保险或理赔状态。 |

### 8.2 责任与可见范围

| 信息 | 产生/确认方 | 可见方 | 不可做事项 |
|---|---|---|---|
| 资金方决策 | 资金方。 | 出口商、资金方、必要的平台异常视角。 | 平台和 Agent 不能伪造。 |
| 决策原因 | 资金方。 | 出口商可见对外原因，资金方可见内部判断。 | 不得用技术错误代替业务原因。 |
| 风险建议 | Agent 辅助生成。 | 资金方。 | 建议不等于最终决策。 |
| 贷后状态 | 资金方、买方、保险方、系统状态共同产生。 | 案件相关资金方、出口商、平台异常视角。 | 无关资金方不可见。 |

## 9. 合约业务输出信息域

### 9.1 业务内容

| 信息类别 | 业务含义 |
|---|---|
| 执行条件 | 授权、事实、规则、资金池、暂停状态等。 |
| 资金池状态 | 足够、不足、暂停、异常。 |
| 规则状态 | 有效、暂停、过期、不适用。 |
| 平台暂停状态 | 平台或规则是否暂停执行。 |
| 执行输出 | 允许、等待、阻断、失败、完成。 |
| 失败原因 | 为什么执行失败，谁处理，下一步是什么。 |

### 9.2 责任与可见范围

| 信息 | 产生/确认方 | 可见方 | 不可做事项 |
|---|---|---|---|
| 合约执行结果 | 合约/规则执行层。 | 案件相关角色、平台。 | 不等于贸易真实性。 |
| 资金池状态 | 合约/平台运营。 | 相关资金方、平台，出口商可见业务影响。 | 出口商不能修改。 |
| 规则阻断 | 规则执行层/平台规则。 | 相关角色。 | 不得无原因阻断。 |
| 执行失败 | 执行层。 | 相关角色和平台。 | 不能只显示技术失败。 |

## 10. Agent 输出信息域

### 10.1 业务内容

| 信息类别 | 业务含义 |
|---|---|
| 案件摘要 | 帮助角色快速理解案件。 |
| 材料缺口 | 告诉用户缺什么。 |
| 风险提示 | 提醒风险点和原因。 |
| 待办任务 | 下一步由谁做什么。 |
| 补材料请求 | 向相关角色请求补充事实或材料。 |
| 自动推进记录 | Agent 在授权范围内做了什么。 |
| 人工升级记录 | 为什么不能自动继续，升级给谁。 |

### 10.2 责任与可见范围

| 信息 | 产生/确认方 | 可见方 | 不可做事项 |
|---|---|---|---|
| Agent 摘要 | Agent 生成。 | 所属角色和案件相关角色。 | 摘要不等于事实。 |
| Agent 建议 | Agent 生成。 | 所属角色。 | 建议不等于审批。 |
| Agent 待办 | Agent 生成。 | 责任角色。 | 不能分派给无关角色。 |
| Agent 自动推进记录 | Agent 执行。 | 所属角色、平台异常视角。 | 不能越权推进。 |
| Agent 升级记录 | Agent 生成。 | 责任角色、平台必要视角。 | 不能自动裁决重大争议。 |

## 11. 异常信息域

### 11.1 异常类型

| 异常 | 业务含义 | 责任方 |
|---|---|---|
| 缺材料 | 关键材料或事实不足。 | 出口商或对应事实方。 |
| 金额不一致 | 合同、订单、发票金额冲突。 | 出口商、资金方。 |
| 质检 fail | 货物不符合要求。 | 质检方、资金方。 |
| 质检 dispute | 事实存在争议或不足。 | 质检方、出口商、资金方。 |
| 物流异常 | 延误、损坏、丢失、节点不一致。 | 物流方。 |
| 买方争议 | 收货、付款或质量争议。 | 买方、出口商、资金方。 |
| 保险不足 | 风险缓释不足。 | 保险方、资金方。 |
| 资金池不足 | 执行资金不足。 | 平台、资金方。 |
| 规则阻断 | 当前规则不允许继续。 | 平台、规则执行层。 |
| 平台暂停 | 平台暂停资金池、规则、案件或账户。 | 平台方。 |
| 执行失败 | 执行层失败。 | 平台、执行层。 |
| Agent 失败 | 自动化不能继续。 | 所属角色、平台必要视角。 |

### 11.2 异常记录必须包含的业务信息

| 记录项 | 说明 |
|---|---|
| 异常原因 | 为什么发生。 |
| 责任方 | 谁需要处理。 |
| 影响范围 | 影响案件、资金池、规则、角色或执行。 |
| 当前状态 | 等待、处理中、已解除、继续阻断。 |
| 解除条件 | 需要满足什么才能继续。 |
| 下一步 | 谁做什么。 |
| 历史记录 | 异常发生、补充、解除或关闭的过程。 |

## 12. 平台运营信息域

| 信息类别 | 业务含义 | 责任方 | 可见方 |
|---|---|---|---|
| 资金池状态 | 正常、不足、暂停、异常、执行失败。 | 平台/执行层。 | 平台、相关资金方、受影响出口商。 |
| 规则版本 | 当前版本、暂停、恢复、升级、影响范围。 | 平台。 | 平台、受影响角色。 |
| 合约状态 | 正常、暂停、升级中、异常。 | 平台/执行层。 | 平台、受影响角色。 |
| Agent 自动化状态 | 正常、失败、堆积、需人工。 | 平台/Agent。 | 平台、所属角色。 |
| 平台操作记录 | 暂停、恢复、升级、维护、分派。 | 平台。 | 平台审计范围。 |

平台运营信息域边界：

- 平台记录系统级状态，不等于平台可以修改所有业务事实。
- 平台操作必须有业务原因、影响范围和恢复条件。
- 平台操作记录不能被无痕删除。

## 13. 审计记录信息域

### 13.1 必须可追踪的关键动作

| 动作 | 记录目的 |
|---|---|
| 钱包注册和角色绑定 | 证明单钱包单角色。 |
| Agent 模式设置和调整 | 证明自动化授权范围。 |
| 融资案件创建和提交 | 证明融资意图和提交时间。 |
| 贸易材料提交和补充 | 证明事实来源和版本历史。 |
| 质检结论提交 | 证明货物事实来源。 |
| 资金方决策 | 证明允许、拒绝、补材料、人工复核的责任。 |
| 平台暂停和恢复 | 证明系统级操作原因。 |
| 合约执行输出 | 证明执行结果和阻断原因。 |
| Agent 自动推进和人工升级 | 证明自动化行为没有越权。 |
| 异常发生和解除 | 证明异常处理过程。 |

### 13.2 审计边界

| 边界 | 说明 |
|---|---|
| 审计记录不等于公开信息 | 不同角色只能看自己有权看的部分。 |
| 审计记录不可无痕覆盖 | 关键动作必须保留历史。 |
| Agent 行为要可追踪 | 自动推进、请求补充、升级必须留痕。 |
| 平台高影响操作要可追踪 | 暂停、恢复、升级、标记异常必须留痕。 |

## 19. 一票否决项

| 一票否决项 | 原因 |
|---|---|
| Agent 摘要被当成原始事实 | 破坏事实来源边界。 |
| 合约输出被当成贸易真实性判断 | 破坏合约业务角色。 |
| 资金方可以无痕修改出口商原始材料 | 破坏证据可信性。 |
| 平台可以删除关键操作记录 | 破坏审计能力。 |
| 角色可见范围突破权限边界 | 破坏数据隔离。 |
| 事实来源角色不清 | 后续无法验收和审计。 |
| 直接写表结构、字段、索引或存储实现 | 阶段越界。 |
| P1/P2 信息域被塞入 P0 必做 | 破坏 MVP 范围。 |

## 20. 本步结论

P0 数据信息域设计的本质不是“设计数据库”。

它的本质是：

> 在业务层定义 ChainTrace P0 必须记录的身份、案件、贸易材料、事实节点、资金方决策、合约输出、Agent 输出、异常、平台运营和审计信息，并明确每类信息的事实来源、记录责任、可见范围和不可越界事项，为后续数据模型设计提供边界。
