# 71 P0 实现阶段 1：Identity and Role Foundation 计划

## 1. 定位

本文件定义 ChainTrace P0 受控实现阶段的第一个阶段：Identity and Role Foundation。

它不是代码实现方案，不是钱包 SDK 选型，不是认证协议设计，不是数据库表结构，不是 API endpoint 文档，不是 UI 页面实现，也不是测试脚本。

它的作用是：

> 在进入具体实现前，细化 Phase 1 的业务范围、能力边界、角色状态、工作区门禁、Agent 模式、越权阻断和验收口径，确保后续实现先把 ChainTrace 的身份和权限基础打稳。

本阶段的核心目标不是“登录成功”，而是：

```text
钱包入口
→ 注册状态
→ 单钱包单角色
→ 账户状态
→ 角色工作区
→ Agent 模式
→ 越权阻断
```

## 3. Phase 1 阶段目标

Phase 1 必须让系统具备以下基础能力：

| 目标 | 说明 |
|---|---|
| 钱包入口状态可表达 | 用户是否连接钱包、是否注册、是否激活必须清楚。 |
| 业务账户可识别 | 钱包不是业务账户，注册后才形成业务账户。 |
| 单钱包单角色成立 | 一个钱包只能绑定一个 P0 业务角色。 |
| 角色工作区可分离 | 出口商、资金方、质检方、平台方进入不同工作区。 |
| 账户状态可控 | 未注册、已激活、受限、暂停状态可表达。 |
| Agent 模式可设置 | 全人工、半自动、全自动模式可表达，但不改变角色。 |
| 越权阻断可解释 | 角色不匹配、账户暂停、未注册等都要给业务解释。 |

## 5. 角色范围

| 角色 | P0 状态 | Phase 1 处理 |
|---|---|---|
| Exporter / 出口商 | P0 必须 | 可注册、可进入出口商工作区入口。 |
| Funding Provider / 资金方 | P0 必须 | 可注册、可进入资金方工作区入口。 |
| Inspection Provider / 质检方 | P0 必须 | 可注册、可进入质检方工作区入口。 |
| Platform Operator / 平台方 | P0 必须 | 可识别、可进入平台工作区入口。 |
| Logistics Provider / 物流方 | P1 | Phase 1 可显示为后置角色，不进入完整注册流程。 |
| Insurance Provider / 保险方 | P1 | Phase 1 可显示为后置角色，不进入完整注册流程。 |
| Buyer / 买方 | P1 | Phase 1 可显示为后置角色，不进入完整注册流程。 |
| Customs / 海关报关 | P2 | 不进入 Phase 1。 |

## 6. 身份状态模型

Phase 1 必须表达以下身份状态。

| 状态 | 业务含义 | 用户可做 |
|---|---|---|
| Wallet not connected | 尚未连接钱包。 | 连接钱包。 |
| Wallet connected, unregistered | 钱包已连接，但没有业务账户。 | 选择唯一角色并注册。 |
| Registration incomplete | 注册信息不完整。 | 补充注册信息。 |
| Registered, pending activation | 已注册但未激活。 | 等待激活或平台处理。 |
| Active | 已激活。 | 进入对应角色工作区。 |
| Restricted | 账户受限。 | 查看受限原因和下一步。 |
| Suspended | 账户暂停。 | 查看暂停原因和平台处理路径。 |

状态表达必须用业务语言，不应只显示技术错误。

## 7. 单钱包单角色规则

### 7.1 核心规则

```text
一个钱包地址在 P0 中只能绑定一个永久业务角色。
```

### 7.2 允许行为

| 行为 | 是否允许 | 说明 |
|---|---|---|
| 未注册钱包选择角色注册 | 允许 | 只能选择一个 P0 角色。 |
| 已注册钱包进入自己角色工作区 | 允许 | 受账户状态限制。 |
| 已注册钱包查看自己的角色状态 | 允许 | 用于工作区门禁。 |
| 已注册钱包修改 Agent 模式 | 允许 | 不改变业务角色。 |

### 7.3 禁止行为

| 行为 | 是否禁止 | 原因 |
|---|---|---|
| 已注册钱包重新选择另一角色 | 禁止 | 破坏单钱包单角色。 |
| 一个钱包同时成为出口商和资金方 | 禁止 | 角色冲突。 |
| Agent 模式切换后获得新角色权限 | 禁止 | Agent 模式不是业务角色。 |
| 未注册钱包直接进入业务工作区 | 禁止 | 缺少业务账户。 |

## 8. Agent 模式范围

Phase 1 只实现 Agent 模式状态表达，不实现复杂 Agent 能力。

| Agent 模式 | Phase 1 表达 | 不实现内容 |
|---|---|---|
| Manual / 全人工确认 | Agent 只提示和整理，关键动作等用户确认。 | 不实现复杂推理和自动流程。 |
| Semi-auto / 半自动托管 | Agent 可处理常规摘要、缺口、提醒等。 | 不实现高风险自动推进。 |
| Full-auto / 全自动 Agent | 表达用户已授权更高自动化。 | 不实现无限制自动化。 |

边界：

- Agent 模式是账户配置，不是业务角色。
- Agent 模式不扩大数据可见范围。
- Agent 模式不允许绕过合约、规则、资金池或平台暂停。
- Phase 1 只表达模式，不实现 Phase 7 的 Agent Assist Layer。

## 9. 角色工作区门禁

Phase 1 必须建立角色工作区门禁，而不是完整工作区功能。

| 工作区 | 允许进入条件 | Phase 1 表达 |
|---|---|---|
| Exporter Workspace | 角色为出口商，账户 Active。 | 进入出口商入口页或占位工作区。 |
| Funding Workspace | 角色为资金方，账户 Active。 | 进入资金方入口页或占位工作区。 |
| Inspection Workspace | 角色为质检方，账户 Active。 | 进入质检方入口页或占位工作区。 |
| Platform Workspace | 角色为平台方，账户 Active。 | 进入平台入口页或占位工作区。 |

门禁必须阻断：

- 未连接钱包。
- 未注册钱包。
- 注册未完成。
- 账户受限。
- 账户暂停。
- 角色不匹配。

## 10. 顶部业务状态条范围

| 状态项 | 业务含义 |
|---|---|
| Wallet state | 未连接、已连接。 |
| Registration state | 未注册、待激活、已激活、受限、暂停。 |
| Business role | 出口商、资金方、质检方、平台方。 |
| Agent mode | 全人工、半自动、全自动。 |
| Workspace access | 当前是否可进入角色工作区。 |

注意：当前不设计 UI 样式，不写组件。

## 11. 业务动作范围

Phase 1 包含以下业务动作。

| 业务动作 | 触发方 | 业务结果 |
|---|---|---|
| Connect wallet | 用户 | 钱包连接成功 / 连接失败。 |
| Check registration status | 系统 / 用户 | 未注册 / 已注册 / 受限 / 暂停。 |
| Select role for registration | 未注册钱包用户 | 已选择唯一角色 / 角色不可用。 |
| Submit registration profile | 未注册钱包用户 | 注册提交 / 需补信息。 |
| Activate account | 系统 / 平台规则 | 已激活 / 待处理 / 受限。 |
| View account status | 已连接钱包用户 | 显示角色、状态、Agent 模式和门禁。 |
| Set Agent mode | 已激活账户 | 模式更新 / 权限不足 / 账户受限。 |
| Enter role workspace | 已激活账户 | 进入对应工作区 / 阻断并解释。 |
| Block unauthorized workspace access | 系统 | 阻断并返回业务解释。 |

## 12. 业务解释口径

越权或阻断不能只显示技术错误。

| 场景 | 业务解释 |
|---|---|
| 未连接钱包 | 请先连接钱包，系统需要确认你的链上身份入口。 |
| 已连接未注册 | 当前钱包尚未注册业务角色，请先选择唯一角色并完成注册。 |
| 角色不匹配 | 当前钱包已注册为出口商，不能进入资金方工作区。 |
| 账户待激活 | 注册已提交，但账户尚未激活。 |
| 账户受限 | 当前账户存在限制，请查看限制原因或联系平台处理。 |
| 账户暂停 | 当前账户已暂停，暂停期间不能进入业务工作区。 |
| Agent 模式不可设置 | 当前账户状态不允许修改 Agent 模式。 |

| 场景 | 前提 | 期望结果 |

## 19. 一票否决项

出现以下任一项，Phase 1 不得通过：

| 一票否决项 | 原因 |
|---|---|
| 一个钱包可以注册多个 P0 角色 | 破坏权限基础。 |
| 已注册钱包可以随意切换业务角色 | 破坏单钱包单角色。 |
| 未注册钱包可以进入业务工作区 | 绕过注册。 |
| 出口商可以进入资金方工作区 | 角色隔离失败。 |
| Agent 模式改变业务角色 | Agent 边界失败。 |
| 账户暂停仍能进入工作区 | 平台控制失败。 |
| 越权阻断只显示技术错误且无业务解释 | 业务不可理解。 |
| Phase 1 混入完整融资流程或 proof/hash 主线 | 阶段越界。 |
| 直接写 endpoint、表结构、合约或 UI 组件 | 当前文档阶段越界。 |

## 21. Phase 1 出口条件

Phase 1 完成后，系统应具备以下能力：

```text
1. 用户可以连接钱包。
2. 系统可以识别钱包是否注册。
3. 未注册钱包必须选择唯一 P0 角色完成注册。
4. 已注册钱包只能进入对应角色工作区。
5. 账户受限或暂停时必须阻断。
6. Agent 模式可以表达和设置，但不改变角色权限。
7. 越权阻断有业务解释。
8. 注册、角色绑定、Agent 模式和越权阻断具备留痕语义。
```

满足以上条件后，才允许进入：

> `72-p0-implementation-phase-2-financing-case-core-plan.md`

即细化 Phase 2：Financing Case Core，开始定义出口商融资案件主线的受控实现范围。

## 22. 本步结论

Phase 1 的本质不是“做一个登录页”。

它的本质是：

> 建立 ChainTrace P0 的身份和权限根基：钱包只是入口，业务账户必须注册，角色必须唯一，工作区必须隔离，Agent 模式不能扩大权限，越权必须被阻断并用业务语言解释。

只有这个基础成立，后续融资案件、材料、事实、资金方、合约、Agent 和控制塔实现才不会失控。
