# 67 P0 原型实现范围

## 1. 定位

本文件定义 ChainTrace P0 规格准备阶段中的原型实现范围。

它不是 UI 设计稿，不是页面路由表，不是组件拆分，不是样式规范，不是前端状态管理方案，不是 API 调用说明，也不是前端实现代码。

它的作用是：

> 从 `58-p0-control-tower-expression-design.md` 已定义的控制塔业务表达出发，准备后续原型实现应该覆盖的角色工作区、主流程、状态表达、卡点表达和资金结果表达，同时明确哪些页面、组件、样式、路由和代码实现仍然后置。

本文件只定义原型实现范围，不写 UI 实现。

## 3. 原型实现范围原则

| 原则 | 说明 |
|---|---|
| 先表达业务闭环 | 原型首先表达融资从意图到资金结果的闭环。 |
| 角色工作区分离 | 不同角色不是同一个后台换菜单。 |
| 控制塔优先 | 原型核心不是列表，而是状态、卡点、责任方和下一步。 |
| Agent 可见 | Agent 摘要、缺口、待办、提醒、升级必须在原型中有表达。 |
| 合约可解释 | 合约/规则输出必须用业务语言表达。 |
| 异常可处理 | 卡点必须有原因、责任方、解除条件和下一步。 |
| 路径差异可见 | 银行辅助、纯 DeFi、低人工、人工升级必须可区分。 |
| 不做 UI 实现 | 当前不写路由、组件、样式、前端状态、代码。 |
| 旧 demo 不继承 | 不继承旧 proof/hash/dashboard 展示页作为主线。 |
| 范围不膨胀 | P1/P2 完整工作区和高级 BI 后置。 |

## 4. P0 原型主线范围

P0 原型必须覆盖以下主线：

```text
钱包连接和角色状态
→ 角色工作区
→ 出口商创建融资意图
→ 贸易材料和事实准备
→ Agent 摘要、缺口和待办
→ 质检事实确认
→ 资金方审查或 DeFi 规则路径
→ 合约/规则执行检查
→ 资金结果和异常卡点
→ 平台运营异常处理
```

原型不以“好看的首页”为目标，而以“真实业务状态可理解”为目标。

|---|---|---|

| 资金方工作区 | 看待审案件、证据、风险、决策和执行条件。 | 必须 |
| 质检方工作区 | 看质检任务、提交结论、补充说明。 | 必须 |
| 平台工作区 | 看资金池、规则、合约、异常、Agent 状态。 | 必须 |
| Agent 辅助表达 | 摘要、缺口、待办、提醒、升级。 | 必须 |
| 控制塔主链路 | 表达源头到终端的业务状态。 | 必须 |

| 异常卡点表达 | 原因、责任方、影响、解除条件、下一步。 | 必须 |

## 6. 钱包与角色入口原型范围

### 6.1 必须表达

| 状态 | 原型应表达 |
|---|---|
| 未连接钱包 | 需要连接钱包才能继续。 |
| 已连接但未注册 | 需要选择唯一业务角色并注册。 |
| 已注册已激活 | 进入对应角色工作区。 |
| 已注册但受限 | 显示受限原因和下一步。 |
| 已暂停 | 显示暂停原因、影响和平台处理路径。 |
| Agent 模式 | 全人工、半自动、全自动的当前授权状态。 |

### 6.2 不得表达成

| 不得表达 | 原因 |
|---|---|
| 钱包连接后直接拥有全部权限 | 破坏注册和角色边界。 |
| 一个钱包可随意切换角色 | 破坏单钱包单角色。 |
| Agent 模式改变角色 | Agent 模式不是业务角色。 |

## 7. 出口商工作区原型范围

### 7.1 出口商必须看到

| 区域 | 业务表达 |
|---|---|
| 融资案件入口 | 创建融资意图、查看草稿、提交状态。 |
| 贸易材料状态 | 合同、订单、发票、装箱单、补充说明的完整性。 |
| Agent 缺口 | 缺什么、为什么缺、谁来补、补完后下一步。 |
| 事实节点 | 质检、物流、买方、保险、通关等状态，其中 P1/P2 可占位。 |
| 资金方 / DeFi 路径 | 当前走银行辅助、纯 DeFi、低人工还是人工升级。 |
| 合约/规则状态 | Allowed、Waiting、Rule blocked、Pool insufficient、Platform paused。 |
| 资金结果 | 能不能借、能借多少、是否放款中、是否到账、失败原因。 |
| 异常卡点 | 卡在哪里、谁处理、怎么解除。 |

### 7.2 出口商可以触发的原型动作

| 动作 | 业务边界 |
|---|---|
| 创建融资意图 | 只创建自己的案件。 |
| 上传或补充材料 | 不覆盖历史版本。 |
| 查看 Agent 摘要和缺口 | 摘要不等于事实。 |
| 提交融资申请 | 必须满足最低材料和权限条件。 |
| 查看状态和资金结果 | 只看自己的案件。 |
| 响应补材料请求 | 补充后仍需资金方/规则/合约判断。 |

### 7.3 出口商不得看到或做

| 禁止 | 原因 |
|---|---|
| 查看其他出口商案件 | 数据隔离。 |
| 查看资金方内部风险偏好 | 商业边界。 |
| 修改质检结论 | 事实来源边界。 |
| 直接触发高风险放款 | 资金方/合约边界。 |

## 8. 资金方工作区原型范围

### 8.1 资金方必须看到

| 区域 | 业务表达 |
|---|---|
| 待审案件 | 分配/匹配给自己的案件。 |
| 案件摘要 | 出口商、买方、商品、金额、融资类型。 |
| 证据完整性 | 合同、订单、发票、质检、物流、买方、保险等是否完整。 |
| Agent 风险建议 | 风险提示、缺口、建议动作。 |
| 决策区 | 允许、拒绝、补材料、降额、人工复核。 |
| 执行条件 | 批准后是否被规则、资金池、暂停或 dispute 阻断。 |
| 贷后基础状态 | 回款、逾期、争议、保险占位。 |

### 8.2 资金方可以触发的原型动作

| 动作 | 业务边界 |
|---|---|
| 查看待审案件 | 只能看自己有权案件。 |
| 要求补材料 | 不修改原始材料。 |
| 允许融资 | 只是资金方决策，不等于到账。 |
| 拒绝融资 | 必须有业务原因。 |
| 降额 | 形成调整建议或决策记录。 |
| 转人工复核 | 高风险和争议进入人工。 |
| 查看执行条件 | 资金方批准后仍需规则和资金池检查。 |

### 8.3 资金方不得看到或做

| 禁止 | 原因 |
|---|---|
| 修改出口商原始发票或合同 | 证据污染。 |
| 冒充质检方确认 Pass | 事实来源越界。 |
| 查看其他资金方专属案件 | 商业隔离。 |
| 绕过资金池不足执行 | 合约/规则边界。 |

## 9. 质检方工作区原型范围

### 9.1 质检方必须看到

| 区域 | 业务表达 |
|---|---|
| 质检任务 | 待接收、已接收、需补充、争议中。 |
| 质检清单 | 商品、数量、包装、标签、照片、说明。 |
| 质检材料状态 | 报告、照片、结论、异常说明是否完整。 |
| 质检结论 | Pass、Fail、Dispute。 |
| 补充请求 | 资金方、平台或 Agent 要求补什么。 |
| 争议状态 | 争议原因、责任方、下一步。 |

### 9.2 质检方可以触发的原型动作

| 动作 | 业务边界 |
|---|---|
| 接收质检任务 | 只处理分配给自己的任务。 |
| 提交质检报告和材料 | 作为质检事实来源。 |
| 提交 Pass/Fail/Dispute | 只影响事实状态，不决定放款。 |
| 补充质检说明 | 不覆盖历史记录。 |

### 9.3 质检方不得看到或做

| 禁止 | 原因 |
|---|---|
| 决定放款 | 资金方职责。 |
| 修改贸易材料原件 | 证据边界。 |
| 查看资金池内部状态 | 与质检任务无关。 |
| 查看无关案件 | 数据隔离。 |

## 10. 平台工作区原型范围

### 10.1 平台必须看到

| 区域 | 业务表达 |
|---|---|
| 运行总览 | 资金池、规则、合约、异常、Agent 自动化状态。 |
| 资金池状态 | 可用、不足、暂停、冻结、堆积、执行失败。 |
| 规则状态 | Active、Paused、Expired、Pending upgrade、Conflict。 |
| 合约执行状态 | 正常、暂停、失败、完成、等待。 |
| 异常队列 | 缺材料、dispute、规则阻断、资金池不足、执行失败。 |
| Agent 状态 | 正常、失败、堆积、需人工接管。 |
| 高影响操作记录 | 暂停、恢复、分派、升级。 |

### 10.2 平台可以触发的原型动作

| 动作 | 业务边界 |
|---|---|
| 暂停/恢复资金池 | 必须有原因、影响范围、恢复条件。 |
| 暂停/恢复规则 | 影响案件必须可见。 |
| 暂停案件 | 只针对异常或风险案件。 |
| 分派异常 | 分派给责任角色。 |
| 查看 Agent 异常 | 自动化失败、堆积、需人工接管。 |

### 10.3 平台不得表达成

| 禁止 | 原因 |
|---|---|
| 平台逐单审批全部案件 | 破坏平台定位。 |
| 平台替资金方批准放款 | 决策权越界。 |
| 平台伪造质检/物流/买方事实 | 事实来源越界。 |
| 平台删除关键审计记录 | 可信基础破坏。 |

## 11. Agent 表达原型范围

Agent 表达必须贯穿角色工作区，但不优先做成聊天窗口。

| Agent 表达 | 原型必须表达 |
|---|---|
| 案件摘要 | 当前案件是什么、处于哪一步、有什么卡点。 |
| 材料缺口 | 缺什么、谁补、为什么影响融资。 |
| 风险提示 | 金额冲突、质检 dispute、资金池不足、规则阻断。 |
| 待办任务 | 谁下一步做什么。 |
| 请求补充 | Agent 已向哪个角色请求补充。 |
| 自动推进记录 | Agent 在授权范围内完成了什么。 |
| 人工升级原因 | 为什么不能自动继续。 |
| 状态解释 | 把合约/规则/平台状态解释为业务语言。 |

边界：

- Agent 摘要不等于事实。
- Agent 建议不等于审批。
- Agent 待办不等于强制动作。
- Agent 升级不等于失败。
- Agent 不得掩盖合约阻断。

## 12. 控制塔主链路原型范围

控制塔必须表达 P0 主链路：

```text
融资意图
→ 材料准备
→ Agent 检查
→ 事实确认
→ 资金方审查 / DeFi 规则路径
→ 合约/规则执行检查
→ 资金结果
```

每个节点必须表达：

| 表达项 | 说明 |
|---|---|
| 节点名称 | 当前处于哪个业务节点。 |
| 节点状态 | Not started / Waiting / Need materials / Pass / Fail / Dispute / Blocked / Completed。 |
| 责任方 | 当前谁处理。 |
| 卡点原因 | 为什么不能继续。 |
| 下一步 | 谁做什么。 |
| 是否可自动推进 | Agent 是否可继续。 |
| 是否需人工升级 | 是否需要人工。 |

## 13. 合约/规则状态原型范围

| 合约/规则状态 | 原型业务表达 |
|---|---|
| Allowed | 当前满足执行条件。 |
| Waiting | 等待某个条件满足。 |
| Need materials | 缺材料或事实。 |
| Need review | 需要人工复核。 |
| Rule blocked | 当前规则不允许继续。 |
| Pool insufficient | 资金池不足或不可用。 |
| Platform paused | 平台暂停相关能力。 |
| Dispute | 有争议，需人工处理。 |
| Execution failed | 执行失败，需平台或责任方处理。 |
| Completed | 执行完成。 |

原型不得表达成：

| 错误表达 | 原因 |
|---|---|
| 合约认定贸易真实 | 合约不判断事实。 |
| 合约自动审批融资 | 合约不是资金方。 |
| 合约裁决 dispute | 合约不裁决争议。 |
| 合约失败等于欺诈 | 失败可能来自资金池、规则、暂停或执行问题。 |

## 14. 异常卡点原型范围

| 异常 | 原型必须表达 |
|---|---|
| 缺材料 | 缺什么、谁补、补完后怎么继续。 |
| 金额不一致 | 冲突位置、责任方、解释或复核路径。 |
| 质检 Fail | 是否拒绝、复检、人工复核或补充说明。 |
| 质检 Dispute | 争议原因、责任方、解除条件。 |
| 物流异常 | 延误、损坏、丢失等占位说明。 |
| 买方争议 | 争议内容、付款责任影响、人工处理。 |
| 保险不足 | 需承保、加费、补材料或拒保占位。 |
| 资金池不足 | 等待、恢复、切换路径或人工处理。 |
| 规则阻断 | 哪类规则阻断、是否需平台处理。 |
| 平台暂停 | 暂停对象、原因、影响、恢复条件。 |
| 执行失败 | 失败原因、责任方、下一步。 |
| Agent 置信不足 | 升级给所属角色人工确认。 |

通过标准：

```text
任何卡点都不能只显示“失败”。
```

## 15. 路径差异原型范围

| 路径 | 原型必须表达 |
|---|---|
| 银行辅助路径 | 资金方审查、决策、批准后仍需合约/规则/资金池检查。 |
| 纯 DeFi 路径 | 无银行人工审批，但必须满足事实、规则、资金池和暂停条件。 |
| 低人工路径 | Agent 在授权、低风险、事实完整、规则明确时推进。 |
| 人工升级路径 | 高风险、争议、失败、权限不清时停止自动推进并交给人。 |

原型不得把四条路径合并成：

- 单一银行审批流。
- 无责任的自动 DeFi 流。
- 普通后台状态列表。

## 16. P1/P2 占位原型范围

| 占位内容 | P0 表达方式 | 后置阶段 |
|---|---|---|
| 物流方工作区 | 只表达物流事实状态和异常占位。 | P1 |
| 保险工作区 | 只表达承保、拒保、补材料、理赔占位。 | P1 |
| 买方工作区 | 只表达订单确认、收货、争议、付款责任占位。 | P1 |
| 海关/报关接入 | 只表达通关事实价值和后置标记。 | P2 |
| 多资金方匹配 | 只保留基础资金方路径。 | 后续资金市场 |
| 复杂贷后催收 | 只表达基础贷后状态。 | 后续贷后 |
| hash/proof 展示 | 不进入 P0 主线。 | 后续证明产品 |

## 18. 原型与概念数据对象关系

| 原型表达 | 对应概念对象 |
|---|---|
| 钱包和账户状态 | Wallet Identity、Business Account、Role Binding、Agent Mode。 |
| 融资案件 | Financing Case、Financing Intent、Case Status、Case Path。 |
| 贸易材料 | Trade Material Set、Material Version Record、Completeness State。 |

## 23. 一票否决项

| 一票否决项 | 原因 |
|---|---|
| 原型范围变成普通后台列表 | 破坏控制塔定位。 |
| 出口商看不到资金结果和卡点 | P0 核心价值失败。 |
| 角色工作区边界消失 | 权限模型失败。 |
| Agent 建议被表达成审批结论 | Agent 越界。 |
| 合约输出被表达成贸易真实性判断 | 合约越界。 |
| 平台被表达成逐单审批员 | 平台定位错误。 |
| 路径差异被合并 | DeFi/银行/低人工/人工升级边界失败。 |
| P1/P2 完整工作区塞入 P0 必做 | MVP 范围失控。 |
| 旧 proof/hash demo 页面成为主线 | 产品定位倒退。 |
| 直接写 UI 组件、路由、样式或前端代码 | 阶段越界。 |
