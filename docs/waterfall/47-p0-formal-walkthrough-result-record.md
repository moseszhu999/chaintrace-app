分类规则：

| 分类 | 含义 | 对冻结的影响 |
|---|---|---|
| A 类 | 不改变业务含义的表达优化。 | 可调整后复核。 |
| B 类 | waterfall 需求缺口。 | 先补需求文档，不得直接冻结。 |
| C 类 | 有价值但后置。 | 记录到后置范围，不阻断 P0。 |
| D 类 | 破坏定位、路径、权限、Agent、DeFi 或异常逻辑。 | 阻断冻结，必须关闭。 |

## 9. 不得通过的情况

正式走读出现以下任一情况，本轮不得通过。

| 不得通过情况 | 处理方式 |
|---|---|
| 原型被理解成销售 demo 或登录前营销页。 | 退回产品定位表达。 |
| 看不出供应链源头到终端控制塔。 | 退回控制塔主表达。 |
| 所有路径都像银行审批。 | 退回路径差异。 |
| 纯 DeFi 仍等待银行审批。 | D 类阻断。 |
| Agent 被理解成审批人、事实制造者或争议裁决者。 | D 类阻断。 |
| 合约被理解成页面、数据库或事实判断者。 | D 类阻断。 |
| 平台能伪造事实或替资金方审批。 | D 类阻断。 |
| 事实方能决定融资。 | D 类阻断。 |
| 角色能看到无关案件或越权动作。 | D 类阻断。 |
| 异常没有原因、责任方、解除条件或下一步。 | 退回异常回流。 |
| 进入 API endpoint、database schema、Solidity、wallet signature 或 hash/proof product page。 | 退回阶段边界。 |
