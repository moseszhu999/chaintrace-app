# 37 P0 API 业务契约

## 1. 定位

本文件定义 ChainTrace P0 阶段的 API 业务契约。

这里的 API 业务契约不是接口设计，不是 endpoint 清单，不是 HTTP 方法，不是请求响应格式，也不是后端实现方案。

它的作用是：

> 用业务语言说明后续系统需要承载哪些业务动作、输入含义、输出含义、失败解释、权限边界和状态影响，让产品原型、Agent、控制塔、规则和 DeFi 智能合约都有一致的业务口径。

本文件不写页面、组件、接口路径、请求参数格式、数据库、Solidity、ABI、部署、钱包签名、hash 或 proof 细节。

## 2. 业务契约原则

| 原则 | 说明 |
|---|---|
| 业务动作优先 | 契约围绕业务动作定义，而不是围绕技术接口定义。 |
| 责任角色清楚 | 每个动作必须知道由谁发起、谁负责、谁可见。 |
| 输入是业务含义 | 只说明需要哪些业务信息，不定义字段名和格式。 |
| 输出是业务结果 | 输出必须说明状态推进、待办、事实、决策、阻断或执行结果。 |
| 失败必须可解释 | 不允许只返回技术失败，必须说明业务原因和下一步。 |
| Agent 可使用 | Agent 能基于契约整理摘要、生成待办、自动推进或升级人工。 |
| 控制塔可表达 | 契约输出必须能进入控制塔节点、状态、卡点和事件。 |
| 合约边界清楚 | DeFi 智能合约只接收业务条件结果，不读取或解释完整材料。 |

## 3. 业务契约通用结构

每个业务动作契约至少需要说明以下内容。

| 契约项 | 业务问题 |
|---|---|
| 动作名称 | 用户或系统正在做什么？ |
| 发起角色 | 谁可以发起这个动作？ |
| 适用路径 | 银行辅助、纯 DeFi、低人工还是通用路径？ |
| 前置条件 | 什么状态下可以发起？ |
| 业务输入 | 需要哪些业务信息或确认？ |
| 业务输出 | 成功后产生什么结果？ |
| 状态影响 | 案件、节点、待办、事实、规则或资金状态如何变化？ |
| 事件影响 | 是否产生业务事件？ |
| 控制塔影响 | 哪个节点变成完成、等待、异常、阻断或跳过？ |
| 失败解释 | 为什么不能继续？ |
| 下一步 | 由谁补材料、确认、授权、处理异常或等待？ |
| 权限边界 | 谁不能发起、不能查看、不能修改？ |

## 4. 通用输入含义

本节只定义输入的业务含义，不定义字段、类型或格式。

| 输入类型 | 业务含义 | 常见来源 |
|---|---|---|
| 角色身份 | 发起人是谁，钱包绑定了什么业务角色。 | 注册和角色绑定 |
| 案件标识 | 当前动作属于哪一笔融资/结算案件。 | 案件创建 |
| 场景路径 | 当前走银行辅助、纯 DeFi、低人工或人工升级。 | 规则 / Agent |
| 贸易信息 | 合同、订单、发票、装箱单、金额、币种、货物。 | 出口商 / 业务方 |
| 事实信息 | 质检、物流、通关、保险、买方确认等事实。 | 事实方 |
| Agent 建议 | 摘要、缺口、风险、待办、自动推进建议。 | Agent |
| 授权信息 | 资金方、用户或平台允许某个动作继续。 | 资金方 / 用户 / 平台 |
| 规则状态 | 当前规则是否适用、有效、暂停或阻断。 | 平台 / 规则 |
| 资金池状态 | 资金池是否可用、足额、暂停或异常。 | 平台 / 合约业务层 |
| 异常说明 | 风险、争议、失败、冲突或补充说明。 | 相关角色 / Agent |

## 5. 通用输出含义

每个业务动作输出都必须落到业务可理解结果。

| 输出类型 | 业务含义 | 控制塔表达 |
|---|---|---|
| 状态推进 | 案件进入下一业务状态。 | 节点完成或进入下一节点。 |
| 生成待办 | 某个角色需要处理。 | 待办层出现责任人和完成标准。 |
| 形成事实 | 质检、物流、通关、保险或买方事实被确认。 | 对应事实节点完成、异常或争议。 |
| 形成决策 | 允许、拒绝、补材料、人工复核、暂停。 | 决策节点更新。 |
| 进入执行检查 | 规则、资金池、授权、抵押/保证进入检查。 | DeFi 执行节点进行中。 |
| 等待条件 | 仍缺材料、事实、授权、资金池或规则条件。 | 节点 Waiting。 |
| 规则阻断 | 当前规则不允许继续。 | 节点 Blocked 并说明解除条件。 |
| 资金池不足 | 资金池不可用、余额不足或暂停。 | 执行节点 Blocked 或 Waiting。 |
| 执行完成 | 放款、结算、锁定、释放或到账。 | 资金结果节点 Completed。 |
| 执行失败 | 尝试执行但没有完成。 | 执行节点 Failed，平台或相关角色处理。 |
| 人工升级 | 争议、高风险、监管、法律责任或越权风险。 | Manual review 节点出现。 |
| 关闭案件 | 案件终止或完成后关闭。 | 案件进入 Closed。 |

## 6. 角色业务契约

### 6.1 出口商 / 资金需求方

| 业务动作 | 业务输入 | 业务输出 | 失败或阻断解释 |
|---|---|---|---|
| 创建融资/结算意图 | 角色身份、贸易基础、金额和币种、期望结果。 | 案件 Draft。 | 钱包未绑定出口商角色、角色被暂停、信息不足。 |
| 提交贸易材料 | 合同、订单、发票、装箱单、补充说明。 | 材料进入案件，Agent 可检查。 | 案件不可编辑、材料缺失、越权提交。 |
| 请求 Agent 检查 | 已有材料和案件状态。 | 生成摘要、缺口和建议。 | 材料不足到无法摘要、Agent 模式未授权自动检查。 |
| 确认提交案件 | 最低材料满足、用户确认。 | 案件进入 Submitted / Material checking。 | 缺关键材料、高风险需人工确认、角色暂停。 |
| 回应补材料 | 补充材料或说明、责任待办。 | 回到检查、事实确认或审查流程。 | 补充内容不对应待办、无权限修改、补充仍不足。 |
| 申请关闭案件 | 关闭原因、当前状态。 | 案件进入关闭申请或 Closed。 | 已进入不可撤回执行、存在未处理争议或规则限制。 |

### 6.2 资金方 / 银行 / 保理机构

资金方契约只适用于银行辅助路径。

| 业务动作 | 业务输入 | 业务输出 | 失败或阻断解释 |
|---|---|---|---|
| 查看待审案件 | 资金方身份、分配关系、银行辅助路径。 | 可审案件列表和摘要。 | 无关案件不可见、路径不需要银行、角色暂停。 |
| 请求补材料 | 缺口说明、责任角色、完成标准。 | 补材料待办生成。 | 缺口不清楚、请求对象无关、案件状态不允许。 |
| 标记人工复核 | 风险、争议、金额、监管或授权原因。 | 案件进入 Manual review。 | 资金方无权处理该案件、理由不足。 |
| 允许放款 | 审查结论、授权金额、条件、有效期。 | 案件进入 Approved / Execution pending。 | 事实不足、高风险未处理、规则或平台暂停。 |
| 拒绝融资 | 拒绝原因和业务说明。 | 案件进入 Rejected / Closed。 | 无拒绝原因、非相关资金方、案件已关闭。 |
| 查看执行条件 | 已授权案件或等待执行案件。 | 看到允许执行、等待、阻断、失败或完成。 | 非相关案件、执行条件尚未生成。 |

### 6.3 事实方

事实方包括质检方、物流方、海关/报关方、保险公司、买方/进口商。

| 业务动作 | 业务输入 | 业务输出 | 失败或阻断解释 |
|---|---|---|---|
| 查看待确认任务 | 事实方身份、任务分配、必要上下文。 | 可处理任务和节点摘要。 | 无关任务不可见、角色类型不匹配。 |
| 接受任务 | 任务可接受、角色确认。 | 节点进入 In progress。 | 任务已关闭、任务不属于该角色、角色暂停。 |
| 提交事实材料 | 报告、照片、运单、通关文件、承保材料、收货说明等。 | 材料进入对应事实节点。 | 材料不属于该节点、缺关键说明、越权提交。 |
| 提交事实结论 | pass / fail / dispute 或对应业务结论。 | 事实节点 Completed / Warning / Dispute。 | 结论缺原因、任务未接受、事实冲突未说明。 |
| 请求补充说明 | 缺口、责任角色、补充原因。 | 生成补充待办。 | 请求对象无关、理由不足、越权要求修改事实。 |

### 6.4 平台方 / 合约运营

| 业务动作 | 业务输入 | 业务输出 | 失败或阻断解释 |
|---|---|---|---|
| 查看运行总览 | 平台身份、运营范围。 | 资金池、规则、暂停、异常、Agent 堆积总览。 | 非平台角色不可见。 |
| 维护规则状态 | 规则、适用路径、暂停/恢复原因、影响范围。 | 规则启用、暂停、恢复或版本口径变化。 | 影响范围不清楚、缺原因、越权修改业务事实。 |
| 维护资金池状态 | 资金池、币种、暂停/恢复/异常原因。 | 资金池状态变化。 | 缺原因、与实际状态不一致、权限不足。 |
| 暂停案件或路径 | 风险、异常、规则或平台原因。 | 案件或路径进入 Blocked / Paused。 | 原因不足、影响范围不清楚。 |
| 恢复案件或路径 | 暂停原因已解除、恢复说明。 | 案件回到可推进状态或重新检查。 | 原因未解除、恢复范围不清楚。 |
| 处理执行失败 | 失败原因、影响范围、下一步责任角色。 | 失败被解释并生成处理路径。 | 不能删除失败记录、不能伪造执行成功。 |
| 处理 Agent 异常 | Agent 失败、堆积、越权风险或置信不足。 | 降级人工、调整模式或分派处理。 | 不能替业务角色确认事实。 |

### 6.5 Agent

Agent 契约始终从属于所属角色和授权模式。

| 业务动作 | 业务输入 | 业务输出 | 失败或阻断解释 |
|---|---|---|---|
| 整理摘要 | 所属角色可见材料、事实、状态。 | 角色可读摘要。 | 材料不足、来源不清、越权读取风险。 |
| 检查缺口 | 当前路径最低要求、材料和事实状态。 | 缺口、责任角色、建议动作。 | 规则口径不清、事实冲突、缺口无法归责。 |
| 生成待办 | 缺口、风险、授权或确认节点。 | 责任角色待办。 | 责任角色不清、请求越权、任务重复冲突。 |
| 自动推进低风险流程 | 授权模式、低风险、事实完整、规则明确。 | 案件进入下一状态或执行检查。 | 超出授权、存在争议、高风险、规则阻断。 |
| 异常升级 | 争议、高风险、越权风险、置信不足。 | 进入人工复核或平台处理。 | 不能压制异常，不能自动裁决重大争议。 |

### 6.6 DeFi 智能合约资金执行层

本节只写业务契约，不写合约函数、Solidity 或链上实现。

| 业务动作 | 业务输入 | 业务输出 | 失败或阻断解释 |
|---|---|---|---|
| 检查规则版本 | 案件路径、规则状态、适用范围。 | 规则有效、暂停、过期或不匹配。 | 当前规则不允许继续或需要平台处理。 |
| 检查资金池状态 | 资金池、币种、金额、暂停状态。 | 资金池可用、不足、暂停或异常。 | 资金池不足、暂停、币种不支持或异常。 |
| 检查授权边界 | 银行授权、用户授权、平台状态。 | 授权满足或不足。 | 授权金额、期限、条件或路径不满足。 |
| 检查抵押/保证条件 | 纯 DeFi 或低人工路径所需保证条件。 | 满足、等待或不足。 | 抵押/保证不足或状态不可用。 |
| 执行资金动作 | 条件满足且未暂停。 | 放款、结算、锁定、释放、到账或完成。 | 执行失败、平台暂停、规则变化、资金池异常。 |
| 阻断执行 | 条件不满足。 | 业务阻断结果和解除条件。 | 必须说明规则、资金池、授权、暂停或事实条件原因。 |

## 7. 路径业务契约

| 路径 | 必要动作链 | 业务输出 |
|---|---|---|
| 银行辅助路径 | 创建案件、材料检查、事实确认、资金方审查、授权、DeFi 执行检查、资金动作。 | 允许放款、拒绝、补材料、人工复核、执行等待、阻断或到账。 |
| 纯 DeFi 路径 | 创建案件、材料/事实检查、规则检查、资金池检查、抵押/保证检查、合约执行。 | 自动执行、等待条件、规则阻断、资金池不足、执行完成或失败。 |
| Agent + 智能合约低人工路径 | Agent 整理和推进、事实完整性检查、规则满足、合约锁定/释放/结算、异常升级。 | 正常低人工推进，异常进入人工。 |

## 8. 失败和异常解释契约

所有失败、阻断和等待都必须用业务语言解释。

| 类型 | 必须解释 | 下一步 |
|---|---|---|
| 权限不足 | 当前角色为什么不能做。 | 切换到有权角色或停止。 |
| 角色暂停 | 哪个角色被暂停，影响哪些动作。 | 平台处理或等待恢复。 |
| 材料不足 | 缺什么材料、谁负责补、完成标准是什么。 | 生成补材料待办。 |
| 事实不足 | 缺哪个事实方确认。 | 生成事实方任务或等待。 |
| 事实冲突 | 哪些事实冲突，影响什么。 | 人工复核或补充说明。 |
| 银行授权不足 | 授权金额、期限、条件或审查状态不满足。 | 资金方处理。 |
| 规则阻断 | 哪条业务规则阻断，解除条件是什么。 | 平台或责任角色处理。 |
| 资金池不足 | 哪个资金池、币种或范围不足。 | 平台或资金池运营处理。 |
| 平台暂停 | 暂停对象和原因。 | 等待平台恢复。 |
| Agent 越权风险 | Agent 为什么不能自动推进。 | 降级人工确认。 |
| 重大争议 | 争议主体、影响和责任边界。 | 人工角色或约定仲裁。 |
| 执行失败 | 资金动作为什么没有完成。 | 平台处理并说明结果。 |

## 9. 控制塔输出契约

业务契约输出必须能被控制塔消费。

| 输出对象 | 控制塔需要表达 |
|---|---|
| 案件 | 当前路径、状态、卡点、责任角色、下一步。 |
| 节点 | Not started、Waiting、In progress、Completed、Warning、Dispute、Blocked、Failed、Skipped。 |
| 待办 | 谁处理、处理什么、完成标准、超时或优先级。 |
| 事实 | 来源、结论、原因、是否影响执行。 |
| 决策 | 允许、拒绝、补材料、人工复核、授权条件。 |
| 规则 | 通过、等待、阻断、暂停、解除条件。 |
| 资金池 | 可用、不足、暂停、异常、恢复条件。 |
| 执行 | 允许执行、等待条件、执行失败、执行完成。 |
| 异常 | 风险类型、影响范围、责任角色、人工处理路径。 |

## 10. Agent 使用契约

Agent 使用业务契约时必须遵守以下边界。

| Agent 能做 | 边界 |
|---|---|
| 根据业务输入整理摘要 | 不能改变原始材料和事实。 |
| 根据输出结果生成待办 | 待办必须有责任角色和完成标准。 |
| 根据失败解释提醒用户 | 不能把技术失败伪装成业务完成。 |
| 根据路径自动推进低风险动作 | 必须在授权模式、规则和事实完整范围内。 |
| 根据异常解释升级人工 | 不能压制争议、高风险或越权风险。 |

## 11. 权限边界

| 权限问题 | 业务契约要求 |
|---|---|
| 谁可发起 | 每个动作必须有允许角色。 |
| 谁可查看 | 输出结果必须按案件相关和节点相关过滤。 |
| 谁可修改 | 事实和材料只能由来源角色补充或确认。 |
| 谁可授权 | 银行辅助路径由资金方授权，纯 DeFi 路径由规则和合约检查。 |
| 谁可暂停 | 平台方处理规则、资金池、路径、案件和系统级暂停。 |
| Agent 权限 | 继承所属角色和授权模式。 |
| 合约权限 | 只执行或阻断资金动作，不判断贸易真实性。 |

## 12. MVP 范围

### 12.1 P0 必须定义

| 范围 | 说明 |
|---|---|
| 通用业务契约结构 | 动作、角色、前置、输入、输出、状态、事件、失败解释。 |
| 角色动作契约 | 出口商、资金方、事实方、平台方、Agent、DeFi 合约。 |
| 路径契约 | 银行辅助、纯 DeFi、低人工路径。 |
| 失败解释契约 | 权限、材料、事实、规则、资金池、暂停、争议、执行失败。 |
| 控制塔输出契约 | 契约结果必须能进入节点、状态、待办和卡点表达。 |
| Agent 使用契约 | Agent 如何使用业务结果整理、推进和升级。 |

## 13. 本步结论

P0 API 业务契约不是接口文档。

它的本质是：

> 把 ChainTrace 的角色动作、业务输入、输出结果、失败解释、权限边界、控制塔表达、Agent 使用和 DeFi 执行约束统一成后续产品原型和技术设计都能遵守的业务口径。

下一步应进入：

> P0 原型验收脚本，用具体业务场景检查控制塔、角色权限、Agent 待办、路径差异和 DeFi 执行解释是否讲得通。
