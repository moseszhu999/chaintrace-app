# 09 保险公司 / 承保方功能需求

## 1. 定位

保险公司进入 ChainTrace，不是为了替资金方审批融资，也不是为了修改贸易事实。

保险公司的核心定位是：

> 作为风险缓释方，判断某笔贸易融资案件是否可承保、如何承保，以及出险后是否进入理赔。

保险公司提供的是承保、加费、拒保、补材料、人工核保和理赔处理能力。它的结论可以影响资金方风险判断，但不等于放款审批结论。

资金方决定是否放款，质检方提供货物事实，物流方提供运输事实，保险公司提供风险缓释判断。

## 2. 进入条件

保险公司进入本工作区前，必须已经完成：

| 条件 | 要求 |
|---|---|
| 钱包已连接 | 用户证明自己控制该钱包。 |
| 钱包已注册 | 该钱包已经完成 ChainTrace 注册。 |
| 角色已锁定 | 该钱包永久绑定为保险公司 / 承保方。 |
| 账户已激活 | 可以接收和处理核保、理赔任务。 |
| 承保能力已登记 | 已填写支持国家、品类、险种、币种、限额等。 |

不处理登录前展示页。
不处理未注册用户的营销转化页面。

## 3. 保险公司最想完成的业务结果

| 优先级 | 业务结果 | 说明 |
|---|---|---|
| P0 | 知道哪些案件需要核保 | 保险公司不应该自己找融资案件。 |
| P0 | 快速看到风险摘要 | 贸易、物流、质检、买方、国家风险要集中呈现。 |
| P0 | 给出承保结论 | 承保、拒保、加费、补材料、人工核保。 |
| P0 | 判断保险覆盖是否支持融资 | 资金方需要知道保险是否足够缓释风险。 |
| P1 | 处理出险和理赔材料 | 损坏、丢失、买方拒付等可能进入理赔。 |
| P1 | 让 Agent 整理核保和理赔材料 | 减少人工反复翻材料。 |

## 4. 保险公司核心功能模块

### 4.1 待核保案件接收

功能要求：

| 功能 | 说明 |
|---|---|
| 查看待核保案件 | 只显示分配给该保险公司或与承保能力匹配的案件。 |
| 查看核保来源 | 出口商主动申请、资金方要求、Agent 建议或平台分派。 |
| 查看融资案件摘要 | 出口商、买方、贸易金额、货物、国家、物流、质检。 |
| 查看保险需求 | 需要覆盖的金额、期间、风险类型。 |
| 接受或拒绝核保任务 | 无法承保时必须说明原因。 |
| 标记核保优先级 | 普通、加急、高金额、高风险、出险相关。 |

Agent 责任：

| Agent 动作 | 说明 |
|---|---|
| 生成核保摘要 | 汇总贸易、物流、质检、买方、国家风险。 |
| 检查材料缺口 | 告诉保险公司缺哪些核保材料。 |
| 标记高风险因素 | 国家、品类、买方、物流异常、质检 dispute 等。 |
| 提醒处理时限 | 防止融资案件因为核保延迟而卡住。 |

### 4.2 核保风险摘要

保险公司打开案件时，第一眼应该看到风险摘要，而不是文件堆。

| 摘要项 | 说明 |
|---|---|
| 贸易摘要 | 贸易金额、货物、交易双方、付款条款。 |
| 出口商风险 | 历史情况、账户状态、异常记录。 |
| 买方风险 | 买方国家、付款责任、争议记录。 |
| 国家 / 地区风险 | 起运国、目的国、中转国风险。 |
| 商品品类风险 | 易损、限制、价格波动、监管敏感等。 |
| 物流风险 | 延误、损坏、丢失、改港、签收状态。 |
| 质检风险 | Pass、Fail、Dispute、Pending。 |
| 通关风险 | 报关、清关、监管异常、不一致。 |
| 融资影响 | 保险覆盖是否足够支持资金方继续审查。 |

保险公司需要回答：

```text
这笔贸易风险在哪里？
是否可以承保？
如果可以，是否需要加费、限额或补材料？
保险覆盖能否支持融资继续推进？
```

### 4.3 核保结论

保险结论必须统一，避免每个承保人用不同说法。

| 结论 | 含义 | 对融资案件的影响 |
|---|---|---|
| Cover approved | 可以承保。 | 有助于资金方继续融资判断。 |
| Cover declined | 拒保。 | 风险缓释不足，资金方可能拒绝或降额。 |
| Premium adjusted | 可承保但需要加费。 | 资金方和出口商需要确认成本。 |
| Need materials | 需要补材料。 | 案件暂时等待补充。 |
| Manual underwriting | 需要人工核保。 | 高风险或信息不足，不能自动处理。 |
| Coverage insufficient | 保险覆盖不足。 | 不能充分支持融资风险缓释。 |

结论必须带原因：

```text
结论：Need materials
原因：物流状态显示延误，但缺少延误原因和预计到达时间。
建议：要求物流方补充延误说明后再核保。
```

### 4.4 保险覆盖判断

保险公司要说明保险是否足够支持融资，不替资金方决定放款。

| 判断项 | 输出 |
|---|---|
| 覆盖金额是否足够 | 足够 / 不足 / 需要调整 |
| 覆盖期间是否足够 | 足够 / 不足 / 不适用 |
| 覆盖风险是否匹配 | 匹配 / 不匹配 / 部分匹配 |
| 除外责任是否影响融资 | 不影响 / 可能影响 / 明显影响 |
| 是否需要加费 | 不需要 / 需要 |
| 是否需要人工核保 | 不需要 / 需要 |
| 是否支持融资继续 | 支持 / 不支持 / 需要资金方判断 |

表达要求：

| 不要说 | 要说 |
|---|---|
| Coverage validation failed | 保险金额低于申请融资金额，不能完全覆盖资金方风险。 |
| Risk high | 买方所在国家风险较高，建议人工核保。 |
| Missing claim data | 货物损坏照片不足，暂时无法判断是否进入理赔。 |

### 4.5 出险和理赔处理

保险公司还需要处理出险后的理赔材料，但 MVP 不把它做成完整保险核心系统。

功能要求：

| 功能 | 说明 |
|---|---|
| 查看出险案件 | 物流损坏、丢失、买方拒付、质检争议等。 |
| 查看出险材料 | 物流异常、照片、质检报告、买方争议、贸易文件。 |
| 请求补充理赔材料 | 缺照片、缺损失说明、缺责任证明等。 |
| 判断是否进入理赔 | 进入理赔 / 不进入 / 需要补材料 / 人工处理。 |
| 给出理赔处理状态 | 待材料、审核中、拒赔、赔付处理中、已完成。 |

Agent 责任：

| Agent 动作 | 说明 |
|---|---|
| 整理出险材料 | 把物流、质检、买方争议汇总成理赔包。 |
| 检查理赔缺口 | 告诉保险公司缺哪些材料。 |
| 生成理赔待办 | 向物流方、质检方、买方或出口商请求补充。 |
| 提醒资金方 | 出险状态可能影响贷后风险。 |

### 4.6 Agent 材料检查

保险公司提交核保或理赔材料后，Agent 先检查是否足够。

| 检查项 | 输出结果 |
|---|---|
| 核保材料是否完整 | 完整 / 不完整 / 需要人工确认 |
| 贸易摘要是否足够 | 足够 / 不足 |
| 物流风险是否清楚 | 清楚 / 不清楚 / 有异常 |
| 质检结论是否清楚 | Pass / Fail / Dispute / Pending |
| 买方风险是否清楚 | 清楚 / 不足 / 高风险 |
| 保险结论是否明确 | 明确 / 不明确 |
| 理赔材料是否完整 | 完整 / 不完整 / 不适用 |

## 5. 保险公司 Agent 模式差异

| 功能节点 | 全人工确认 | 半自动托管 | 全自动 Agent |
|---|---|---|---|
| 接收核保任务 | 用户手动接受 | Agent 推荐可接任务 | 授权范围内自动接受低风险核保任务 |
| 风险摘要 | 用户查看材料 | Agent 生成摘要，用户确认 | Agent 自动生成摘要并标记异常 |
| 材料缺口 | 用户人工检查 | Agent 检查并生成待办 | Agent 自动请求相关方补充 |
| 核保结论 | 必须用户确认 | 用户确认结论 | 低风险标准案件可生成结论草稿，高风险必须人审 |
| 加费 / 拒保 | 必须用户确认 | 必须用户确认 | 必须用户确认 |
| 理赔处理 | 用户人工处理 | Agent 整理材料 | Agent 只整理，不自动裁决重大理赔 |
| 通知资金方 | 用户或系统通知 | Agent 建议通知内容 | Agent 自动同步低风险状态变化 |

重要边界：

| 边界 | 说明 |
|---|---|
| Agent 可以整理核保材料 | 可以汇总贸易、物流、质检、买方、国家风险。 |
| Agent 可以提示缺口 | 可以告诉保险公司缺什么。 |
| Agent 可以生成结论草稿 | 低风险、标准规则下可以辅助。 |
| Agent 不可以绕过规则承保 | 承保必须符合保险规则和授权边界。 |
| Agent 不可以自动裁决重大理赔 | 重大出险、拒赔、赔付必须人审。 |

## 6. 权限边界

保险公司可以：

| 权限 | 说明 |
|---|---|
| 查看分配给自己的核保案件 | 不能看无关融资案件。 |
| 查看必要贸易、物流、质检、买方信息 | 只看核保和理赔需要的信息。 |
| 给出承保 / 拒保 / 加费 / 补材料 / 人工核保结论 | 作为风险缓释判断。 |
| 查看保险覆盖是否支持融资 | 给资金方提供风险缓释参考。 |
| 处理理赔材料 | 对出险案件进行理赔处理。 |
| 请求相关方补充材料 | 出口商、物流方、质检方、买方等。 |

保险公司不可以：

| 禁止事项 | 原因 |
|---|---|
| 修改自己的永久角色 | 钱包角色不可变。 |
| 替资金方决定放款 | 放款权属于资金方。 |
| 修改贸易事实 | 贸易事实来自出口商和相关事实方。 |
| 修改质检结论 | 质检结论属于质检方。 |
| 修改物流事实 | 物流事实属于物流方。 |
| 绕过规则承保 | 破坏保险和平台风险边界。 |
| 查看无关融资案件 | 数据隔离。 |

## 8. 本步结论

保险公司工作区的本质不是“保险销售页面”。

它的本质是：

> 保险公司作为风险缓释方，判断案件是否可承保、是否需要加费或补材料，以及出险后是否进入理赔；Agent 帮它整理核保和理赔材料，但不替资金方放款，也不伪造贸易事实。

> 买方 / 进口商 / 付款方功能需求。
