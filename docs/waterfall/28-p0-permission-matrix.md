# 28 P0 权限矩阵

## 1. 定位

本文件定义 ChainTrace P0 阶段的业务权限矩阵。

权限矩阵不是数据库权限设计，不是接口权限设计，也不是钱包签名实现说明。

它的作用是：

> 用业务语言明确每个角色能看什么、能做什么、不能做什么，以及 Agent 在不同授权模式下可以代办到什么程度。

本文件不写页面、组件、API、数据库、Solidity、钱包签名、hash 或 proof 细节。

## 2. 权限基本原则

| 原则 | 说明 |
|---|---|
| 钱包绑定单一角色 | 每个钱包注册后永久绑定一个业务身份。 |
| 角色不混用 | 出口商不能同时作为资金方、质检方或平台方操作。 |
| 案件相关可见 | 角色只能查看与自己有关的案件、任务、事实和结果。 |
| 事实来源独立 | 谁提供事实，谁负责事实；其他角色不能篡改。 |
| 决策权按路径归属 | 银行辅助路径中资金方决策；纯 DeFi 路径中合约按规则执行。 |
| 平台管规则和秩序 | 平台方维护规则、资金池、暂停和异常，不伪造业务事实。 |
| Agent 权限从属角色 | Agent 不是单独用户，权限来自所属角色和授权模式。 |
| 关键动作可解释 | 任何授权、阻断、拒绝、暂停、升级都必须能说明原因。 |

## 3. 角色可见范围

| 角色 | 可见范围 | 不可见范围 |
|---|---|---|
| 出口商 / 资金需求方 | 自己创建的案件、自己需补的材料、Agent 摘要、状态、卡点、执行结果。 | 其他出口商案件、资金方内部审查意见、平台全局运营数据。 |
| 资金方 / 银行 / 保理机构 | 分配或匹配给自己的银行辅助案件、材料摘要、事实状态、风险摘要、执行条件结果。 | 无关案件、其他资金方审查意见、平台内部规则维护细节。 |
| 质检方 | 分配给自己的质检任务、必要货物摘要、质检要求、补充要求。 | 融资审批意见、资金池状态、无关案件完整材料。 |
| 平台方 / 合约运营 | 平台运行状态、规则状态、资金池状态、异常、执行失败、Agent 堆积、审计记录。 | 不能以业务事实方身份伪造贸易、质检、物流、保险或买方事实。 |
| DeFi 智能合约 | 业务条件结果、资金池状态、规则状态、授权边界、暂停状态。 | 不查看或解释完整业务材料，不替代 Agent 阅读材料。 |
| Agent | 所属角色可见范围内的材料、状态、待办和摘要。 | 所属角色不可见的案件、事实和运营数据。 |
| 物流方 | 分配给自己的物流任务、货物运输摘要、需确认的物流节点。 | 融资审批意见、无关贸易材料、质检和保险事实。 |
| 海关 / 报关方 | 分配给自己的通关事实任务、必要贸易摘要和报关/清关文件。 | 放款决策、资金池、无关案件。 |
| 保险公司 | 分配给自己的核保或理赔案件、贸易/物流/质检/买方/国家风险摘要。 | 资金方内部审批、平台规则修改入口、无关案件。 |
| 买方 / 进口商 | 与自己相关的订单、收货、争议、付款责任任务。 | 出口商其他融资申请、资金方审批、质检内部过程。 |

## 5. 关键业务对象权限

### 5.1 案件

| 权限 | 允许角色 | 说明 |
|---|---|---|
| 创建 | 出口商 / 业务方 | 创建融资或结算意图。 |
| 查看 | 案件相关角色 | 按角色和任务范围查看。 |
| 补充 | 出口商、相关事实方 | 只能补自己负责的材料或事实。 |
| 推进 | Agent、资金方、平台方、合约 | 按场景和状态推进。 |
| 关闭 | 出口商申请、平台按规则处理、合约记录结果 | 关闭必须有原因。 |

### 5.2 贸易材料

| 权限 | 允许角色 | 不允许 |
|---|---|---|
| 提交 | 出口商 | 资金方、质检方、平台方不得代替提交。 |
| 补充 | 出口商 | 其他角色不能篡改。 |
| 摘要 | Agent | Agent 只能整理，不改变原始事实。 |
| 查看 | 相关资金方、事实方、平台方按需查看 | 无关角色不可见。 |

### 5.3 事实结论

| 事实类型 | 提供方 | 其他角色权限 |
|---|---|---|
| 质检结论 | 质检方 | 其他角色只能查看、质疑、请求补充，不能修改。 |
| 物流事实 | 物流方 | 其他角色只能查看、请求补充，不能伪造。 |
| 通关事实 | 海关 / 报关方 | 其他角色只能查看、请求补充，不能替代确认。 |
| 保险结论 | 保险公司 | 其他角色不能绕过规则承保。 |
| 买方责任 | 买方 / 进口商 | 其他角色不能代替确认收货或付款责任。 |

### 5.4 资金执行

| 权限 | 银行辅助路径 | 纯 DeFi 路径 | 低人工路径 |
|---|---|---|---|
| 决策来源 | 资金方授权 | 规则、资金池、抵押/保证、事实状态 | 预设规则、授权边界、事实状态 |
| 合约作用 | 授权后执行或阻断 | 直接执行或阻断 | 锁定、释放、结算或阻断 |
| 人工介入 | 授信、放款、高风险 | 异常、争议、规则例外 | 重大争议、高风险责任 |
| 不允许 | 合约绕过资金方授权 | 强制等待银行审批 | 自动裁决重大法律争议 |

## 6. Agent 权限矩阵

Agent 权限由两个因素决定：

```text
所属角色权限
×
用户选择的 Agent 模式
```

| Agent 模式 | 可以自动做 | 必须人工确认 | 不允许 |
|---|---|---|---|
| 全人工确认 | 整理摘要、检查缺口、生成建议、提醒。 | 提交、确认、授权、拒绝、关闭、执行。 | 伪造事实、越权查看、自动决策。 |
| 半自动托管 | 整理、提醒、生成待办、推进低风险非关键步骤。 | 放款授权、平台暂停、争议处理、高风险动作。 | 自动裁决重大争议、绕过规则。 |
| 全自动 Agent | 在授权范围内推进低风险、规则明确、事实完整的流程。 | 超额、异常、争议、高风险、监管敏感事项。 | 伪造事实、替资金方高风险放款、替平台绕过规则。 |

Agent 可以：

| 能力 | 边界 |
|---|---|
| 整理材料 | 不改变原始材料。 |
| 生成摘要 | 必须可追溯到已有事实。 |
| 检查缺口 | 只能指出缺口和责任角色。 |
| 生成待办 | 不能替责任角色完成事实确认。 |
| 请求补材料 | 不能编造材料。 |
| 推进低风险流程 | 必须在授权范围内。 |
| 升级异常 | 不能自行压制异常。 |

## 7. 路径权限差异

| 权限问题 | 银行辅助路径 | 纯 DeFi 路径 | 低人工路径 |
|---|---|---|---|
| 谁决定能否放款 | 资金方 | DeFi 规则和合约执行条件 | 预设规则和授权边界 |
| 谁提供事实 | 出口商、质检方、P1/P2 事实方 | 出口商、事实方 | 事实方、Agent 整理 |
| 谁执行资金动作 | DeFi 智能合约 | DeFi 智能合约 | DeFi 智能合约 |
| 谁处理异常 | 资金方、平台方、相关事实方 | 平台方、相关角色 | 人工角色或约定仲裁 |
| 银行是否可见 | 可见相关案件 | 默认不可见 | 默认不可见 |
| 平台是否可操作 | 可暂停、处理规则和异常 | 可暂停、处理资金池和规则 | 可暂停、处理规则和异常 |

## 8. 越权场景清单

以下行为必须被视为越权：

| 越权行为 | 原因 |
|---|---|
| 出口商修改质检结论 | 破坏事实独立性。 |
| 质检方修改贸易材料 | 污染贸易证据。 |
| 资金方修改质检、物流、保险或买方事实 | 破坏事实来源。 |
| 平台方替资金方审批银行辅助融资 | 角色越界。 |
| DeFi 合约绕过银行辅助场景中的资金方授权 | 执行边界错误。 |
| Agent 编造材料或事实 | 违反 Agent Native 基本原则。 |
| Agent 自动裁决重大法律争议 | 超出自动化边界。 |
| 买方修改出口商融资申请 | 角色越界。 |
| 保险公司绕过规则承保 | 风险缓释边界错误。 |
| 无关角色查看无关案件 | 权限隔离失败。 |

## 10. 本步结论

P0 权限矩阵的核心不是技术鉴权。

它的核心是：

> 在单钱包单角色的前提下，确保每个角色只能查看和操作自己的业务范围；Agent 的权限从属于角色和授权模式；银行只在需要银行参与的场景中决策；纯 DeFi 和低人工路径由规则、事实和智能合约约束资金执行。
