# 08 海关 / 报关 / 通关数据方功能需求

## 1. 定位

海关 / 报关 / 通关数据方进入 ChainTrace，不是为了审批融资，也不是为了替资金方做放款判断。

它的核心定位是：

> 为融资案件提供贸易真实性、通关状态和监管异常事实。

海关 / 报关 / 通关数据方是监管和通关事实来源。它负责说明是否报关、是否清关、贸易方向是否一致、监管状态是否正常，以及品类、金额、国家、贸易主体是否存在不一致。

资金方仍然决定是否放款。通关事实只增强贸易真实性和监管可信度。

## 2. 进入条件

海关 / 报关 / 通关数据方进入本工作区前，必须已经完成：

| 条件 | 要求 |
|---|---|
| 钱包已连接 | 用户证明自己控制该钱包。 |
| 钱包已注册 | 该钱包已经完成 ChainTrace 注册。 |
| 角色已锁定 | 该钱包永久绑定为海关 / 报关 / 通关数据方。 |
| 账户已激活 | 可以接收和处理通关事实确认任务。 |
| 服务范围已登记 | 已填写国家/地区、贸易方向、支持文件类型或数据范围。 |

不处理登录前展示页。
不处理未注册用户的营销转化页面。

## 3. 海关 / 报关 / 通关数据方最想完成的业务结果

| 优先级 | 业务结果 | 说明 |
|---|---|---|
| P0 | 知道哪些案件需要确认通关事实 | 不应该自己搜索融资案件。 |
| P0 | 确认贸易方向 | 出口、进口或双向事实必须清楚。 |
| P0 | 上传或确认报关 / 清关文件 | 形成贸易真实性和监管事实证据。 |
| P0 | 确认监管状态 | 正常、异常、待补充、无法确认等。 |
| P0 | 提示关键信息不一致 | 金额、品类、国家、贸易主体不一致必须明确。 |
| P1 | 处理补充通关说明 | 当资金方或平台认为通关事实不足时补充。 |
| P1 | 让 Agent 检查通关材料完整性 | 减少反复补材料。 |

## 4. 海关 / 报关 / 通关数据方核心功能模块

### 4.1 待确认通关事实

功能要求：

| 功能 | 说明 |
|---|---|
| 查看待确认通关事实 | 只显示分配给该角色或与服务范围匹配的案件。 |
| 查看案件摘要 | 出口商、买方、货物、金额、国家、运输方式。 |
| 查看需要确认的事实 | 报关、清关、出口记录、进口记录、监管异常等。 |
| 接受或拒绝确认任务 | 无法确认时必须说明原因。 |
| 标记任务优先级 | 普通、加急、高金额、高风险、监管异常。 |

Agent 责任：

| Agent 动作 | 说明 |
|---|---|
| 生成通关事实摘要 | 告诉确认方需要核对哪些监管事实。 |
| 提醒关键字段 | 金额、品类、国家、贸易主体、贸易方向。 |
| 标记可能不一致项 | 先把疑点展示给确认方。 |
| 提醒截止时间 | 防止案件因为通关确认延迟而卡住。 |

### 4.2 报关 / 清关事实确认

通关事实必须用统一业务语言表达。

| 事实类型 | 说明 |
|---|---|
| 出口报关 | 是否存在出口报关记录或文件。 |
| 进口清关 | 是否存在进口清关记录或文件。 |
| 出口记录 | 出口日期、国家、品类、金额、主体等。 |
| 进口记录 | 进口日期、国家、品类、金额、主体等。 |
| 监管异常 | 扣留、查验、退运、信息不一致、限制品类等。 |
| 无法确认 | 因权限、数据范围、文件不足无法确认。 |

统一状态：

| 状态 | 含义 | 对融资案件的影响 |
|---|---|---|
| Confirmed | 通关事实已确认。 | 增强贸易真实性。 |
| Pending | 正在等待确认或补充。 | 案件可能继续等待。 |
| Inconsistent | 与贸易材料存在不一致。 | 需要补充说明或人工复核。 |
| Regulatory exception | 存在监管异常。 | 可能阻断或进入高风险处理。 |
| Not found | 未找到对应记录。 | 需要出口商或相关方解释。 |
| Unable to confirm | 当前角色无法确认。 | 需要其他来源或人工处理。 |

### 4.3 贸易方向确认

贸易方向必须明确，因为融资案件可能依赖出口、进口或双向事实。

| 贸易方向 | 说明 |
|---|---|
| 出口 | 该案件主要证明出口事实。 |
| 进口 | 该案件主要证明进口事实。 |
| 双向 | 同时需要出口和进口记录。 |
| 无法确认 | 当前材料或数据不足以判断。 |

确认项：

| 确认项 | 说明 |
|---|---|
| 出口商是否一致 | 报关主体是否与案件出口商一致。 |
| 买方 / 收货方是否一致 | 清关或进口记录是否与买方相关。 |
| 国家是否一致 | 起运国、目的国是否与贸易材料一致。 |
| 货物品类是否一致 | 报关品类是否与订单、发票、装箱单一致。 |
| 金额是否一致 | 报关金额是否与发票或订单金额存在明显差异。 |

### 4.4 不一致和监管异常提示

海关 / 报关 / 通关数据方不替资金方审批，但必须把监管事实讲清楚。

| 异常类型 | 说明 |
|---|---|
| 金额不一致 | 报关金额与发票、订单金额明显不一致。 |
| 品类不一致 | 报关品类与贸易材料品类不一致。 |
| 国家不一致 | 起运国、目的国或中转信息异常。 |
| 主体不一致 | 报关主体、出口商、买方或收货方不一致。 |
| 贸易方向不一致 | 案件说出口，但只有进口或无法匹配出口记录。 |
| 监管异常 | 查验、扣留、退运、限制品类、监管提示等。 |
| 文件缺失 | 缺少报关单、清关文件或必要编号。 |

处理方式：

| 处理 | 说明 |
|---|---|
| 标记异常 | 明确异常类型。 |
| 上传说明 | 解释异常来源和影响。 |
| 请求补充 | 要求出口商、报关行或其他角色补材料。 |
| 进入人工复核 | 高风险监管异常交给资金方或平台判断。 |

### 4.5 Agent 材料检查

通关材料提交后，Agent 先检查是否足够给融资链条使用。

| 检查项 | 输出结果 |
|---|---|
| 是否上传报关 / 清关文件 | 已上传 / 未上传 / 不适用 |
| 是否确认贸易方向 | 已确认 / 未确认 |
| 金额是否一致 | 一致 / 不一致 / 无法判断 |
| 品类是否一致 | 一致 / 不一致 / 无法判断 |
| 国家是否一致 | 一致 / 不一致 / 无法判断 |
| 贸易主体是否一致 | 一致 / 不一致 / 无法判断 |
| 是否存在监管异常 | 无 / 有 / 无法判断 |
| 是否需要补充材料 | 不需要 / 需要补充 |

Agent 给确认方的表达必须是业务语言：

| 不要说 | 要说 |
|---|---|
| Customs validation failed | 报关金额与发票金额不一致，需要补充说明。 |
| Missing clearance data | 还缺少清关文件，无法确认进口事实。 |
| Country mismatch | 报关目的国与订单目的国不一致。 |
| Regulatory exception | 当前存在监管异常，建议资金方人工复核。 |

## 5. 海关 / 报关 / 通关数据方 Agent 模式差异

| 功能节点 | 全人工确认 | 半自动托管 | 全自动 Agent |
|---|---|---|---|
| 接收任务 | 用户手动接受 | Agent 推荐可处理任务 | 授权范围内自动接受低风险确认任务 |
| 材料整理 | 用户手动上传和确认 | Agent 整理文件摘要，用户确认 | Agent 自动归类文件并检查缺口 |
| 贸易方向确认 | 用户选择 | Agent 建议方向，用户确认 | Agent 可生成方向草稿，异常仍需人确认 |
| 不一致提示 | 用户人工发现 | Agent 标记疑点 | Agent 自动生成不一致清单 |
| 补充材料 | 用户处理 | Agent 生成补充任务 | Agent 自动请求相关方补充 |
| 监管异常 | 用户人工说明 | Agent 汇总异常 | 必须交给人确认 |

重要边界：

| 边界 | 说明 |
|---|---|
| Agent 可以整理通关材料 | 可以归类报关、清关、进出口记录。 |
| Agent 可以检查字段一致性 | 可以提示金额、品类、国家、主体差异。 |
| Agent 可以请求补充 | 可以向相关角色生成补材料待办。 |
| Agent 不可以伪造监管事实 | 不能编造报关、清关或监管状态。 |
| Agent 不可以自动裁决监管异常 | 高风险监管事项必须人处理。 |

## 6. 权限边界

海关 / 报关 / 通关数据方可以：

| 权限 | 说明 |
|---|---|
| 查看分配给自己的通关确认任务 | 不能看无关融资案件。 |
| 查看必要贸易信息 | 只看完成通关确认所需信息。 |
| 上传或确认报关 / 清关文件 | 提供贸易真实性和监管事实。 |
| 确认贸易方向 | 出口、进口、双向或无法确认。 |
| 标记监管异常 | 对监管事实进行说明。 |
| 提示信息不一致 | 金额、品类、国家、主体等。 |
| 补充通关说明 | 对资金方或平台要求进行补充。 |

海关 / 报关 / 通关数据方不可以：

| 禁止事项 | 原因 |
|---|---|
| 修改自己的永久角色 | 钱包角色不可变。 |
| 决定是否融资 | 放款权属于资金方。 |
| 替资金方审批 | 通关事实不等于授信结论。 |
| 替出口商补贸易材料 | 出口商负责自己的贸易材料。 |
| 替平台修改规则 | 规则权属于平台/合约运营。 |
| 修改质检、物流、保险事实 | 各角色事实边界必须清楚。 |
| 查看无关融资案件 | 数据隔离。 |

## 8. 本步结论

海关 / 报关 / 通关数据方工作区的本质不是“监管系统复制品”。

它的本质是：

> 作为贸易真实性和监管事实来源，提供报关、清关、进出口记录和监管异常事实；Agent 帮它检查通关材料完整性和字段一致性，但不替代真实监管事实，也不替资金方审批。
