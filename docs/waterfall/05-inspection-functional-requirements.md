# 05 质检方 / 第三方检验机构功能需求

## 1. 定位

质检方进入 ChainTrace，不是为了审批融资，也不是为了决定是否放款。

质检方的核心定位是：

> 为贸易融资案件提供独立、可信、可复核的货物事实。

质检方在链条中的作用是事实提供方，不是资金决策方。

资金方决定是否放款，平台决定规则和异常处理，质检方只负责把货物质量、数量、包装、现场状态等事实讲清楚。

## 2. 进入条件

质检方进入本工作区前，必须已经完成：

| 条件 | 要求 |
|---|---|
| 钱包已连接 | 用户证明自己控制该钱包。 |
| 钱包已注册 | 该钱包已经完成 ChainTrace 注册。 |
| 角色已锁定 | 该钱包永久绑定为质检方 / 第三方检验机构。 |
| 账户已激活 | 可以接收和处理质检任务。 |
| 服务能力已登记 | 已填写服务国家/地区、可质检品类、支持材料类型等。 |

不处理登录前展示页。
不处理未注册用户的营销转化页面。

## 3. 质检方最想完成的业务结果

| 优先级 | 业务结果 | 说明 |
|---|---|---|
| P0 | 知道哪些案件需要我质检 | 质检方不应该自己找案件。 |
| P0 | 知道每个案件要检查什么 | Agent 生成清晰质检清单。 |
| P0 | 上传质检报告、照片、现场说明 | 形成案件事实证据。 |
| P0 | 给出 pass / fail / dispute 结论 | 用统一结论进入融资链条。 |
| P0 | 知道我的材料是否被接受 | Agent 检查材料完整性，资金方可查看。 |
| P1 | 处理补充质检要求 | 当资金方或平台认为证据不足时，质检方补充。 |
| P1 | 处理争议或复检 | 当出口商、买方或资金方对质检有疑问时，进入 dispute 处理。 |

## 4. 质检方核心功能模块

### 4.1 待质检任务接收

功能要求：

| 功能 | 说明 |
|---|---|
| 查看待质检案件 | 只显示分配给该质检方或与其服务能力匹配的案件。 |
| 查看质检优先级 | 普通、加急、高金额、高风险、争议复检。 |
| 查看质检截止时间 | 质检任务必须有完成期限。 |
| 查看服务地点 | 国家、城市、仓库、港口、工厂或其他地点。 |
| 查看货物基本信息 | 品类、数量、包装、订单信息。 |
| 接受或拒绝质检任务 | 如果质检方无法服务，需要明确拒绝原因。 |

Agent 责任：

| Agent 动作 | 说明 |
|---|---|
| 自动匹配任务 | 根据地区、品类、语言、能力匹配质检方。 |
| 生成任务摘要 | 告诉质检方这是什么货、在哪里、为什么需要质检。 |
| 标记关键检查点 | 让质检方知道本案最重要的风险点。 |
| 提醒截止时间 | 防止案件因为质检延迟而卡住。 |

### 4.2 质检清单生成

质检方不应该自己猜要检查哪些项目。

功能要求：

| 检查项 | 说明 |
|---|---|
| 货物品类 | 是否与订单/发票一致。 |
| 数量 | 是否与订单、装箱单一致。 |
| 包装 | 是否符合贸易文件要求。 |
| 外观 | 是否存在明显损坏、污染、缺陷。 |
| 标签 / 标识 | 是否与买方要求或出口文件一致。 |
| 批次 / 型号 | 是否与合同或订单一致。 |
| 现场照片 | 是否覆盖关键角度和关键标识。 |
| 异常说明 | 如果发现问题，必须说明问题位置和影响。 |
| 抽检范围 | 记录抽检比例、抽检样本或检查范围。 |
| 质检结论 | pass / fail / dispute。 |

不同品类可以有不同清单，但 MVP 阶段先使用通用清单。

### 4.3 质检材料上传

功能要求：

| 材料类型 | 说明 |
|---|---|
| 质检报告 | 正式报告或结构化说明。 |
| 现场照片 | 货物、包装、标签、数量、损坏点等。 |
| 视频 | 可选，用于高风险或争议案件。 |
| 现场说明 | 文字描述现场情况。 |
| 异常说明 | 对不一致、损坏、缺失、争议进行说明。 |
| 补充文件 | 其他支持材料。 |

输入原则：

| 原则 | 说明 |
|---|---|
| 标准选项优先 | 质检结论、品类、问题类型、严重程度使用固定选项。 |
| 现场说明允许手写 | 现场细节无法完全标准化，可以手写。 |
| 文件与结论绑定 | 报告和照片必须对应到一个明确质检结论。 |
| Agent 自动检查完整性 | 上传后自动判断材料是否够用。 |
| 质检方最终确认 | 提交前质检方需要确认结论。 |

### 4.4 质检结论

质检结论必须统一，避免每个质检员用不同说法。

| 结论 | 含义 | 对融资案件的影响 |
|---|---|---|
| Pass | 货物基本符合检查要求。 | 支持继续融资审查。 |
| Fail | 货物明显不符合要求。 | 通常阻断或导致拒绝。 |
| Dispute | 存在争议、信息不足或无法明确判断。 | 进入人工复核或补充流程。 |

质检结论必须带原因：

```text
结论：Dispute
原因：现场数量与装箱单不一致，且出口商未能提供修正说明。
建议：要求出口商补充数量差异说明，或安排复检。
```

### 4.5 Agent 材料检查

质检方提交材料后，Agent 先检查是否足够给资金方使用。

功能要求：

| 检查项 | 输出结果 |
|---|---|
| 是否上传报告 | 已上传 / 未上传 |
| 是否上传照片 | 足够 / 不足 / 未上传 |
| 是否填写质检结论 | 已填写 / 未填写 |
| 结论是否与材料一致 | 一致 / 不一致 / 需要人工确认 |
| 是否有异常说明 | 有 / 无 / 不需要 |
| 是否覆盖关键检查点 | 覆盖 / 未覆盖 / 部分覆盖 |
| 是否需要补充材料 | 不需要 / 需要补充 |

Agent 给质检方的表达必须是业务语言：

| 不要说 | 要说 |
|---|---|
| File validation failed | 缺少货物标签照片，资金方可能无法确认货物型号。 |
| Missing required field | 你还没有选择 pass/fail/dispute 结论。 |
| Evidence weak | 现场照片不足，建议补充包装和数量照片。 |
| Inconsistent result | 你选择了 Pass，但异常说明里写了数量不一致，请确认。 |

### 4.6 补充质检与复检

质检不是一次性动作。资金方、平台、出口商或买方可能要求补充。

功能要求：

| 场景 | 处理方式 |
|---|---|
| 照片不足 | 质检方补充照片。 |
| 报告不清楚 | 质检方补充说明。 |
| 结论与材料冲突 | 质检方修正结论或解释原因。 |
| 出口商不同意结论 | 进入 dispute，可能安排复检。 |
| 买方提出质量争议 | 质检方补充或协助复核。 |
| 资金方要求复核 | 质检方提交补充报告。 |
| 平台发现异常 | 质检方说明或重新提交。 |

补充材料不能覆盖原结论历史，应该形成新的补充记录。

## 5. 质检方 Agent 模式差异

| 功能节点 | 全人工确认 | 半自动托管 | 全自动 Agent |
|---|---|---|---|
| 接收任务 | 用户手动接受 | Agent 推荐可接任务 | Agent 在授权范围内自动接受低风险任务 |
| 生成清单 | 用户查看并确认 | Agent 自动生成，用户确认关键点 | Agent 自动生成通用清单 |
| 上传材料 | 用户手动上传 | Agent 辅助整理材料 | Agent 自动整理上传材料结构 |
| 检查材料完整性 | 用户逐项确认 | Agent 检查并提示缺口 | Agent 自动检查，只提示异常 |
| 提交结论 | 必须用户确认 | 用户确认结论 | 低风险、标准任务可按授权提交草稿，但最终结论仍建议人工确认 |
| 补充材料 | 用户处理 | Agent 生成补充任务 | Agent 自动提示并整理补充材料 |
| 争议处理 | 用户人工处理 | Agent 汇总争议点 | Agent 只做整理，不自动裁决争议 |

重要边界：

| 边界 | 说明 |
|---|---|
| Agent 可以整理事实 | 可以提取报告、照片、异常点。 |
| Agent 可以提示缺口 | 可以告诉质检方缺什么。 |
| Agent 不应伪造现场事实 | 不能替代真实现场质检。 |
| Agent 不应自动裁决重大争议 | 争议必须交给人处理。 |
| Pass/Fail/Dispute 原则上应由质检方确认 | 尤其是高金额、高风险、争议案件。 |

## 6. 权限边界

质检方可以：

| 权限 | 说明 |
|---|---|
| 查看分配给自己的质检任务 | 不能看无关案件。 |
| 查看必要贸易信息 | 只看完成质检所需信息。 |
| 上传质检报告和现场材料 | 为案件提供事实证据。 |
| 给出 pass/fail/dispute 结论 | 作为质检事实进入融资流程。 |
| 补充质检材料 | 对资金方或平台要求进行补充。 |
| 参与争议说明 | 解释自己的质检依据。 |

质检方不可以：

| 禁止事项 | 原因 |
|---|---|
| 修改自己的永久角色 | 钱包角色不可变。 |
| 决定是否放款 | 放款权属于资金方。 |
| 修改出口商原始贸易材料 | 防止证据污染。 |
| 修改资金方决策 | 决策权属于资金方。 |
| 修改平台规则 | 规则权属于平台/合约运营。 |
| 查看无关融资案件 | 数据隔离。 |
| 伪造物流、海关、保险事实 | 质检方只负责质检事实。 |

## 7. MVP 范围

### P0 必须做

| 功能 | 说明 |
|---|---|
| 质检方角色进入 | 已注册质检方钱包进入质检工作区。 |
| 查看待质检任务 | 看到分配给自己的案件。 |
| 查看质检清单 | 知道要检查什么。 |
| 上传质检报告和照片 | 形成质检证据。 |

| 提交质检结论 | 让出口商和资金方可查看。 |

## 8. 本步结论

质检方工作区的本质不是“上传报告页面”。

它的本质是：

> 质检方作为独立事实来源，为融资案件提供可信的货物状态证据；Agent 帮它接收任务、生成清单、检查材料完整性、提示补充，但不替代真实现场事实和重大争议判断。
