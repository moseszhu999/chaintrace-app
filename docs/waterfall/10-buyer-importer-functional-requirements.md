# 10 买方 / 进口商 / 付款方功能需求

## 1. 定位

买方 / 进口商 / 付款方进入 ChainTrace，不是为了申请融资，也不是为了替资金方审批。

买方的核心定位是：

> 确认这笔贸易是否与自己有关、是否收货、是否存在争议、是否承担付款责任。

买方是付款责任和收货事实确认方。它的确认可以增强贸易真实性和回款可信度，但不等于融资审批。

出口商提出融资需求，资金方决定是否放款，买方只确认自己相关的贸易事实和付款责任。

## 2. 进入条件

买方进入本工作区前，必须已经完成：

| 条件 | 要求 |
|---|---|
| 钱包已连接 | 用户证明自己控制该钱包。 |
| 钱包已注册 | 该钱包已经完成 ChainTrace 注册。 |
| 角色已锁定 | 该钱包永久绑定为买方 / 进口商 / 付款方。 |
| 账户已激活 | 可以查看和确认与自己相关的贸易事实。 |
| 付款主体信息已完成 | 已填写企业名称、国家、付款联系人、支持币种等。 |

不处理登录前展示页。
不处理未注册用户的营销转化页面。

## 3. 买方最想完成的业务结果

| 优先级 | 业务结果 | 说明 |
|---|---|---|
| P0 | 知道哪些订单需要我确认 | 买方只处理与自己相关的贸易。 |
| P0 | 确认订单真实性 | 确认这笔订单是否真实存在。 |
| P0 | 确认收货状态 | 未收货、部分收货、已收货、拒收等。 |
| P0 | 提出争议 | 金额、质量、数量、交付、付款责任有问题时提出 dispute。 |
| P0 | 确认付款到期责任 | 确认是否存在付款义务和到期时间。 |
| P1 | 让 Agent 提醒关键确认事项 | 不遗漏收货、争议、付款到期等关键动作。 |

## 4. 买方核心功能模块

### 4.1 待确认订单 / 收货 / 付款责任

功能要求：

| 功能 | 说明 |
|---|---|
| 查看待确认订单 | 只显示与该买方相关的贸易和融资案件。 |
| 查看订单摘要 | 出口商、货物、金额、付款条款、交付状态。 |
| 查看确认请求来源 | 出口商邀请、资金方要求、Agent 提醒或平台分派。 |
| 查看确认类型 | 订单真实性、收货状态、付款责任、争议处理。 |
| 接受或拒绝确认请求 | 如果不是本方贸易，必须明确拒绝原因。 |

Agent 责任：

| Agent 动作 | 说明 |
|---|---|
| 生成确认摘要 | 告诉买方需要确认什么。 |
| 提醒关键风险 | 金额不一致、交付延误、质检 dispute、物流异常。 |
| 提醒确认时限 | 防止融资案件因为买方未响应而卡住。 |
| 把确认动作转成待办 | 订单确认、收货确认、付款责任确认、争议说明。 |

### 4.2 订单真实性确认

买方首先要确认这笔贸易是否与自己有关。

| 结论 | 含义 | 对融资案件的影响 |
|---|---|---|
| Confirmed | 订单真实，买方认可该贸易关系。 | 增强贸易真实性。 |
| Not recognized | 买方不认可该订单。 | 高风险异常，通常需要人工复核。 |
| Need clarification | 信息不足，需要出口商补充。 | 案件等待补充。 |
| Dispute | 订单存在争议。 | 进入争议处理。 |

确认项：

| 确认项 | 说明 |
|---|---|
| 出口商是否正确 | 是否与买方真实交易对象一致。 |
| 订单金额是否正确 | 是否与买方内部订单一致。 |
| 货物品类和数量是否正确 | 是否与采购意图一致。 |
| 付款条款是否正确 | 是否认可付款期限、币种、方式。 |
| 收货安排是否正确 | 是否认可交付地点和收货安排。 |

### 4.3 收货状态确认

买方需要确认货物是否收到，以及是否存在问题。

| 状态 | 含义 | 对融资案件的影响 |
|---|---|---|
| Not received | 尚未收货。 | 需要继续跟踪物流。 |
| Partially received | 部分收货。 | 需要说明差异和影响。 |
| Received | 已收货。 | 支持付款责任判断。 |
| Rejected | 拒收。 | 进入争议或高风险处理。 |
| Damaged received | 收到但有损坏。 | 可能触发质检、保险或争议。 |
| Unable to confirm | 当前无法确认。 | 需要补充或人工处理。 |

买方提交收货确认时，可以上传：

| 材料 | 说明 |
|---|---|
| 签收记录 | 证明买方或指定收货方收货。 |
| 收货照片 | 证明货物状态。 |
| 损坏照片 | 证明损坏、短少或包装问题。 |
| 收货说明 | 说明收货数量、时间、地点。 |
| 异常说明 | 说明拒收、部分收货、质量争议等。 |

### 4.4 争议处理

买方可以提出 dispute，但不能用争议功能随意拖延或修改他人事实。

| 争议类型 | 说明 |
|---|---|
| 订单争议 | 买方不认可订单或订单条款。 |
| 金额争议 | 发票、订单或付款金额不一致。 |
| 数量争议 | 收货数量与订单或装箱单不一致。 |
| 质量争议 | 货物质量、包装、规格存在问题。 |
| 物流争议 | 延误、损坏、丢失、签收信息争议。 |
| 付款责任争议 | 买方不认可付款义务或到期时间。 |

处理方式：

| 处理 | 说明 |
|---|---|
| 提交争议说明 | 买方说明争议点。 |
| 上传支持材料 | 照片、签收记录、沟通记录、内部订单等。 |
| 请求出口商补充 | 需要出口商解释金额、数量、条款等。 |
| 请求质检或物流补充 | 质量、数量、运输争议需要事实方说明。 |
| 进入人工复核 | 重大争议交给资金方或平台处理。 |

### 4.5 付款责任确认

买方确认付款责任，是供应链金融回款判断的重要事实。

| 结论 | 含义 | 对融资案件的影响 |
|---|---|---|
| Payment obligation confirmed | 买方确认存在付款责任。 | 增强应收账款或回款可信度。 |
| Payment obligation pending | 付款责任待条件满足。 | 需要继续跟踪交付或验收。 |
| Payment obligation disputed | 买方对付款责任有争议。 | 进入人工复核或补充材料。 |
| No obligation | 买方不认可付款责任。 | 高风险异常，可能影响融资。 |

确认项：

| 确认项 | 说明 |
|---|---|
| 付款金额 | 买方认可的应付金额。 |
| 付款币种 | 使用固定币种选项。 |
| 到期时间 | 付款应到期日期或条件。 |
| 付款条件 | 收货、验收、发票、清关等条件。 |
| 是否存在抵扣或扣款 | 是否因质量、数量、延误等减少付款。 |

### 4.6 Agent 提醒和材料检查

Agent 帮买方处理确认事项，但不替买方承担付款责任。

| 检查项 | 输出结果 |
|---|---|
| 是否确认订单真实性 | 已确认 / 未确认 / 有争议 |
| 是否确认收货状态 | 已确认 / 未确认 / 有异常 |
| 是否确认付款责任 | 已确认 / 待条件 / 有争议 / 不认可 |
| 争议是否有说明 | 有 / 无 / 不需要 |
| 争议材料是否足够 | 足够 / 不足 / 不适用 |
| 是否需要相关方补充 | 不需要 / 需要出口商 / 需要物流方 / 需要质检方 |

Agent 给买方的表达必须是业务语言：

| 不要说 | 要说 |
|---|---|
| Confirmation required | 这笔订单需要你确认是否真实。 |
| Missing dispute reason | 你提出了争议，但还没有说明争议原因。 |
| Payment status invalid | 你选择了确认付款责任，但没有填写到期时间。 |
| Receipt mismatch | 你确认部分收货，需要说明缺少的数量或货物。 |

## 5. 买方 Agent 模式差异

| 功能节点 | 全人工确认 | 半自动托管 | 全自动 Agent |
|---|---|---|---|
| 接收确认请求 | 用户手动接受 | Agent 汇总待确认事项 | Agent 自动生成确认待办并提醒 |
| 订单真实性 | 用户手动确认 | Agent 比对材料并建议结论 | Agent 可生成草稿，最终确认仍建议用户确认 |
| 收货状态 | 用户手动确认 | Agent 根据物流和签收材料建议状态 | Agent 可整理状态草稿，异常必须人确认 |
| 付款责任 | 必须用户确认 | 用户确认关键条款 | 付款责任确认必须用户确认 |
| 争议提出 | 用户人工提交 | Agent 辅助整理争议点 | Agent 不能自动提出重大争议 |
| 补充材料 | 用户处理 | Agent 生成补充清单 | Agent 自动提醒和整理材料 |
| 到期提醒 | 用户主动查看 | Agent 推送提醒 | Agent 自动提醒关键到期事项 |

重要边界：

| 边界 | 说明 |
|---|---|
| Agent 可以提醒买方确认事实 | 可以生成待办和摘要。 |
| Agent 可以整理争议材料 | 可以归类照片、签收、订单差异。 |
| Agent 可以提醒付款到期 | 可以根据已确认条款提醒。 |
| Agent 不可以替买方承担付款责任 | 付款义务必须由买方确认。 |
| Agent 不可以自动裁决重大争议 | 争议必须交给人处理。 |

## 6. 权限边界

买方可以：

| 权限 | 说明 |
|---|---|
| 查看与自己相关的贸易事实 | 不能看无关融资案件。 |
| 确认订单真实性 | 确认贸易是否与自己有关。 |
| 确认收货状态 | 未收货、部分收货、已收货、拒收等。 |
| 提出 dispute | 对订单、金额、质量、数量、物流、付款责任提出争议。 |
| 确认付款责任 | 确认是否存在付款义务和到期条件。 |
| 上传支持材料 | 签收、照片、说明、争议材料等。 |

买方不可以：

| 禁止事项 | 原因 |
|---|---|
| 修改自己的永久角色 | 钱包角色不可变。 |
| 修改出口商融资申请 | 融资申请属于出口商。 |
| 替资金方审批 | 放款权属于资金方。 |
| 替质检方出结论 | 质检事实属于质检方。 |
| 修改物流事实 | 物流事实属于物流方。 |
| 查看无关融资案件 | 数据隔离。 |
| 删除关键确认记录 | 影响付款责任审计。 |

## 8. 本步结论

买方工作区的本质不是“客户门户”。

它的本质是：

> 买方作为付款责任确认方，确认贸易是否真实、是否收货、是否争议、是否承担付款责任；Agent 提醒和整理关键事实，但不替买方确认付款义务，也不替资金方审批。
