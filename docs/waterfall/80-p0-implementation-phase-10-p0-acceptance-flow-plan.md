# 80 P0 实现阶段 10：P0 Acceptance Flow 计划

## 1. 定位

本文件定义 ChainTrace P0 受控实现阶段的第十个阶段：P0 Acceptance Flow。

它不是自动化测试代码，不是 Playwright/Cypress/Jest/Vitest 脚本，不是 API 测试集合，不是 CI/CD 配置，不是正式 QA 测试报告，也不是验收截图模板。

它的作用是：

> 在 Phase 1–9 已经定义身份、案件、材料、质检、资金方、合约状态、Agent、控制塔和平台运营之后，形成 P0 最终手工业务验收流，确认 ChainTrace P0 是否真正完成“出口商融资意图到资金结果”的业务闭环。

本阶段的核心目标不是“写测试代码”，而是：

```text
正常融资闭环
→ 银行辅助路径
→ 纯 DeFi 路径
→ 低人工路径
→ 人工升级路径
→ 权限隔离
→ 事实来源边界
→ Agent 边界
→ 合约/规则边界
→ 平台运营边界
→ 后置范围确认
```

## 3. Phase 10 阶段目标

Phase 10 必须完成以下验收目标：

| 目标 | 说明 |
|---|---|
| 正常闭环可验收 | 出口商从融资意图走到资金结果。 |
| 银行辅助路径可验收 | 资金方审查、允许/拒绝/降额/人工复核成立。 |
| 纯 DeFi 路径可验收 | 没有资金方审批时，事实、规则、资金池、暂停共同决定结果。 |
| 低人工路径可验收 | Agent 在授权、低风险、事实完整、规则明确时有限推进。 |
| 人工升级路径可验收 | 高风险、Dispute、失败、置信不足、权限不清会升级。 |
| 权限隔离可验收 | 单钱包单角色、角色工作区、案件可见范围成立。 |
| 事实来源可验收 | 出口商材料、质检结论、资金方决策、合约输出和 Agent 输出边界清楚。 |
| 控制塔可验收 | 每个角色都知道当前状态、卡点、责任方和下一步。 |
| 平台运营可验收 | 资金池、规则、暂停、执行失败、Agent 自动化异常可处理。 |
| 后置范围可验收 | P1/P2、proof/hash、复杂后台和技术实现细节没有混入 P0。 |

## 5. P0 验收总闭环

P0 最小验收闭环如下：

```text
1. 出口商连接钱包并进入出口商工作区。
2. 出口商创建融资意图。
3. 系统形成融资案件。
4. 出口商补充融资目标和贸易基本信息。
5. 出口商提交合同、订单、发票、装箱单和补充说明。
6. Agent 检查材料缺口并生成待办。
7. 材料完整后进入质检事实节点。
8. 质检方提交质检材料并给出 Pass / Fail / Dispute。
9. 案件进入资金方审查或纯 DeFi 规则路径。
10. 资金方允许/拒绝/降额/人工复核，或 DeFi 路径执行规则检查。
11. 合约/规则/资金池/平台暂停状态检查执行条件。
12. Agent 解释状态、生成待办或人工升级。
13. Control Tower 展示状态、卡点、责任方、下一步和资金结果。
14. 平台处理资金池、规则、暂停、执行失败和 Agent 自动化异常。
15. 出口商看到最终资金结果：等待、允许、拒绝、降额、阻断、失败或完成。
```

## 6. 正常融资闭环验收

| 步骤 | 操作 | 期望结果 |
|---:|---|---|
| 1 | 出口商连接钱包。 | 系统识别为出口商角色。 |
| 2 | 出口商进入出口商工作区。 | 只能看到自己的工作区。 |
| 3 | 创建融资意图。 | 形成融资案件草稿。 |
| 4 | 补充融资目标和贸易基本信息。 | 案件进入材料准备状态。 |
| 5 | 提交合同、订单、发票、装箱单。 | 材料集合完整。 |
| 6 | Agent 检查缺口。 | 无阻断缺口或给出明确待办。 |
| 7 | 质检方接收任务并提交 Pass。 | 质检事实为 Pass。 |
| 8 | 资金方查看案件。 | 看到证据完整性和 Agent 风险建议。 |
| 9 | 资金方允许融资。 | 形成允许决策，但不显示已到账。 |
| 10 | 执行条件检查。 | 规则、资金池、平台暂停均通过。 |
| 11 | 资金执行完成。 | Funding Result 为 Completed。 |
| 12 | 出口商查看控制塔。 | 看到完整链路、资金结果和历史卡点。 |

通过标准：

```text
出口商从融资意图开始，最终能看到资金执行完成；过程中每一步状态、责任方和下一步都可理解。
```

## 7. 银行辅助路径验收

| 场景 | 操作 | 期望结果 |
|---|---|---|
| 资金方允许 | 资金方选择 Allow financing。 | 案件进入执行条件检查，不直接到账。 |
| 资金方拒绝 | 资金方选择 Reject financing。 | 出口商看到拒绝原因。 |
| 资金方降额 | 资金方选择 Reduce amount。 | 出口商看到调整金额和原因。 |
| 资金方要求补材料 | 资金方选择 Need materials。 | 出口商收到补材料待办。 |
| 资金方人工复核 | 资金方选择 Manual review。 | 自动推进停止，进入人工处理。 |
| 资金方越权修改材料 | 资金方尝试改出口商原始材料。 | 阻断。 |
| 资金方越权修改质检 | 资金方尝试修改 Pass/Fail/Dispute。 | 阻断。 |

通过标准：

```text
银行辅助路径中，资金方能审查和决策，但不能修改事实；允许融资仍需合约/规则/资金池检查。
```

## 8. 纯 DeFi 路径验收

| 场景 | 条件 | 期望结果 |
|---|---|---|
| 条件满足 | 材料完整、质检 Pass、规则 Active、资金池 Available、平台未暂停。 | Allowed / Completed。 |
| 材料缺口 | 缺合同、订单、发票或装箱单。 | Need materials。 |
| 质检 Dispute | Dispute 未解决。 | Need review / Dispute，自动推进停止。 |
| 规则不适用 | Rule not applicable。 | Rule blocked / Need review。 |
| 资金池不足 | Pool Low balance。 | Pool insufficient。 |
| 平台暂停 | Case/Pool/Rule/Execution paused。 | Platform paused。 |

通过标准：

```text
纯 DeFi 路径可以存在，但必须受到事实、规则、资金池、平台暂停和异常状态约束，不能无条件自动完成。
```

## 9. 低人工路径验收

| 场景 | 前提 | 期望结果 |
|---|---|---|
| Agent 生成摘要 | 角色有权查看案件。 | 摘要可见且标识为 Agent 输出。 |
| Agent 生成缺口 | 材料缺失或冲突。 | 生成责任方和下一步。 |
| Agent 生成待办 | 缺口明确。 | 待办分配给正确角色。 |
| Agent 请求补材料 | Semi-auto/Full-auto 且责任方明确。 | 生成补材料请求，不伪造材料。 |
| Agent 请求低风险推进 | 授权、低风险、事实完整、规则明确、未暂停。 | 可以推进或请求执行检查。 |
| Agent 遇到 Rule blocked | 规则阻断。 | 停止推进并解释。 |
| Agent 遇到 Platform paused | 平台暂停。 | 停止自动化。 |

通过标准：

```text
低人工路径是“受控 Agent 辅助推进”，不是 Agent 超级审批或无限自动化。
```

## 10. 人工升级路径验收

| 升级原因 | 期望升级对象 | 期望结果 |
|---|---|---|
| 高金额或高风险 | 资金方。 | 自动推进停止，人工复核。 |
| 材料金额冲突 | 出口商 / 资金方。 | 提交说明或人工复核。 |
| 质检 Fail | 资金方 / 质检方。 | 进入复核、拒绝或补充路径。 |
| 质检 Dispute | 出口商 / 质检方 / 资金方。 | 自动推进停止，争议处理。 |
| 规则冲突 | 平台。 | Need review。 |
| 资金池不足 | 平台 / 资金方。 | Pool insufficient 处理。 |
| 平台暂停 | 平台。 | 等待恢复。 |
| 执行失败 | 平台 / 执行责任方。 | Execution failed 处理。 |
| Agent 置信不足 | 所属角色。 | 人工确认。 |
| 权限不清 | 回流 waterfall。 | 不用代码临时绕过。 |

通过标准：

```text
所有高风险、争议、失败和权限不清都能停止自动化，并清楚说明谁来处理。
```

## 11. 权限隔离验收

| 场景 | 期望结果 |
|---|---|
| 一个钱包注册多个角色 | 阻断。 |
| 已注册钱包切换业务角色 | 阻断。 |
| 未注册钱包进入工作区 | 阻断。 |
| 出口商进入资金方工作区 | 阻断。 |
| 出口商查看其他出口商案件 | 阻断。 |
| 资金方查看无关案件 | 阻断。 |
| 质检方查看无关任务 | 阻断。 |
| Agent 查看无权案件 | 阻断。 |
| 账户暂停后继续操作 | 阻断。 |
| 越权阻断只显示技术错误 | 不通过。 |

通过标准：

```text
权限隔离不仅要阻断，还要用业务语言解释原因。
```

## 12. 事实来源边界验收

| 事实/输出 | 合法来源 | 非法来源 |
|---|---|---|
| 出口商贸易材料 | 出口商。 | 资金方、质检方、平台、Agent。 |
| 补充说明 | 对应责任方。 | 无权角色。 |
| 质检 Pass/Fail/Dispute | 质检方。 | Agent、资金方、平台、合约。 |
| 资金方决策 | 资金方。 | Agent、平台、合约。 |
| 合约/规则输出 | 合约/规则状态表达层。 | Agent、资金方手动伪造。 |
| Agent 摘要/建议/待办 | Agent。 | 不得伪装成事实。 |
| 平台暂停/恢复 | 平台。 | 无权角色。 |

通过标准：

```text
每类事实或输出都有明确来源，不能由其他角色伪造或无痕覆盖。
```

| Agent 行为 | 期望结果 |
|---|---|
| 摘要案件 | 允许，标识为 Agent 输出。 |
| 检查材料缺口 | 允许，输出责任方和下一步。 |
| 生成补材料待办 | 允许。 |
| 请求补材料 | 授权范围内允许。 |
| 解释状态 | 允许。 |
| 低风险推进 | 条件满足时允许。 |
| 查看无权案件 | 阻断。 |
| 伪造材料 | 阻断。 |
| 生成质检结论 | 阻断。 |
| 替资金方审批 | 阻断。 |
| 裁决 Dispute | 阻断。 |
| 绕过 Rule blocked | 阻断。 |
| 绕过 Platform paused | 阻断。 |

通过标准：

```text
Agent 是角色助理，不是超级用户、事实来源、资金方审批者或争议裁决者。
```

## 14. 合约/规则边界验收

| 场景 | 期望结果 |
|---|---|
| 合约读取材料完整性 | 可以作为条件。 |
| 合约读取质检状态 | 可以作为条件。 |
| 合约读取资金方允许 | 可以作为条件。 |
| 合约读取资金池状态 | 可以作为条件。 |
| 合约读取平台暂停 | 可以作为条件。 |
| 合约判断贸易真实性 | 不允许。 |
| 合约生成质检结论 | 不允许。 |
| 合约替资金方审批 | 不允许。 |
| 合约裁决 Dispute | 不允许。 |
| 输出只有 success/fail | 不通过。 |

通过标准：

```text
合约/规则是执行约束和状态输出层，不是事实判断者、审批者或争议裁决者。
```

| 平台行为 | 期望结果 |
|---|---|
| 查看异常队列 | 允许。 |
| 查看资金池状态 | 允许。 |
| 查看规则状态 | 允许。 |

## 16. 控制塔验收

| 控制塔要求 | 通过标准 |
|---|---|
| 主链路完整 | 从融资意图到资金结果可见。 |
| 角色视角清楚 | 出口商、资金方、质检方、平台方视图不同。 |
| 卡点可处理 | 原因、责任方、影响、解除条件、下一步清楚。 |
| Agent 输出标识 | Agent 输出不伪装成事实或审批。 |
| 合约状态解释 | Allowed、Waiting、Rule blocked、Pool insufficient 等可理解。 |
| 资金结果清楚 | 出口商能理解等待、允许、拒绝、降额、失败或完成。 |
| 路径差异保留 | 银行辅助、纯 DeFi、低人工、人工升级可区分。 |
| 越权视图阻断 | 无权内容不可见。 |

不通过情况：

- 控制塔变普通列表。
- 只显示技术 error。
- 出口商看不到资金结果。
- 所有角色看到同一工作区。
- Agent 输出和事实混在一起。

## 18. P0 最终通过标准

P0 Acceptance Flow 最终通过，必须同时满足：

```text
1. 单钱包单角色成立。
2. 角色工作区和可见范围成立。
3. 出口商可以创建融资案件。
4. 材料和缺口流程成立。
5. 质检事实节点成立。
6. 资金方审查路径成立。
7. 纯 DeFi 路径受事实、规则、资金池和暂停约束。
8. 合约/规则状态可解释。
9. Agent 能辅助但不越权。
10. 控制塔能表达状态、卡点、责任方、下一步和资金结果。
11. 平台能处理基础运营异常但不替审批和不伪造事实。
12. 正常闭环、异常闭环、权限隔离和后置范围都能被手工验收。
```

## 19. P0 一票否决总表

| 一票否决项 | 原因 |
|---|---|
| 一个钱包多个角色自由切换 | 权限根基失败。 |
| 出口商无法看到资金结果 | P0 核心价值失败。 |
| 案件只是文件夹，没有融资意图和资金结果 | 产品主线失败。 |
| 资金方可以修改原始材料或质检结论 | 事实来源失败。 |
| Agent 可以伪造事实、审批或裁决争议 | Agent 边界失败。 |
| 合约判断贸易真实性或替资金方审批 | 合约边界失败。 |
| 平台替资金方批准或伪造事实 | 平台边界失败。 |
| 纯 DeFi 无事实/规则/资金池/暂停约束 | 风险边界失败。 |
| 异常只显示 error，无责任方和下一步 | 控制塔失败。 |
| 路径差异消失 | 产品定位失败。 |
| P1/P2 或 proof/hash 旧主线混入 P0 验收 | 范围失控。 |

## 23. 本步结论

Phase 10 的本质不是“写测试脚本”。
