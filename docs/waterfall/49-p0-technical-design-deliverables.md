# 49 P0 技术设计交付物清单

## 1. 定位

本文件定义 ChainTrace P0 后续技术设计阶段最终应交付哪些业务对齐材料、边界记录和回流记录。

P0 技术设计交付物清单不是接口文档，不是数据库模型，不是智能合约规格，不是前端实现说明，也不是测试代码清单。

它的作用是：

> 在技术设计阶段结束时，用统一交付物证明后续技术设计已经继承 P0 业务冻结口径，并且没有越过接口、数据、合约、Agent、权限、原型实现和测试设计的阶段边界。

本文件不写组件、样式、布局尺寸、图表库、页面路由、接口路径、请求响应格式、数据库、Solidity、ABI、钱包签名、hash 或 proof 细节。

## 2. 交付原则

| 原则 | 说明 |
|---|---|
| 交付业务对齐证据 | 技术设计交付物必须证明继承了冻结业务口径。 |
| 交付边界记录 | 每个设计域必须说明能做什么、不能做什么。 |
| 交付回流记录 | 发现需求缺口必须有 waterfall 回流记录。 |
| 交付后置清单 | P1/P2 和实现阶段内容必须后置。 |
| 不交付实现规格 | 当前不交付 endpoint、表结构、Solidity、组件或代码。 |
| 不改变产品定位 | 技术设计不能把系统改回前站或单纯银行审批系统。 |
| 不改变路径差异 | 银行辅助、纯 DeFi、低人工和人工升级必须保留。 |
| 不改变 Agent 和 DeFi 边界 | Agent 不越权，合约不判断贸易真实性。 |

## 3. 技术设计交付包总目录

| 交付物 | 必须包含 | 交付目的 |
|---|---|---|
| 交付包封面 | 版本、范围、日期、责任人、状态。 | 明确交付对象。 |
| 业务输入映射 | waterfall 输入文档和对应设计域。 | 证明技术设计来自已冻结需求。 |
| 设计域边界说明 | 接口、数据、合约、Agent、权限、原型、测试边界。 | 防止设计域越界。 |
| 业务输出对齐表 | 每个设计域输出哪些业务结果。 | 保持业务可解释。 |
| 权限边界记录 | 角色可见范围、可做事项和越权阻断。 | 保持单钱包单角色。 |
| 路径差异记录 | 银行辅助、纯 DeFi、低人工、人工升级如何被保留。 | 防止路径合并。 |
| Agent 边界记录 | Agent 可做、不可做、自动推进和人工升级。 | 防止 Agent 越权。 |
| DeFi 边界记录 | 合约业务输出、执行条件、暂停、资金池、失败和完成。 | 防止合约越界。 |
| 异常回流记录 | 主要异常的原因、责任方、解除条件和下一步。 | 证明异常可处理。 |
| 技术缺口回流记录 | 发现的问题、回流文档、处理状态。 | 防止直接绕过需求。 |
| 后置清单 | P1/P2、正式实现和证明产品后置项。 | 控制范围。 |
| 交付结论 | 是否允许进入正式技术规格或实现准备。 | 明确下一阶段。 |

## 4. 业务输入映射交付物

| 输入来源 | 技术设计应映射到 | 必须证明 |
|---|---|---|
| P0 PRD 总纲 | 全部设计域。 | 产品定位未被改变。 |
| 场景路径矩阵 | 接口、合约、Agent、测试。 | 路径差异保留。 |
| 权限矩阵 | 接口、数据、Agent、权限、测试。 | 单钱包单角色和越权阻断保留。 |
| 业务动作清单 | 接口、测试。 | 每个动作有责任角色和业务结果。 |
| 业务事件清单 | 接口、数据、测试。 | 每个事件有业务含义。 |
| 业务规则清单 | 接口、数据、合约、Agent、测试。 | 状态、阻断、异常和升级规则保留。 |
| DeFi 规则清单 | 合约、接口、数据、测试。 | 合约业务输出保留。 |
| 控制塔业务要求 | 原型、数据、测试。 | 源头到终端和卡点表达保留。 |
| 视觉原型交付包 | 原型、测试、产品细化。 | 业务表达和评审结论保留。 |
| 技术设计边界说明 | 全部设计域。 | 没有越界到实现。 |

## 5. 接口设计交付物

接口设计交付物只能交付业务对齐材料，不交付接口实现规格。

| 交付项 | 应包含 | 不得包含 |
|---|---|---|
| 业务动作分组 | 创建意图、补材料、确认事实、授权、执行检查、暂停、恢复、异常升级。 | endpoint 名称。 |
| 业务结果清单 | 成功、等待、需补材料、规则阻断、权限不足、进入人工、执行失败。 | 错误码格式。 |
| 角色动作边界 | 每个角色可以触发的业务动作。 | 鉴权实现。 |
| 状态解释 | 状态对用户的业务含义。 | 状态字段结构。 |
| 失败解释 | 失败原因、责任方和下一步。 | 响应体结构。 |
| 回流记录 | 接口设计发现的业务缺口和回流文档。 | 直接补接口实现。 |

## 6. 数据设计交付物

数据设计交付物只能交付业务信息域和记录责任，不交付数据库实现。

| 交付项 | 应包含 | 不得包含 |
|---|---|---|
| 信息域清单 | 案件、贸易材料、事实节点、资金方决策、Agent 输出、DeFi 结果、异常、审计。 | 表结构。 |
| 事实来源记录 | 谁确认事实、谁负责事实。 | 数据库关系模型。 |
| 关键记录要求 | 授权、阻断、暂停、恢复、执行结果可追踪。 | 字段和索引。 |
| 可见范围说明 | 角色相关可见、无关案件不可见。 | 权限表结构。 |
| Agent 记录边界 | Agent 摘要不等于事实结论。 | Agent memory 实现。 |
| DeFi 记录边界 | 合约结果不等于贸易真实性判断。 | 链上/链下存储结构。 |

## 7. 合约设计交付物

合约设计交付物只能交付业务约束和业务输出，不交付合约实现规格。

| 交付项 | 应包含 | 不得包含 |
|---|---|---|
| 合约业务角色说明 | 合约是业务规则和资金执行约束层。 | 合约代码。 |
| 执行前条件说明 | 授权、事实、规则、资金池、暂停、保证条件。 | 函数签名。 |
| 业务输出清单 | 允许、不允许、等待、规则阻断、资金池不足、平台暂停、失败、完成。 | ABI。 |
| 暂停边界说明 | 平台、路径、规则、资金池或案件暂停的业务含义。 | 权限实现。 |
| 资金池状态说明 | 可用、不足、暂停、币种不支持。 | 资金池合约结构。 |
| 不负责事项记录 | 不判断贸易真实性、不替资金方审批、不裁决重大争议。 | 技术绕过方案。 |

## 8. Agent 设计交付物

Agent 设计交付物只能交付业务模式和边界，不交付模型或工具实现。

| 交付项 | 应包含 | 不得包含 |
|---|---|---|
| Agent 模式说明 | 全人工确认、半自动托管、全自动 Agent。 | 模型选择。 |
| 可做事项清单 | 整理材料、摘要、缺口、待办、提醒、补材料请求、低风险推进、异常升级。 | 工具调用实现。 |
| 不可做事项清单 | 不伪造事实、不越权、不绕过规则、不高风险放款、不裁决重大争议。 | 自动裁决实现。 |
| 自动推进条件 | 授权范围、低风险、事实完整、规则明确。 | 阈值和评分算法。 |
| 人工升级条件 | 争议、高风险、越权、置信不足、执行失败、规则冲突。 | 提示词模板。 |
| 权限继承记录 | Agent 权限继承所属角色。 | 独立 Agent 账号体系。 |

## 9. 权限设计交付物

| 交付项 | 应包含 | 不得包含 |
|---|---|---|
| 角色可见范围 | 出口商、资金方、事实方、平台方、Agent 各自可见范围。 | 鉴权中间件实现。 |
| 角色动作边界 | 每个角色能做和不能做的业务动作。 | 权限表结构。 |
| 越权场景清单 | 无关案件、事实篡改、越权审批、Agent 越权。 | token 或 session 设计。 |
| 越权业务解释 | 被阻断时用户应理解的业务原因。 | 错误码格式。 |
| 平台运营边界 | 规则、资金池、暂停、执行失败和 Agent 堆积。 | 管理后台实现。 |

## 10. 原型实现准备交付物

原型实现准备交付物只能交付业务表达材料，不交付页面或组件实现。

| 交付项 | 应包含 | 不得包含 |
|---|---|---|
| 控制塔表达说明 | 源头到终端、节点状态、卡点、资金结果。 | 页面路由。 |
| 角色视角说明 | 不同角色看到不同范围。 | UI 组件结构。 |
| 路径表达说明 | 银行辅助、纯 DeFi、低人工、人工升级。 | 前端状态实现。 |
| Agent 表达说明 | 摘要、缺口、待办、建议、升级。 | Agent UI 组件。 |
| DeFi 表达说明 | 允许、等待、阻断、资金池不足、暂停、失败、完成。 | 合约调用实现。 |
| 异常表达说明 | 原因、责任方、解除条件、下一步。 | 交互代码。 |
| 后置范围说明 | P1/P2、实现细节、证明产品后置。 | 旧页面迁移。 |

## 11. 测试设计交付物

测试设计交付物只能交付业务验收口径，不交付测试代码。

| 交付项 | 应包含 | 不得包含 |
|---|---|---|
| 正常闭环验收 | 出口商意图、Agent 整理、事实确认、资金方/规则、DeFi 执行、资金结果。 | 测试框架。 |
| 银行辅助验收 | 资金方审查、补材料、授权、合约检查。 | API 测试代码。 |
| 纯 DeFi 验收 | 无银行审批、规则和资金池驱动执行。 | 链上测试脚本。 |
| 低人工验收 | Agent 推进、合约执行、异常升级。 | 自动化工具。 |
| 异常验收 | dispute、规则阻断、资金池不足、平台暂停、执行失败。 | 断言格式。 |
| 权限验收 | 无关案件不可见、越权动作阻断。 | 鉴权测试实现。 |
| 后置范围验收 | P1/P2 和实现细节不进入 P0。 | 发布流程。 |

## 12. 回流记录交付物

| 回流项 | 必须记录 | 不得处理方式 |
|---|---|---|
| 新角色 | 发现来源、影响范围、回流文档、处理状态。 | 直接新增角色实现。 |
| 新路径 | 场景、触发条件、影响角色、回流文档。 | 直接改流程。 |
| 新状态 | 状态含义、用户解释、影响节点。 | 直接新增状态字段。 |
| 新异常 | 责任方、解除条件、下一步。 | 直接新增错误码。 |
| 新合约输出 | 业务含义、阻断或执行影响。 | 直接新增函数。 |
| Agent 权限调整 | 权限影响、风险、回流文档。 | 直接放开自动化。 |
| 数据记录不足 | 记录责任、审计要求、可见范围。 | 直接新增表字段。 |
| P1/P2 提前 | 提前原因、范围影响、决策结论。 | 直接塞入 P0。 |

| 后置项 | 后置阶段 | P0 交付状态 |
|---|---|---|
| 技术架构选型细节 | 后续正式技术设计 | P0 只记录边界。 |
| API endpoint 和请求响应 | 接口规格阶段 | P0 只记录业务动作和业务结果。 |
| 数据库表结构 | 数据模型阶段 | P0 只记录信息域和记录责任。 |
| Solidity / ABI / 部署 | 合约实现阶段 | P0 只记录业务输出和约束。 |

| 测试代码和自动化 | 测试实现阶段 | P0 只记录业务验收口径。 |

## 14. 技术设计交付通过标准

| 通过项 | 通过标准 |
|---|---|
| 业务输入映射完整 | 每个冻结业务输入都有对应设计域接收。 |
| 设计域边界清楚 | 每个设计域知道可做和不可做事项。 |
| 角色权限未被扩大 | 单钱包单角色和案件相关可见被保留。 |
| 路径差异未被合并 | 银行辅助、纯 DeFi、低人工和人工升级被保留。 |
| Agent 边界清楚 | Agent 可做、不可做和升级条件清楚。 |
| DeFi 边界清楚 | 合约业务输出和不负责事项清楚。 |
| 异常可解释 | 异常有原因、责任方、解除条件和下一步。 |
| 回流记录完整 | 需求缺口有回流文档和处理状态。 |
| 后置清单完整 | 实现细节和 P1/P2 没有进入 P0。 |

## 15. 不得作为交付完成的情况

| 不得通过 | 原因 |
|---|---|
| 只交付技术分层，没有业务输入映射 | 无法证明继承 waterfall。 |
| 技术设计改变产品定位 | 破坏业务冻结。 |
| 所有路径被合并成银行审批 | 破坏 DeFi 和低人工场景。 |
| Agent 被设计成审批人 | 破坏 Agent Native 边界。 |
| 合约被设计成事实判断者 | 破坏事实来源边界。 |
| 角色权限被技术设计放大 | 破坏权限隔离。 |
| 异常只有技术错误 | 业务不可处理。 |
| 直接出现 endpoint、表结构、Solidity、ABI、组件或代码 | 阶段越界。 |
| 旧版页面或代码口径回流 | 违背全新瀑布重建设计分支。 |

## 18. 本步结论

P0 技术设计交付物清单的本质不是“交付技术实现方案”。

它的本质是：

> 用可检查的业务对齐材料证明技术设计阶段没有偏离 ChainTrace P0 的产品定位、控制塔主线、角色权限、路径差异、Agent 边界、DeFi 执行、异常回流和后置范围，并且没有提前进入具体实现规格。

下一步应进入：

> P0 技术设计评审清单，用于评审技术设计交付物是否继承业务冻结口径、是否越界、是否需要回流 waterfall 文档。
