# 33 P0 业务规则清单

## 1. 定位

本文件定义 ChainTrace P0 阶段的业务规则清单。

业务规则清单不是规则引擎设计，不是 API 设计，不是数据库表设计，也不是智能合约函数清单。

它的作用是：

> 用业务语言说明哪些规则会影响路径选择、材料完整性、事实确认、Agent 自动推进、银行辅助决策、DeFi 资金执行、阻断和人工升级。

本文件不写页面、组件、API、数据库、Solidity、钱包签名、hash 或 proof 细节。

## 2. 规则设计原则

| 原则 | 说明 |
|---|---|
| 规则服务业务路径 | 规则首先决定案件走银行辅助、纯 DeFi、低人工还是人工升级。 |
| 规则必须可解释 | 每条规则命中后必须能说明原因、影响和下一步。 |
| 规则不伪造事实 | 规则只能使用已有材料、事实和授权结果。 |
| 规则不替代人类边界 | 高风险、争议、监管和重大法律责任仍需人工处理。 |
| 规则不等于合约代码 | 当前只定义业务口径，不定义 Solidity 或技术实现。 |
| 规则可被平台暂停 | 平台方可以暂停、恢复或调整规则适用范围。 |
| 规则变更必须可追踪 | 规则启用、暂停、恢复和调整必须有原因。 |

## 3. 规则分类

| 规则类型 | 作用 |
|---|---|
| 路径选择规则 | 判断案件进入银行辅助、纯 DeFi、低人工或人工升级路径。 |
| 材料完整性规则 | 判断贸易材料是否满足最低要求。 |
| 事实确认规则 | 判断质检、物流、通关、保险、买方等事实是否满足要求。 |
| Agent 自动化规则 | 判断 Agent 能否自动整理、提醒、推进或必须升级人工。 |
| 银行辅助规则 | 判断是否需要资金方审查、授权、补材料或人工复核。 |
| DeFi 执行规则 | 判断是否允许合约执行、等待、阻断或失败处理。 |
| 风险和阻断规则 | 判断是否因为风险、争议、规则、资金池或平台状态阻断。 |
| 人工升级规则 | 判断哪些情况必须交给人处理。 |
| 平台运营规则 | 判断平台何时暂停、恢复、处理异常或调整规则。 |

## 4. 路径选择规则

| 规则 | 命中条件 | 业务结果 |
|---|---|---|
| 需要银行授信 | 金额高、风险高、机构资金参与或业务规则要求银行授权。 | 进入银行辅助路径。 |
| 银行不是必需 | 场景规则明确、资金池可约束、事实足够、风险边界清楚。 | 可进入纯 DeFi 路径。 |
| 低人工可执行 | 规则明确、授权边界明确、事实结构化、无重大争议。 | 可进入 Agent + 智能合约低人工路径。 |
| 存在重大争议 | 买方争议、质检 dispute、事实冲突、法律责任不清。 | 进入人工升级路径。 |
| 平台暂停命中 | 案件、规则、资金池或路径处于暂停。 | 进入等待或阻断。 |
| Agent 置信不足 | Agent 无法可靠判断材料或事实是否满足规则。 | 降级为人工确认。 |

路径选择必须解释：

```text
为什么进入这条路径？
是否需要银行？
是否允许纯 DeFi 执行？
是否可以低人工自动推进？
如果不能继续，责任角色是谁？
```

## 5. 材料完整性规则

| 规则 | 命中条件 | 业务结果 |
|---|---|---|
| 基础贸易材料缺失 | 合同、订单、发票、装箱单或贸易说明缺失。 | 进入 Need materials。 |
| 金额信息缺失 | 融资金额、币种、合同金额或发票金额缺失。 | 生成补材料待办。 |
| 主体信息缺失 | 出口商、买方、付款方或相关主体信息不完整。 | 生成补材料待办。 |
| 材料格式不足 | 材料无法支撑业务理解，或关键字段不可读。 | Agent 请求补充清晰材料。 |
| 材料冲突 | 合同、订单、发票、装箱单金额、品类、数量或主体不一致。 | 进入人工复核或补说明。 |
| 补材料完成 | 责任角色提交补充材料或说明。 | 回到 Material checking、Fact confirming 或 Under review。 |

材料规则不得：

| 不得 | 原因 |
|---|---|
| 用 Agent 生成内容替代原始材料 | Agent 只能整理，不是事实来源。 |
| 覆盖原始材料 | 补充和修正必须形成新记录。 |
| 让无关角色补贸易材料 | 贸易材料由出口商负责。 |

## 6. 事实确认规则

| 规则 | 命中条件 | 业务结果 |
|---|---|---|
| 需要质检事实 | 商品、金额、风险或规则要求独立货物事实。 | 创建质检任务。 |
| 质检 pass | 质检方提交通过结论。 | 可进入银行审查或 DeFi 检查。 |
| 质检 fail | 质检方提交不通过结论。 | 进入人工复核、拒绝或阻断。 |
| 质检 dispute | 质检方提交争议结论。 | 进入人工升级。 |
| 需要物流事实 | 路径或风险要求确认揽收、出港、到港、签收或异常。 | 创建物流事实待办。 |
| 物流异常 | 延误、损坏、丢失、改港或签收冲突。 | 进入人工复核或补说明。 |
| 需要买方确认 | 需要订单、收货、争议或付款责任事实。 | 创建买方确认待办。 |
| 买方提出争议 | 买方否认订单、收货或付款责任。 | 暂停自动执行并人工升级。 |
| 需要保险结论 | 风险缓释规则要求承保、加费、拒保或补材料。 | 创建保险核保待办。 |
| 保险拒保 | 保险公司拒保或要求人工核保。 | 风险升级或进入人工复核。 |
| 需要通关事实 | 国家、品类、金额或监管规则要求通关确认。 | 创建海关/报关事实待办。 |
| 通关异常 | 报关金额、品类、国家、贸易主体不一致或监管异常。 | 进入人工复核。 |

## 7. Agent 自动化规则

| 规则 | 命中条件 | 业务结果 |
|---|---|---|
| 全人工确认模式 | 用户选择全人工确认。 | Agent 只整理、提醒、建议，不自动提交或推进关键动作。 |
| 半自动托管模式 | 用户选择半自动托管。 | Agent 自动整理、检查、生成待办，低风险步骤可推进。 |
| 全自动 Agent 模式 | 用户选择全自动且授权范围明确。 | 低风险、规则明确、事实完整时自动推进。 |
| 自动推进允许 | 材料完整、事实满足、风险低、规则未暂停、授权范围内。 | Agent 推进到下一状态或 DeFi checking。 |
| 自动推进禁止 | 高风险、争议、事实冲突、规则例外、授权不清。 | 进入人工确认或 Manual review。 |
| Agent 置信不足 | 材料不清、事实冲突、规则无法确定。 | 降级为人工确认。 |
| Agent 越权风险 | Agent 尝试查看不可见案件、伪造事实或绕过规则。 | 阻断并升级平台或所属角色。 |

Agent 规则必须符合：

```text
Agent 可以整理、检查、提醒、生成待办、推进低风险流程；
Agent 不可以伪造事实、裁决重大争议、替资金方高风险放款、替平台绕过规则。
```

## 8. 银行辅助规则

银行辅助规则只适用于需要银行、保理机构或资金方参与的场景。

| 规则 | 命中条件 | 业务结果 |
|---|---|---|
| 进入资金方审查 | 路径选择结果为银行辅助，且最低材料和必要事实满足。 | 进入 Under review。 |
| 资金方要求补材料 | 资金方认为材料或事实不足。 | 生成补材料待办。 |
| 资金方人工复核 | 金额高、风险高、材料冲突、事实争议或规则警示。 | 进入 Manual review。 |
| 资金方允许放款 | 资金方审查通过并给出授权。 | 进入 Approved 或 Execution pending。 |
| 资金方拒绝融资 | 资金方不接受风险或材料无法满足要求。 | 进入 Rejected 或 Closed。 |
| 授权过期 | 资金方授权超过有效期或条件变化。 | 重新审查或阻断。 |
| 授权金额不足 | 执行金额超过资金方授权范围。 | 阻断或重新授权。 |

银行辅助规则不得：

| 不得 | 原因 |
|---|---|
| 让合约绕过资金方授权 | 银行辅助路径中资金方授权是前置条件。 |
| 让平台替资金方审批 | 平台不负责放款决策。 |
| 让 Agent 替资金方做高风险放款 | Agent 只能建议和整理。 |

## 9. 纯 DeFi 执行规则

| 规则 | 命中条件 | 业务结果 |
|---|---|---|
| 纯 DeFi 准入满足 | 场景不要求银行、规则明确、事实满足、授权清楚。 | 进入 DeFi checking。 |
| 资金池可用 | 资金池状态正常且金额足够。 | 可继续执行。 |
| 资金池不足 | 可用资金不足或占用超限。 | 等待、阻断或平台处理。 |
| 抵押/保证满足 | 纯 DeFi 场景要求的抵押或保证条件满足。 | 可继续执行。 |
| 抵押/保证不足 | 抵押、保证、覆盖比例或授权不满足。 | 等待条件或阻断。 |
| 规则版本有效 | 当前规则适用于该场景且未暂停。 | 可继续执行。 |
| 规则版本无效 | 规则过期、不匹配或暂停。 | 规则阻断。 |
| 平台未暂停 | 案件、路径、规则和资金池均未暂停。 | 可继续执行。 |
| 条件全部满足 | 规则、资金池、授权、抵押/保证、事实状态均满足。 | 允许执行。 |
| 条件不满足 | 任一必要条件不满足。 | 不允许执行或等待条件。 |

纯 DeFi 规则必须保证：

| 要求 | 说明 |
|---|---|
| 无银行场景不等银行 | 不需要银行审批时不强制进入资金方审查。 |
| 执行前检查完整 | 规则、资金池、授权、抵押/保证、暂停状态必须检查。 |
| 阻断有业务原因 | 用户能看到阻断类型、责任角色和解除条件。 |
| 事实缺失不执行 | 事实条件不满足时不能执行资金动作。 |

## 10. 低人工法律/运营规则

| 规则 | 命中条件 | 业务结果 |
|---|---|---|
| 保证金释放条件满足 | 约定条件达成、事实状态满足、无争议。 | 合约释放保证金。 |
| 保证金释放条件不足 | 条件未达成或事实缺失。 | 等待条件或阻断。 |
| 付款责任到期 | 到期日到达、付款责任确认、无争议。 | Agent 提醒并可进入结算。 |
| 付款责任争议 | 买方否认、付款责任不清或收货争议。 | 暂停自动执行并人工升级。 |
| 资金托管释放条件满足 | 条款条件明确且事实满足。 | 合约释放或结算。 |
| 重大法律责任争议 | 法律责任、赔偿、违约或仲裁事项不清。 | 不自动裁决，进入人工或约定仲裁。 |

低人工规则必须保持：

```text
正常低风险事项可由 Agent + 合约推进；
重大争议、高风险责任和法律判断不得自动裁决。
```

## 11. 风险和阻断规则

| 规则 | 命中条件 | 业务结果 |
|---|---|---|
| 主体不一致 | 合同、订单、发票、买方、付款方主体冲突。 | 人工复核或补说明。 |
| 金额不一致 | 合同、发票、融资金额、授权金额明显冲突。 | 人工复核或阻断。 |
| 品类不一致 | 贸易品类、质检品类、报关品类冲突。 | 人工复核。 |
| 国家或通关异常 | 国家、监管状态、报关/清关事实异常。 | 人工复核或阻断。 |
| 质检不通过 | 质检 fail。 | 阻断、拒绝或人工复核。 |
| 买方争议 | 买方提出 dispute。 | 暂停自动执行。 |
| 保险拒保 | 风险缓释条件不满足。 | 风险升级或人工复核。 |
| 物流严重异常 | 损坏、丢失、改港、重大延误。 | 人工复核或阻断。 |
| 平台暂停 | 平台暂停案件、路径、规则或资金池。 | 阻断或等待。 |
| 执行失败 | 合约或资金动作失败。 | 平台处理并解释。 |

## 12. 人工升级规则

以下规则命中时，必须升级人工：

| 规则 | 升级对象 | 业务要求 |
|---|---|---|
| 高金额融资 | 资金方 / 平台方 | 人工确认授权和风险。 |
| 高风险国家、主体或品类 | 资金方 / 平台方 | 人工复核。 |
| 质检 dispute | 质检方 / 资金方 / 平台方 | 争议处理。 |
| 买方 dispute | 买方 / 出口商 / 资金方 | 暂停自动执行并处理争议。 |
| 事实冲突 | 相关事实方 / 资金方 / 平台方 | 解释冲突来源。 |
| 规则例外 | 平台方 | 不能由 Agent 自行绕过。 |
| 资金池异常 | 平台方 | 处理资金池状态。 |
| 重大法律责任争议 | 人工角色 / 约定仲裁 | 不允许自动裁决。 |
| Agent 越权风险 | 所属角色 / 平台方 | 降级模式或阻断。 |

## 13. 平台运营规则

| 规则 | 命中条件 | 业务结果 |
|---|---|---|
| 暂停规则 | 规则存在风险、过期、错误或需要人工调整。 | 相关案件等待或阻断。 |
| 恢复规则 | 暂停原因解除。 | 相关案件可重新检查。 |
| 暂停资金池 | 资金池不足、异常、占用超限或运营风险。 | 资金动作等待或阻断。 |
| 恢复资金池 | 资金池状态恢复。 | 可重新进入执行检查。 |
| 暂停案件 | 单个案件存在争议、高风险或执行异常。 | 案件进入 Blocked 或 Manual review。 |
| 恢复案件 | 暂停原因解除。 | 回到原流程或重新检查。 |
| 调整 Agent 模式 | Agent 堆积、异常或越权风险。 | 降级或限制自动化。 |

平台规则必须说明：

```text
暂停什么？
为什么暂停？
影响哪些案件或路径？
谁可以恢复？
恢复后回到哪个状态？
```

## 14. 规则输出要求

每条业务规则命中后至少输出：

| 输出 | 说明 |
|---|---|
| 规则结果 | 通过、等待、阻断、升级人工、允许执行或不允许执行。 |
| 命中原因 | 材料、事实、金额、风险、授权、资金池、平台或争议原因。 |
| 影响状态 | 案件进入哪个业务状态。 |
| 责任角色 | 谁需要处理下一步。 |
| 解除条件 | 如何从等待或阻断回到流程。 |
| 是否产生事件 | 是否形成业务事件和审计记录。 |

## 15. MVP 范围

### 15.1 P0 必须定义

| 范围 | 说明 |
|---|---|
| 路径选择规则 | 判断银行辅助、纯 DeFi、低人工、人工升级。 |
| 材料完整性规则 | 判断缺材料、冲突、补材料和回流。 |
| 事实确认规则 | 判断质检、物流、保险、买方、通关事实状态。 |
| Agent 自动化规则 | 判断自动推进、降级、升级和越权风险。 |
| 银行辅助规则 | 判断资金方审查、授权、拒绝和补材料。 |
| DeFi 执行规则 | 判断规则、资金池、授权、抵押/保证、暂停和执行。 |
| 风险阻断规则 | 判断风险、争议、异常和平台阻断。 |
| 人工升级规则 | 明确哪些情况必须交给人。 |

### 15.2 暂不做

| 范围 | 原因 |
|---|---|
| 规则引擎实现 | 当前只写业务规则。 |
| API 参数和返回值 | 当前不写接口。 |
| 数据库规则表 | 当前不做数据库设计。 |
| Solidity 条件表达式 | 当前不写合约代码。 |
| 具体评分模型 | 当前不定义数学风控模型。 |
| hash / proof 规则 | 当前不进入底层证明细节。 |

## 16. 本步结论

P0 业务规则清单明确：

> ChainTrace P0 的规则不是技术规则引擎，而是业务推进口径。它决定案件走哪条路径、材料是否完整、事实是否满足、Agent 能否自动推进、银行是否需要审查、DeFi 合约是否允许执行、何时阻断、何时必须交给人。

下一步建议进入：

> P0 DeFi 合约业务规则清单，专门拆出哪些规则需要由智能合约执行或约束。
