成功标准不是“放款成功”，而是“决策更清楚”。

P0 的目标是证明 ChainTrace 能把分散的贸易事实组织成 proof graph、gate checklist、state explanation 和 audit summary。

Success criteria

试点结束时，客户应能判断是否进入 P1 集成。

交付物

P0 结束后交付的是决策包，不是生产系统。

deliverables
Proof Graph

客户案例的 proof nodes、依赖关系和缺口清单。

State Explanation

当前可以进入什么步骤、为什么被阻塞、谁负责补材料。

P1 Scope

明确下一阶段要接入的角色、系统、API 或业务流程。

失败也有价值

如果 P0 发现材料无法映射，也能避免错误投入。

failure modes
材料不足

客户当前材料无法支持融资风控。

角色缺失

关键事实责任方不愿参与。

价值不成立

重复审核成本不高，暂不值得集成。
