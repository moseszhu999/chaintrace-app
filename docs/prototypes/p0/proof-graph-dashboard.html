Proof Graph 把所有角色提交的 proof 收束成一张案件图。

它不是又一个角色，而是 ChainTrace 的核心状态层：判断哪些 proof 已提交、哪些 gate 通过、哪些 gate 阻塞、为什么不能放款。

核心判断

Proof Graph 给出机器可读解释，但最终风险动作仍需人类确认。

state reasoning
可以判断

哪些 proof 已存在、依赖是否满足、gate 是否通过、阻塞原因是什么。

可以解释

为什么可以继续银行审核、为什么还不能结算或放款。

不能执行

不能签名、不能写链、不能发交易、不能放款、不能修改任何原始 proof。
