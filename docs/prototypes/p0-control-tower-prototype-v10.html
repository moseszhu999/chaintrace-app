1. End-to-end financing control tower lane

source to funding result
P0

Exporter financing intent

Exporter starts a receivable financing / settlement case with buyer, goods, amount, route and requested outcome.

Need material

Trade materials

Contract, PO, invoice, packing list and supplement notes are reviewed as business evidence, not as a generic file list.

Agent

Agent gap check

Agent summarizes case context, finds gaps, creates role-owned to-dos and explains next action.

Fact

Inspection fact

Inspector confirms pass, fail or dispute. Inspection supports review but never approves funding.

P1 node

Logistics / customs / insurance

P0 keeps these as visible nodes or placeholders. Full workflows remain deferred.

P1 node

Buyer confirmation

Buyer confirmation, dispute and payment responsibility are visible as business facts, not a full buyer system.

Path-specific

Funder review

Only active in bank-assisted path. Funder may request materials, reject, or allow pilot review.

Execution guard

DeFi business check

Rule active, pool available, platform not paused and conditions satisfied. Contract does not judge trade truth.

Ops

Platform operation

Platform sees pool, rules, pause, failures and Agent backlog, but cannot forge facts or approve for a funder.

Exception

Blocker and recovery

Every blocked or failed state must show reason, owner, recovery condition and next step.

Result

Funding result

Preparing, need materials, under review, allowed, blocked, failed, completed or rejected are distinct.

Boundary

Completed only when funded

Allowed is not completed. Funder allowed is not funded. Inspection pass is not funding approval.

What the reviewer should understand in 3 minutes

ProductChainTrace P0 is a supply-chain finance control tower centered on one financing / settlement case.
Main user questionWhere is the case now, why is it blocked, who owns the next action and what is the funding result?
Agent NativeAgent summarizes, checks gaps, creates to-dos, suggests next action and escalates. It does not approve or create facts.
Crypto NativeDeFi contract state is shown as business output: allow, wait, block, pool insufficient, paused, failed, completed.
Review resultIf any stage cannot be understood in business language, the prototype is not ready to freeze.

3. Role views and hard boundaries

single wallet single role
Exporter

Creates and fixes the case

Can see own case, trade materials, material gap, Agent to-do, fact progress and funding result. Cannot see unrelated cases or edit fact-party conclusions.

Funder / bank

Reviews related cases

Can review relevant packages, request more information, allow pilot review or reject. Cannot modify inspection, logistics, buyer or insurance facts.

Fact party

Confirms assigned facts

Inspector, logistics, customs, insurance and buyer roles only see their assigned node and necessary context. They do not decide financing.

Platform operator

Handles operational blockers

Can view rules, pool, pause, execution failure and Agent backlog. Cannot forge exporter evidence, fact-party conclusions or funder approval.

4. Path difference panels

do not collapse into one bank approval path
Bank-assisted

Funder review is required

Funder reviews material completeness and facts, can request more information, approve for pilot review or reject. DeFi still checks rule, pool and pause after allowance.

Do not show bank allowance as completed funding.
Pure DeFi

Bank review is skipped

Facts, rules, pool state and guarantee conditions drive execution. If conditions are incomplete, the result is waiting, blocked or failed, not silent approval.

Do not force pure DeFi to wait for bank approval.
Low-human

Agent and contract handle low-risk flow

Agent can advance low-risk, authorized, rule-clear tasks. Major dispute, high risk, legal responsibility and execution failure go to human review.

Do not let Agent decide major disputes.

5. Agent boundary

Can doSummarize case, detect material gaps, create role-owned tasks, explain state, recommend next action, escalate exceptions.
Cannot doForge evidence, approve funding, submit inspection conclusion, judge disputes, bypass rules, change role permissions.
Walkthrough testEvery Agent output must point back to visible material, fact, role, rule or exception.

6. DeFi business output

OutputsAllowed, not allowed, waiting condition, rule blocked, pool insufficient, platform paused, execution failed, completed.
Cannot doJudge trade truth, approve funding, create facts, resolve disputes or replace platform recovery.
Walkthrough testEvery DeFi state must be explainable without Solidity, ABI, wallet signature or transaction detail.

7. Exception recovery matrix

blocked is not a dead end
Material gapExporter owns supplement. Agent creates to-do with completion standard.
Inspection disputeFact party and related business role own clarification. Automation pauses.
Funder needs more infoFunder request creates responsible task; exporter or fact party responds.
Rule blockedPlatform or responsible role handles rule explanation and recovery condition.
Pool insufficientPlatform / pool operation handles availability, currency and pause state.
Platform pausedPlatform explains reason, impact range and restore condition.
Execution failedPlatform handles failure explanation and next action; user sees result is not completed.
